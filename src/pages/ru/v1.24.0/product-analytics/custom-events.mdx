Пользовательские события отлично подходят для добавления аналитики за счёт записи событий, специфичных для вашей предметной области, вместе с повторами сессий. OpenReplay использует 2 типа событий: функциональные (`event`) и технические (`issue`). Все события индексируются для удобства поиска и синхронизируются с записями сессий.

<Aside type="tip">
  У нас также есть подробное руководство о том, как отправлять пользовательские события, вы можете ознакомиться с ним [здесь](/tutorials/custom-events).
</Aside>

## Функциональные события

Функциональные события индексируются и упрощают поиск конкретных записей сессий. Если они были успешно получены OpenReplay, они будут доступны в качестве фильтров в строке omnisearch.

![Functional Event](/static/custom-events-1.png#center)

### С помощью NPM

Используйте метод `event` для сигнализации о функциональных событиях, таких как *order completed* или *product added*. Он принимает 2 параметра: *name* (string) и *payload* (нерекурсивный объект JS).

```js
tracker.event('product_added', 'shoes'); // after tracker.start()
```

### С помощью JavaScript-сниппета

Ниже приведён пример отправки функционального события с помощью сниппета.

## Технические события

Технические события отображаются в повторе сессии на вкладке *Events* в DevTools, а также в виде аннотаций при воспроизведении. Они также учитываются в воронках (Funnels) для сопоставления падений конверсии с техническими проблемами.

![Technical Event](/static/custom-events-2.png#center)

### С помощью NPM

`issue` используется для отправки технических событий, таких как ошибки, которые могут возникать в вашем стеке или других нижестоящих системах. `issue` принимает 2 параметра: *name* (string) и *payload* (нерекурсивный объект JS).

```js
tracker.issue('payment_error', {code: 500, context: 42}); // after tracker.start()
```

### С помощью JavaScript-сниппета

Ниже приведён пример отправки технического события с помощью сниппета.

