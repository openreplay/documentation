Этот метод позволяет вам сообщать о «функциональном событии». С помощью функциональных событий вы можете отмечать моменты в сессии пользователя, которые важны с функциональной точки зрения и которые наш трекер по умолчанию не может распознать и отметить самостоятельно. Примером функционального события может служить добавление пользователем товара в корзину покупок. Хотя наш трекер сможет записать это действие, он не сможет отметить его как значимое событие на временной шкале вашей сессии. Однако благодаря этому методу вы можете сделать это вручную.

Подробнее см. в разделе [пользовательские события](/installation/custom-events#functionalevents).

## Сигнатура

```js
import OR from '@openreplay/react-native'

OR.tracker.event("name", JSON.stringify({ key: "value" }))
```

## Параметры

- `key: string`: Имя ключа, под которым будет сохранена информация. Этот ключ можно использовать для фильтрации и поиска внутри платформы OpenReplay.
- `payload: string`: Содержимое события. Это могут быть любые данные, закодированные в виде строки (JSON).
