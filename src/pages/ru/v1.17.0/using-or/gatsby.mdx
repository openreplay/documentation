Установка трекера OpenReplay в проект на основе Gatsby выполняется относительно просто.

![Sesion replay of a Gatsby site](/images/gatsby-ss.png)

Учитывая, что Gatsby использует React под капотом, всё, что нам нужно сделать, — это добавить код трекера на главную страницу и вызвать метод `start` с помощью хука `useEffect`.

Давайте посмотрим.

## Зачем настраивать трекер на статическом сайте?

Отсутствие динамического контента в вашем приложении не означает, что вы не захотите воспользоваться некоторыми другими возможностями OpenReplay.

Хотя вы не будете использовать повторы сессий для поиска ошибок, вы сможете собирать интересные метрики и создавать [собственные настраиваемые дашборды](https://docs.openreplay.com/tutorials/custom-dashboard) на основе этих метрик.

Вы можете использовать метрики производительности, web vitals и даже метрики, связанные с ресурсами, чтобы получить общее представление о том, какой опыт получают пользователи на вашем сайте.

## Настройка трекера внутри Gatsby

Если у вас статический сайт на Gatsby, точкой входа которого является файл `index.tsx` внутри папки `pages`, вам следует добавить в него следующий код:

```jsx
import Tracker from '@openreplay/tracker';
const tracker = new Tracker({
  projectKey: process.env.GATSBY_OPENREPLAY_PROJECT_KEY
});

const Index = ({data, location }) => {

  React.useEffect(() => {
    tracker.start();
  }, [])

   //the rest of your code here...
}
```

Трекер создаётся при компиляции страницы (будь то SSR или рендеринг на стороне клиента), но метод start может быть вызван только из браузера, поэтому нам нужно убедиться, что он вызывается после монтирования компонента (отсюда и хук `useEffect`). Пустой массив в качестве второго аргумента хука `useEffect` гарантирует, что колбэк выполнится только один раз.

💡**Примечание для пользователей self-hosted:** если вы используете self-hosted версию OpenReplay, вам также потребуется указать свойство конфигурации `ingestPoint` при создании трекера. Это свойство задаёт конечную точку приёма данных трекера. Пользователям облака это не нужно, поскольку по умолчанию трекер знает, где находится SaaS-версия этой конечной точки, но если вы размещаете его самостоятельно, вам нужно будет её указать (это должно быть что-то вроде `https://openreplay.mydomain.com/ingest`)

## Работа с переменной окружения в Gatsby

В целях безопасности мы рекомендуем не прописывать ключ проекта жёстко прямо в коде. Это означает, что вам нужно будет экспортировать его как переменную окружения, чтобы Gatsby мог её прочитать.

Ключ проекта OpenReplay будет затем храниться в переменной с именем `GATSBY_OPENREPLAY_PROJECT_KEY`

Обратите внимание, что имя начинается с префикса `GATSBY_`, который указывает Gatsby сделать переменную доступной и во фронтенд-коде. В противном случае вы не сможете прочитать её значение, если только не выполняете код Node.js.
