Заставить трекер работать в приложении Vue довольно просто. 

## Добавление кода отслеживания

Хотя вы можете [получить версию кода для копирования и вставки прямо с платформы при создании нового проекта](/using-or/snippet#getting-the-snippet), этот код слишком упрощённый и сильно усложнит его настройку и добавление дополнительных плагинов.

Если вы создаёте SPA (Single Page Application), всё, что вам нужно сделать, — это добавить следующий код в скрипт setup в вашем основном файле.

```html
<script setup>
//... your code here
const tracker = new Tracker({
  projectKey: "<your project key>"
});

tracker.start();
//... more code here
</script>
```


<Aside type="tip">
**Примечание для пользователей self-hosted**: если вы используете self-hosted версию OpenReplay, вам также потребуется указать свойство конфигурации `ingestPoint`. 
Это свойство задаёт конечную точку приёма данных трекера. Пользователям облачной версии оно не нужно, 
потому что по умолчанию трекер уже знает, где находится SaaS-версия этой конечной точки, но если вы размещаете его самостоятельно, вам нужно будет её указать 
(она должна выглядеть примерно так: `https://openreplay.mydomain.com/ingest`) 
</Aside>

### Работа с «projectKey» в вашем коде

Обратите внимание, что `projectKey`, который вы настраиваете, — это тот, который предоставляется платформой OpenReplay. 
По соображениям безопасности это значение не следует хранить жёстко прописанным в коде, а лучше в каком-либо конфигурационном файле/системе.

Один из вариантов — добавить переменную окружения в файл `.env` в корне вашего проекта и добавить к переменной префикс `VITE_`, что сделает её доступной в вашем коде Vue.
Позже вы сможете использовать её с помощью `import.meta.env.<your variable name>`.
Вот так:

```javascript
const tracker = new Tracker({
  projectKey: import.meta.env.VITE_VUE_APP_PROJECT_KEY,
});
 ```

## Создаёте что-то более сложное?
Если вы создаёте сложное приложение на Vue, скорее всего, вы используете [Nuxt](/using-or/nuxt) или что-то подобное. Обязательно загляните в наш [раздел Настройка](/using-or), чтобы найти фреймворк, с которым вы работаете.
