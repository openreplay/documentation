Запустить трекер в React-приложении довольно просто. 

## Добавление кода отслеживания

Хотя вы можете [получить версию кода для копирования и вставки прямо с платформы при создании нового проекта](/using-or/snippet#getting-the-snippet), этот код слишком упрощён и значительно усложнит настройку и добавление дополнительных плагинов.

Предполагая, что вы создаёте SPA (Single Page Application), всё, что вам нужно сделать, — это добавить следующий код в точку входа вашего приложения:

```jsx
import Tracker from '@openreplay/tracker';

const tracker = new Tracker({
  projectKey: "<your project key>",  
});

function YourMainComponent({props}) {
    //some code here...

    useEffect(() => {
        tracker.start();
    }, [])

    return //....
}
```

Другими словами, вы можете создать экземпляр трекера в начале скрипта, а после загрузки страницы запустить его с помощью хука `useEffect`.

<Aside type="tip">
**Примечание для пользователей self-hosted-версии**: если вы используете self-hosted-версию OpenReplay, вам также потребуется указать свойство конфигурации `ingestPoint`. 
Это свойство задаёт конечную точку приёма (endpoint) данных трекера. Облачным пользователям это не нужно, 
поскольку по умолчанию трекер уже знает, где находится SaaS-версия этой конечной точки, но если вы размещаете его самостоятельно (self-hosting), вам нужно будет её указать 
(это должно быть что-то вроде `https://openreplay.mydomain.com/ingest`) 
</Aside>

### Работа с «projectKey» в вашем коде

Обратите внимание: `projectKey`, который вы настраиваете, — это значение, предоставляемое платформой OpenReplay. По соображениям безопасности это значение не следует хранить жёстко прописанным (hardcoded) в коде, а лучше — в каком-либо файле/системе конфигурации.

Существуют разные способы решения этой задачи; в зависимости от вашей настройки вы можете использовать, например, [поддержку переменных окружения (ENV) в `create-react-app`](https://create-react-app.dev/docs/adding-custom-environment-variables/).
