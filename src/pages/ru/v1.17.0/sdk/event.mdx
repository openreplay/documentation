Этот метод позволяет вам сообщать о «функциональном событии». С помощью функциональных событий вы можете отмечать моменты в сессии пользователя, которые важны с функциональной точки зрения и которые наш трекер по умолчанию не может уловить и пометить самостоятельно. Примером функционального события может служить добавление пользователем товара в корзину. Хотя наш трекер сможет записать это действие, он не будет знать, что его следует пометить как значимое событие на временной шкале вашей сессии. Однако благодаря этому методу вы можете сделать это вручную.

<Aside type="tip">
  У нас также есть подробное руководство о том, как отправлять пользовательские события, вы можете ознакомиться с ним [здесь](/tutorials/custom-events).
</Aside>

Дополнительные сведения см. в разделе [пользовательские события](/installation/custom-events#functionalevents).

## Сигнатура

```js 
event(key: string, payload?: any, issue?: boolean): void
```

## Параметры

- `key: string`: Имя ключа, под которым будет сохранена информация. Этот ключ можно использовать для фильтрации и поиска в платформе OpenReplay.
- `payload: any`: Содержимое события. Это могут быть любые сериализуемые данные.
- `issue: boolean` : Указывает, является ли это функциональным событием (если установлено значение `false`) или «техническим событием» (если установлено значение `true`). По умолчанию его значение равно `false`.
