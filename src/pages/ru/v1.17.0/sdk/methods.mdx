## Методы

- [Constructor](/sdk/constructor): Узнайте всё о параметрах инициализации трекера.
- [event()](/sdk/event): С помощью этого метода отправляйте пользовательские события для записи в составе ваших воспроизведений.
- [getSessionToken()](/sdk/get-session-token): Получите токен текущей активной сессии.
- [getSessionID()](/sdk/get-session-id): Получите ID текущей активной сессии.
- [getSessionURL()](/sdk/get-session-url): Получите url для воспроизведения текущей сессии. 
- [handleError()](/sdk/handle-error): Захватывайте информацию о пользовательских ошибках и сохраняйте её в составе вашей записи.
- [isActive()](/sdk/is-active): Получите текущее состояние активной сессии.
- [issue()](/sdk/issue): Захватывайте пользовательские технические события и сохраняйте их в составе воспроизведения.
- [setUserID()](/sdk/set-user-id): Идентифицируйте ваших пользователей с помощью этого метода.
- [setMetadata()](/sdk/set-metadata): Установите поле метаданных и сохраните его в составе записи.
- [start()](/sdk/start): Начните запись сессии.
- [stop()](/sdk/stop): Остановите запись сессии с помощью этого метода.
- [clearPersistFlag()](/sdk/clear-persist-flag): Удаляет постоянные Feature Flags из sessionStorage браузера.
- [getFeatureFlag()](/sdk/get-feature-flag): Возвращает `IFeatureFlag` по ключу, если он существует.
- [onFlagsReload()](/sdk/on-flags-reload): Устанавливает callback, который будет использоваться при загрузке Feature Flags.
- [isFlagEnabled()](/sdk/is-flag-enabled): Возвращает `true`, если флаг с данным ключом существует и активен.
- [reloadFlags()](/sdk/reloadFlags): Перезагружает все флаги (может быть полезно, если в сессии пользователя произошли изменения метаданных, userId или любой другой информации, которая может потребоваться для определённых условий флагов).
- [getAllFeatureFlags()](/sdk/get-all-feature-flags): Возвращает все флаги (`IFeatureFlag[]`), активные в данный момент.
- [coldStart()](/sdk/cold-start): Начинает буферизацию сообщений (обновляя буфер) с возможностью вызова `start()` для включения записи сессии и отправки содержимого буфера.
- [startOfflineRecording()](/sdk/start-offline-recording): Запускает офлайн-запись сессии, которую можно отправить с помощью `uploadOfflineRecording`.
- [uploadOfflineRecording()](/sdk/upload-offline-recording): Загружает сохранённый буфер сессии на бэкенд.
- [forceFlushBatch()](/sdk/force-flush-batch): Принудительно отправляет текущий пакет сообщений, не дожидаясь его завершения.
- [trackWs()](/sdk/track-ws): Создаёт хук для захвата сообщений WebSocket.
