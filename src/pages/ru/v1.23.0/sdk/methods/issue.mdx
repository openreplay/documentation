Этот метод можно использовать для регистрации "[технического события](/installation/custom-events)", также известного как "issue" (проблема). Такие проблемы могут использоваться для передачи дополнительной информации о проблеме, которую трекер не может обнаружить по умолчанию. Хорошим примером может служить ситуация, когда пользователь пытается оплатить покупку на нашем сайте электронной коммерции и получает ошибку от платёжного шлюза. Хотя трекер мог бы записать payload с ошибкой, он не определил бы это как проблему. Этот метод позволяет вам пометить такое поведение как issue для дальнейшей фильтрации сессий.

<Aside type="tip">
  У нас также есть подробное руководство о том, как отслеживать пользовательские события, вы можете ознакомиться с ним [здесь](/tutorials/custom-events).
</Aside>

## Сигнатура

```js
issue(key: string, payload?: any): void
```

## Параметры

- `key: string`: Имя ключа, под которым будут сохранены данные события. Вы можете использовать это значение в полях фильтра внутри платформы OpenReplay.
- `payload: any`: Данные, связанные с отслеживаемым событием. Здесь можно сохранить любой сериализуемый объект.

## Возвращаемое значение

Этот метод не возвращает никакого значения, вы сможете увидеть отслеживаемую информацию внутри платформы OpenReplay.
