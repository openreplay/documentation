## Методы

- [Constructor](/sdk/constructor): Узнайте всё о параметрах инициализации трекера.
- [event()](/sdk/methods/event): Отправляйте пользовательские события, чтобы они записывались в составе ваших повторов, с помощью этого метода.
- [getSessionToken()](/sdk/methods/get-session-token): Получите токен текущей активной сессии.
- [getSessionID()](/sdk/methods/get-session-id): Получите ID текущей активной сессии.
- [getSessionURL()](/sdk/methods/get-session-url): Получите url для воспроизведения текущей сессии. 
- [handleError()](/sdk/methods/handle-error): Захватывайте информацию о пользовательских ошибках и сохраняйте её в составе вашей записи.
- [isActive()](/sdk/methods/is-active): Получите текущее состояние активной сессии.
- [issue()](/sdk/methods/issue): Захватывайте пользовательские технические события и сохраняйте их в составе повтора.
- [setUserID()](/sdk/methods/set-user-id): Идентифицируйте ваших пользователей с помощью этого метода.
- [setMetadata()](/sdk/methods/set-metadata): Задайте поле метаданных и сохраните его в составе записи.
- [start()](/sdk/methods/start): Начните запись сессии.
- [stop()](/sdk/methods/stop): Остановите запись сессии с помощью этого метода.
- [coldStart()](/sdk/methods/cold-start): Начинает буферизацию сообщений (обновляя буфер) с возможностью вызвать `start()`, чтобы включить запись сессии и отправить содержимое буфера.
- [startOfflineRecording()](/sdk/methods/start-offline-recording): Запускает офлайн-запись сессии, которую можно отправить с помощью `uploadOfflineRecording`.
- [uploadOfflineRecording()](/sdk/methods/upload-offline-recording): Загружает сохранённый буфер сессии на бэкенд.
- [forceFlushBatch()](/sdk/methods/force-flush-batch): Принудительно отправляет текущий пакет сообщений, не дожидаясь его завершения.
- [trackWs()](/sdk/methods/track-ws): Создаёт хук для захвата сообщений WebSocket.
- [incident()](/sdk/methods/incident): Отмечайте значимые события или инциденты во время сессии пользователя.
