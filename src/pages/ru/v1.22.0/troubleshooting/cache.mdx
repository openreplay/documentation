OpenReplay требуется доступ к ресурсам, таким как ваши таблицы стилей, иконки и шрифты, чтобы воспроизведение сессий работало. Фактически эти файлы копируются и затем кэшируются нашим бэкендом, чтобы вы могли просматривать старые записи, даже если ваше веб-приложение изменилось. Выполните приведённые ниже шаги, чтобы очистить кэш:

1. Выполните `k9s -n db`
2. Используйте стрелки клавиатуры для навигации по списку и перейдите к контейнеру `minio-*`
3. Нажмите s, чтобы получить доступ к оболочке контейнера Minio (объектное хранилище)
4. Выполните `export MINIO_ACCESS_KEY=<accessKey from vars.yaml>`
5. Выполните `export MINIO_SECRET_KEY=<secretKey from vars.yaml>`
6. Выполните `mc alias set minio http://localhost:9000 $MINIO_ACCESS_KEY $MINIO_SECRET_KEY`
7. Выполните `mc rm --recursive --dangerous --force minio/sessions-assets`, чтобы удалить все кэшированные файлы (CSS, шрифты и иконки)
8. Используйте `exit`, чтобы выйти из контейнера Minio
9. Выполните `:quit`, чтобы выйти из CLI Kubernetes
10. Перезапустите сервис `assets`, выполнив `openreplay -R`
