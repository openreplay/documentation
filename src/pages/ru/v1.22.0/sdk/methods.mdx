## Методы

- [Constructor](/sdk/constructor): Узнайте всё о параметрах инициализации трекера.
- [event()](/sdk/methodsevent): С помощью этого метода отправляйте пользовательские события, чтобы они записывались как часть ваших повторов.
- [getSessionToken()](/sdk/methods/get-session-token): Получить токен текущей активной сессии.
- [getSessionID()](/sdk/methods/get-session-id): Получить ID текущей активной сессии.
- [getSessionURL()](/sdk/methods/get-session-url): Получить URL для воспроизведения текущей сессии. 
- [handleError()](/sdk/methods/handle-error): Захватывайте информацию о пользовательских ошибках и сохраняйте её как часть вашей записи.
- [isActive()](/sdk/methods/is-active): Получить текущее состояние активной сессии.
- [issue()](/sdk/methods/issue): Захватывайте пользовательские технические события и сохраняйте их как часть повтора.
- [setUserID()](/sdk/methods/set-user-id): Идентифицируйте своих пользователей с помощью этого метода.
- [setMetadata()](/sdk/methods/set-metadata): Установите поле метаданных и сохраните его как часть записи.
- [start()](/sdk/methods/start): Начать запись сессии.
- [stop()](/sdk/methods/stop): Остановить запись сессии с помощью этого метода.
- [clearPersistFlag()](/sdk/methods/clear-persist-flag): Удаляет постоянные Feature Flags из sessionStorage браузера.
- [getFeatureFlag()](/sdk/methods/get-feature-flag): Возвращает `IFeatureFlag` по ключу, если он существует.
- [onFlagsReload()](/sdk/methods/on-flags-reload): Устанавливает callback, который будет использоваться при загрузке Feature Flags.
- [isFlagEnabled()](/sdk/methods/is-flag-enabled): Возвращает `true`, если флаг с этим ключом существует и активен.
- [reloadFlags()](/sdk/methods/reload-flags): Перезагружает все флаги (может быть полезно, если в сессии пользователя произошло какое-либо изменение метаданных, userId или любой другой информации, которая может потребоваться для определённых условий флагов).
- [getAllFeatureFlags()](/sdk/methods/get-all-feature-flags): Возвращает все флаги (`IFeatureFlag[]`), которые активны в данный момент.
- [coldStart()](/sdk/methods/cold-start): Начинает буферизацию сообщений (обновляя буфер) с возможностью вызвать `start()`, чтобы включить запись сессии и отправить содержимое буфера.
- [startOfflineRecording()](/sdk/methods/start-offline-recording): Запускает офлайн-запись сессии, которую можно отправить с помощью `uploadOfflineRecording`.
- [uploadOfflineRecording()](/sdk/methods/upload-offline-recording): Загружает сохранённый буфер сессии на бэкенд.
- [forceFlushBatch()](/sdk/methods/force-flush-batch): Принудительно отправляет текущий пакет сообщений, не дожидаясь его завершения.
- [trackWs()](/sdk/methods/track-ws): Создаёт хук для захвата сообщений WebSocket.
