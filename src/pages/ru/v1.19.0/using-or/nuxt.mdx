Запустить трекер в приложении на Nuxt довольно просто.

## Добавление кода отслеживания

Хотя вы можете получить готовую для копирования версию кода прямо с платформы при создании нового проекта, этот код слишком примитивен и может не соответствовать лучшим практикам, которых ожидает ваша команда или компания.

Поэтому вместо этого вы можете попробовать модуляризировать предоставленный код следующим образом:

```jsx
import Tracker from '@openreplay/tracker';
import {v4 as uuidV4} from 'uuid'

function defaultGetUserId() {
   return uuidV4() 
}

export function startTracker(config) {

    console.log("Starting tracker...")

    const getUserId = (config?.userIdEnabled && config?.getUserId) ? config.getUserId : defaultGetUserId
    let userId = null;

    const trackerConfig = {
        projectKey: config?.projectKey
    }

    const tracker = new Tracker(trackerConfig);

    if(config?.userIdEnabled) {
        userId = getUserId()
        tracker.setUserID(userId)
    }

    tracker.start();
    return {
        tracker,
        userId
    }
}
```

Эта функция отвечает за создание экземпляра Tracker и вызов метода `start`, который запускает весь процесс.

Эта функция также принимает объект «конфигурации», который вы можете расширять для будущих возможностей. В этом примере показано, как опционально включить возможность уникальной идентификации пользователей. Вы можете использовать UUID по умолчанию, который будет уникальным для каждого пользователя при каждом посещении страницы. В качестве альтернативы вы можете предоставить пользовательскую функцию `getUserId` в составе конфигурации.

С помощью вашей пользовательской функции вы можете передавать ID клиента вашего магазина, адрес электронной почты пользователя или что-либо ещё, что уникально идентифицирует пользователя внутри вашего приложения. Таким образом, вы можете отслеживать его session replays на протяжении времени на случай, если у него возникнет конкретная проблема, о которой другие не сообщают.

В этом примере ожидается, что ключ проекта, который вы можете получить в интерфейсе проектов OpenReplay, будет передан внутри объекта `config` в ключе `projectKey`.

## Написание плагина только для клиента

Чтобы использовать эту функцию `startTracker`, вам нужно создать плагин только для клиента в каталоге `plugins` вашего проекта Nuxt.

Обязательно добавьте суффикс `.client` к имени файла вашего плагина. Это нужно для того, чтобы Nuxt загружал его только на клиенте.

Код плагина может выглядеть так:

```jsx
import {startTracker} from '../utils/tracker'

export default defineNuxtPlugin( (nuxtApp) => {
    
    return {
        provide: {
            startTracking: () => {
                let config = useRuntimeConfig().public
                
                let {userId} = startTracker({
                    userIdEnabled: true,
                    projectKey: config.openreplayProjectKey
                })
                //Optional if you need it
                let uid = useUserId()
                uid.value = userId
            }
        }
    }
})
```

Если вы хотите узнать больше о создании плагинов Nuxt, можете ознакомиться с [их документацией](https://v3.nuxtjs.org/guide/directory-structure/plugins).

Итак, этот плагин предоставляет функцию под названием `startTracking`, которая отвечает за вызов функции `startTracker`, определённой нами ранее. Но что особенно важно, внутри кода плагина вы можете получить переменную окружения, в которой вы сохранили ключ проекта. Помните, что эта информация не должна быть жёстко прописана (hard-coded) внутри вашего проекта из соображений безопасности.

С помощью функции `useRuntimeConfig` вы можете получить доступ к публичным переменным окружения. В этом примере я обращаюсь к `openreplayProjectKey`, которая определена как `NUXT_PUBLIC_OPENREPLAY_PROJECT_KEY`.

💡 **Обратите внимание** на префикс `NUXT_PUBLIC` — он гарантирует, что Nuxt предоставит эту переменную фронтенду; в противном случае она будет доступна только коду вашего бэкенда.

Приведённый выше пример также показывает, как использовать возвращаемое значение ID пользователя вместе с composable, чтобы делиться этим значением между компонентами. Эти две строки полностью необязательны.

## Использование трекера

После того как плагин сохранён в файл, вы можете использовать его следующим образом из вашего файла `app.vue`.

```jsx
<script setup>
onMounted( () => {
  const {$startTracking} = useNuxtApp()
  $startTracking()
})
</script>

<template>
  <div>
    <NuxtPage />
  </div>
</template>
```

Обязательно вызывайте трекер после того, как сработало событие mounted. Это можно сделать с помощью composable `onMounted`. После этого трекер должен начать отправлять данные на платформу после развёртывания.
