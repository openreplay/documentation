## Очистка данных воспроизведения

Вы можете очищать то, что захватывает OpenReplay, как на глобальном, так и на детальном уровне. Любые скрытые или игнорируемые данные не будут захвачены в источнике. Другими словами, эти данные никогда не покинут браузер пользователя.

### Глобальный уровень

Данные можно очищать на уровне трекера при настройке OpenReplay.

#### Я использую OpenReplay Script

![Конфигурация проекта в OpenReplay](/static/gdpr-1.png#center)

1) Нажмите на *Tracking Code* вашего проекта в разделе 'Preferences > Projects'.
2) Параметры записи данных должны появиться прямо перед фрагментом кода:
- **Inputs**: Чтобы игнорировать, скрывать или записывать все вводимые значения ваших пользователей.
- **Do not record any numeric text**: Чтобы игнорировать числовые значения, содержащиеся в текстовых элементах.
- **Do not record email addresses**: По умолчанию все адреса электронной почты, содержащиеся в текстовых элементах, скрываются и не будут отображаться в записях. Вы можете отключить это, сняв флажок.
3) Изменение параметров записи данных влияет на фрагмент кода. Обязательно скопируйте/вставьте обновлённый скрипт в ваше веб-приложение.

#### Я использую OpenReplay NPM

Существует набор связанных с конфиденциальностью параметров, которые вы можете передать в конструктор:
- `obscureTextEmails?: boolean` Скрывает адреса электронной почты в текстовых элементах. Адреса электронной почты будут преобразованы в случайную цепочку звёздочек. По умолчанию: true.
- `obscureTextNumbers?: boolean` Скрывает числа в текстовых элементах. Числа будут преобразованы в случайную цепочку звёздочек. По умолчанию: false.
- `obscureInputEmails?: boolean` Скрывает адреса электронной почты в полях ввода. Значения электронной почты будут преобразованы в случайную цепочку звёздочек. По умолчанию: true.
- `obscureInputNumbers?: boolean` Скрывает числа в полях ввода. Числовые значения будут преобразованы в случайную цепочку звёздочек. По умолчанию: false.
- `obscureInputDates?: boolean` Скрывает даты в полях ввода. Значения дат будут преобразованы в случайную цепочку звёздочек. По умолчанию: false.
- `defaultInputMode?: 0 | 1 | 2` Режим захвата по умолчанию для вводимых значений. Соответственно: обычный, скрытый или игнорируемый. По умолчанию: 1 (скрытый).

Очищенные текст, адреса электронной почты и числа скрываются или удаляются перед отправкой данных на серверы OpenReplay. Поэтому изменения, применённые к указанным выше параметрам, не могут быть ретроактивными и будут применяться только к вновь собранным данным.

### Детальный уровень

Вы также можете очищать ваши данные на уровне кода для большей детализации. Это полезно для скрытия или игнорирования элементов DOM и полей ввода.

#### Использование атрибутов HTML

- `data-openreplay-obscured` для маскирования текстового содержимого тегов `<input>`, `<img>` и других элементов HTML (например, `<div>`) за исключением `<svg>` и `<canvas>`.

```html
<!--Obscuring input fields-->
<form action="/action_page.php">
  Name: <input type="text" name="fname" data-openreplay-obscured><br>
  <input type="submit" value="Submit">
</form>
<!--Obscuring a div element-->
<div style="background-color:lightblue" data-openreplay-obscured>
  <h3>This is a sensitive information</h3>
  <p>This is an important paragraph</p>
  <img src="important_image.jpg">
</div>
```

- `data-openreplay-hidden` для удаления содержимого тегов `<input>`, `<img>`, `<svg>` и других элементов HTML (например, `<div>`) за исключением `<canvas>`. Содержимое элемента (включая его дочерние компоненты) удаляется, но его props и аргументы сохраняются, как в этом примере:

```html
<div style="background-color:lightblue" data-openreplay-hidden>
  <div>This is a sensitive information</div>
  <svg>An important chart</svg>
</div>
```

Это приведёт к тому, что трекер запишет следующий DOM:

```html
<div style="background-color:lightblue" data-openreplay-hidden>
</div>
```

#### Использование функции domSanitizer

- Функция `domSanitizer: (node: Element) => SanitizeLevel` позволяет избежать необходимости инструментировать каждый компонент HTML, который нужно очистить. `SanitizeLevel` может быть `Plain` (0), `Obscured` (1) или `Hidden` (2). Эта функция передаётся в конструктор трекера. Ниже приведён пример удаления всех элементов HTML, имеющих определённое имя CSS-класса:

```js
// Import SanitizeLevel enum
import Tracker, { SanitizeLevel } from "@openreplay/tracker";

const tracker = new Tracker({
  projectKey: PROJECT_KEY,
  domSanitizer: (node: Element) => {
    const elementClassNames = node.classList

    if (elementClassNames.contains('to_be_redacted_class')) {
      return SanitizeLevel.Hidden
    }

    if (node.id === 'sensitiveData') {
      return SanitizeLevel.Obscured
    }

    return SanitizeLevel.Plain
  }
})
tracker.start()
```

> **Примечание:** Все очищенные элементы маскируются/удаляются на уровне источника (трекера), чтобы конфиденциальные данные никогда не достигали экземпляра OpenReplay.

## Очистка данных DevTools

Вы можете использовать SDK OpenReplay для очистки сетевых данных и состояния приложения.

### Сеть

Захват запросов/ответов может регистрировать крайне конфиденциальную или личную информацию, которую ваше приложение может отправлять/получать. Обязательно надлежащим образом удаляйте ваши сетевые данные.

#### Fetch/XHR

Используйте параметр `sanitizer` для удаления конфиденциальных данных из запросов/ответов Fetch/XHR. Подробнее см. [сетевые параметры](/installation/network-options#options).

Также ознакомьтесь с этим пошаговым руководством о том, как [захватывать и очищать сетевые запросы](/tutorials/capture-request).

#### GraphQL

Чтобы очистить данные GraphQL, вам необходимо клонировать ключевые переменные и результаты операции перед вызовом трекера. Подробнее см. как [очищать записанные данные в GraphQL](/tutorials/graphql/#sanitizing-the-recorded-data).

### Управление состоянием

Удаляйте конфиденциальные данные из вашего хранилища состояния frontend с помощью SDK OpenReplay.

#### Redux

Используйте параметры `actionFilter`, `actionTransformer` и `stateTransformer` для удаления конфиденциальных действий и поддеревьев из вашего хранилища. Подробнее см. [документацию плагина redux](/plugins/redux/#options).

#### VueX

Используйте параметры `filter`, `transformer` и `mutationTransformer` для удаления конфиденциальных мутаций и поддеревьев из вашего хранилища. Подробнее см. [документацию плагина vuex](/plugins/vuex/#options).

#### Zustand

Используйте параметры `filter`, `transformer` и `mutationTransformer` для удаления конфиденциальных мутаций и поддеревьев из вашего хранилища. Подробнее см. [документацию плагина zustand](/plugins/zustand/#options).

#### Pinia

Используйте параметры `filter`, `transformer` и `mutationTransformer` для удаления конфиденциальных мутаций и поддеревьев из вашего хранилища. Подробнее см. [документацию плагина pinia](/plugins/pinia/#options).

#### NgRx

Используйте параметры `actionFilter`, `actionTransformer` и `stateTransformer` для удаления конфиденциальных действий и поддеревьев из вашего хранилища. Подробнее см. [документацию плагина ngrx](/plugins/ngrx/#options).
