Этот метод позволяет вам сообщать о «функциональном событии». С помощью функциональных событий вы можете отмечать моменты в сессии пользователя, которые важны с функциональной точки зрения и которые по умолчанию наш трекер не может зафиксировать и отметить самостоятельно. Примером функционального события может быть добавление пользователем товара в корзину покупок. Хотя наш трекер сможет записать это действие, он не будет знать, что его нужно отметить как значимое событие на временной шкале вашей сессии. Однако благодаря этому методу вы можете сделать это вручную.

См. [пользовательские события](/installation/custom-events#functionalevents) для получения дополнительных сведений.

## Сигнатура

```swift
ORTracker.shared.event(name: String, payload: Encodable?)
```

## Параметры

- `key: string`: Имя ключа, под которым будет сохранена информация. Этот ключ может использоваться для фильтрации и поиска внутри платформы OpenReplay.
- `payload: Encodable`: Содержимое события. Это могут быть любые кодируемые (JSON) данные.
