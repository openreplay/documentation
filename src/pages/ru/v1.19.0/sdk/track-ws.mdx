Если информации, которую наш трекер собирает по умолчанию при возникновении ошибки JS, недостаточно, вы можете использовать этот метод. Метод `handleError` вручную сообщает о перехваченных исключениях, отклонённых промисах или событиях ошибок. Подробнее см. в разделе [сообщение об ошибках](/installation/error-reporting#manuallyloggingexceptions).

## Сигнатура

```ts 
trackWs: (channelName: string) => (msgType: string, data: string, dir: 'up' | 'down') => void
```

## Параметры

- `channelName: string`: Имя WS-канала.

## Возвращаемое значение

Возвращает хук для применения к ws-сообщениям, который принимает в качестве параметров тип сообщения (или имя), данные и направление сообщения (up/down === отправлено/получено).
