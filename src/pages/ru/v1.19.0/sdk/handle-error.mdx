Если информации, которую наш трекер отслеживает по умолчанию при возникновении ошибки JS, недостаточно, вы можете использовать этот метод. Метод `handleError` вручную сообщает о перехваченных исключениях, отклонённых промисах или событиях ошибок. Подробнее см. в разделе [отчёты об ошибках](/installation/error-reporting#manuallyloggingexceptions).

## Сигнатура

```js 
handleError: (e: Error | ErrorEvent | PromiseRejectionEvent, metadata?: Record<string, any>) => void
```

## Параметры

- `e: Error | ErrorEvent | PromiseRejectionEvent`: Сам перехватываемый объект ошибки.
- `metadata: Record<string, any>`: **[Необязательно]** Структура наподобие словаря со всей дополнительной информацией, которую вы хотите сохранить в связи с этой ошибкой.

## Возвращаемое значение

Этот метод не возвращает никаких значений; сохранённую информацию можно увидеть внутри платформы OpenReplay.
