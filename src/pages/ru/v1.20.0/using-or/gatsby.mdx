Установка трекера OpenReplay в проект на основе Gatsby довольно проста.

![Sesion replay of a Gatsby site](/images/gatsby-ss.png)

Учитывая, что под капотом Gatsby использует React, всё, что нам нужно сделать, — это добавить код трекера на главную страницу и вызвать метод `start` с помощью хука `useEffect`.

Давайте посмотрим.

## Зачем настраивать трекер на статическом сайте?

Отсутствие динамического контента в вашем приложении не означает, что вы не захотите воспользоваться некоторыми другими возможностями OpenReplay.

Хотя вы не будете использовать повторы сессий для поиска багов, вы сможете собирать интересные метрики и создавать [собственные кастомные дашборды](https://docs.openreplay.com/tutorials/custom-dashboard) на основе этих метрик.

Вы можете использовать метрики производительности, web vitals и даже метрики, связанные с ресурсами, чтобы получить общее представление о том, каким для ваших пользователей является взаимодействие с вашим сайтом.

## Настройка трекера внутри Gatsby

Для статического сайта на Gatsby, точкой входа которого является файл `index.tsx` в папке `pages`, вам нужно добавить в него следующий код:

```jsx
import Tracker from '@openreplay/tracker';
const tracker = new Tracker({
  projectKey: process.env.GATSBY_OPENREPLAY_PROJECT_KEY
});

const Index = ({data, location }) => {

  React.useEffect(() => {
    tracker.start();
  }, [])

   //the rest of your code here...
}
```

Трекер создаётся при компиляции страницы (будь то SSR или рендеринг на стороне клиента), но метод start может быть вызван только из браузера, поэтому нам нужно убедиться, что он вызывается после монтирования компонента (отсюда и хук `useEffect`). Пустой массив в качестве второго аргумента хука `useEffect` гарантирует, что колбэк выполнится только один раз.

💡**Примечание для пользователей self-hosted:** если вы используете self-hosted версию OpenReplay, вам также потребуется указать свойство конфигурации `ingestPoint` при создании экземпляра трекера. Это свойство задаёт конечную точку (endpoint) приёма данных трекера. Пользователям облачной версии это не нужно, потому что по умолчанию трекер знает, где находится SaaS-версия этой конечной точки, но если вы используете self-hosted версию, вам нужно будет её указать (это должно быть что-то вроде `https://openreplay.mydomain.com/ingest`)

## Работа с переменной ENV в Gatsby

В качестве меры безопасности мы рекомендуем не хардкодить ключ проекта прямо в коде. Это означает, что вам нужно будет экспортировать его как переменную ENV, чтобы Gatsby мог его прочитать.

Тогда ключ проекта OpenReplay будет храниться в переменной с именем `GATSBY_OPENREPLAY_PROJECT_KEY`

Обратите внимание, что имя имеет префикс `GATSBY_`, который сообщает Gatsby, что нужно сделать переменную доступной и во фронтенд-коде. В противном случае вы не сможете прочитать значение, если только не выполняете код на Node.js.
