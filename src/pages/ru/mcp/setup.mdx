На этой странице описывается установка приложения OpenReplay MCP, его настройка на работу с вашим экземпляром и вход в систему.

## 1. Установите сервер MCP

Добавьте сервер OpenReplay MCP в конфигурацию MCP вашего клиента. 

### Claude Desktop

Claude Desktop устанавливает OpenReplay MCP как **расширение** в один клик (пакет `.mcpb`):

1. Скачайте `openreplay.mcpb` со [страницы релизов](https://github.com/openreplay/openreplay/releases) OpenReplay.
2. Дважды щёлкните по файлу — Claude Desktop откроется и предложит установить его в качестве расширения.
3. Нажмите **Install**.

Вот и всё. Не нужно редактировать конфигурацию JSON и устанавливать набор инструментов Node.

## 2. Направьте MCP на свой экземпляр OpenReplay

В Claude Desktop откройте панель настроек расширения и укажите в поле **OpenReplay URL** адрес, который вы используете в браузере — `https://app.openreplay.com` для Cloud или ваш собственный домен для самостоятельного размещения. MCP автоматически определяет хост API.

<img src="/mcp/config-panel.png" alt="Настройка OpenReplay URL в параметрах приложения MCP" style={{ width: '700px', height: 'auto' }} />

Для ChatGPT и Codex URL берётся из переменной окружения `OPENREPLAY_BACKEND_URL`, которую вы задали на шаге 1.

Вы также можете переключать экземпляры в любой момент прямо в чате: *«используй OpenReplay по адресу https://openreplay.mycompany.com»*.

## 3. Войдите через браузер

MCP использует процесс авторизации в браузере — вам никогда не придётся вставлять пароль в чат.

<Aside type="tip" title="Процесс вкратце" >
  1. Попросите Claude **войти в OpenReplay**.
  2. Claude возвращает ссылку авторизации и пытается открыть её в вашем браузере.
  3. Вы входите в OpenReplay (если ещё не вошли) и нажимаете **Authorize**.
  4. Вы сообщаете Claude *«я авторизовал доступ»*.
  5. Claude завершает вход и подтверждает, что токен выдан.
</Aside>

### Шаг за шагом

**1.** В чате напишите:

```
Log me in to OpenReplay
```

**2.** Claude вызывает инструмент `login_browser` и отвечает URL вида `…/mcp/authorize?state=…`. В большинстве систем он также автоматически открывает ссылку в браузере по умолчанию.

**3.** На вкладке OpenReplay подтвердите запрос доступа, нажав **Authorize**.

<img src="/mcp/authorize.png" alt="Экран авторизации OpenReplay" style={{ width: '700px', height: 'auto' }} />

**4.** Вернитесь к Claude и напишите:

```
I authorized it
```

**5.** Claude завершает вход (с помощью инструмента `complete_login`) и подтверждает, что токен сгенерирован. Токен сохраняется локально, поэтому в следующем чате повторять это не нужно.

<Aside type="caution" title="Запасной вариант с паролем" >
  Если ваш хост не может открывать окна браузера, MCP также поддерживает вход по электронной почте и паролю — просто скажите *«Войди под электронной почтой `you@example.com` и паролем `...`»*. Процесс через браузер предпочтителен всегда, когда он доступен.
</Aside>

## Проверка подключения

После входа попросите Claude:

```
List my OpenReplay projects
```

Если всё настроено правильно, вы увидите названия и идентификаторы своих проектов. Оттуда перейдите на страницу [Возможности](/mcp/capabilities), чтобы узнать, что ещё можно запросить.
