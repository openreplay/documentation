Pinia — это библиотека управления состоянием, заменяющая VueX в проектах Vue. Там, где раньше вы использовали VueX, теперь вы будете использовать Pinia. С помощью нашего [плагина Pinia](/plugins/pinia) вы можете отслеживать изменения состояния на протяжении всего жизненного цикла вашего приложения.

## Установка плагина

Плагин легко устанавливается с помощью следующей команды:

```jsx
npm i @openreplay/tracker-vuex
```

## Использование плагина внутри вашего приложения Vue

В этом примере мы будем использовать демонстрационное приложение Vue, чтобы показать, как можно добавить этот плагин и взаимодействовать с ним внутри реального приложения.

**Примечание**: полный исходный код этого примера находится в этом [репозитории GitHub](https://github.com/deleteman/openreplay-vue-pinia-example), загляните туда, если вам нужны дополнительные сведения.

После установки плагина мы изменим файл `main.js`, добавив приведённую ниже инструкцию импорта:

```jsx
import trackerVuex from '@openreplay/tracker-vuex/cjs';
```

А затем мы воспользуемся функцией `startTracker`, чтобы одновременно создать экземпляр трекера и использовать плагин.

Результатом вызова этой функции будет объект, содержащий результаты использования метода `use` с плагином.

**Обратите внимание**, что имя деструктурированного свойства совпадает со значением свойства `name`. Пока вы сохраняете их одинаковыми, всё будет в порядке.

```jsx
const {piniaPlugin} = startTracker({
    projectKey: "<your project key>",
    plugins: [{
        name: 'piniaPlugin',
        fn: trackerVuex
    }]
})
```

Следующий шаг — создать именованную обёртку для хранилища (store wrapper), которую мы позже будем использовать для настройки плагина в нужном хранилище.

```jsx
const piniaStoreWrapper = piniaPlugin("products")
saveStore(piniaStoreWrapper)
```

`piniaStoreWrapper` позже нужно будет использовать с хранилищем, которое мы решим отслеживать. Однако на данный момент хранилище ещё не создано, поэтому мы сохраним его с помощью функции `saveStore`.

### Вспомогательные функции

Прежде чем двигаться дальше, давайте рассмотрим функции `saveStore` и `startTracker`.

Первая является частью модуля `storesManager`, который выглядит так:

```jsx
let storeWrapper = null;

export function saveStore(s) {
    storeWrapper = s;
}

export function getStore(store) {
    return storeWrapper;
}
```

Этот модуль позволит нам использовать `piniaStoreWrapper`, который мы создали ранее, где-нибудь в другом месте.

Что касается функции `startTracker`, её код выглядит так:

```jsx
export function startTracker(config) {
    console.log("Starting tracker...")

    console.log("Project key used: ", config.projectKey)
    const trackerConfig = {
        projectKey: config.projectKey,
         //ingestPoint: config.ingestPoint,
        __DISABLE_SECURE_MODE: true
    }

    const tracker = new Tracker(trackerConfig);

    const pluginReturns = {}
    config?.plugins.forEach( p => {
        console.log("Using plugin...", p.name)
        pluginReturns[p.name] = tracker.use(p.fn())
    })

    tracker.start();
    return {
        tracker,
        ...pluginReturns
    }
}
```

Эта функция создаёт экземпляр трекера с предоставленной нами конфигурацией. Обратите внимание, что я оставил опцию `ingestPoint` закомментированной — это потому, что мы работаем напрямую с версией SaaS. Но если вы планируете использовать этот код с self-hosted-версией, вам нужно будет указать её, и она должна выглядеть примерно так: `<your-instance>.yourhost/ingest`

### Отслеживание ваших хранилищ Pinia

Когда весь код настройки готов, перейдите в то место, где вы используете хранилище; в нашем примере давайте отредактируем файл `App.vue`, чтобы вызвать функцию `getStore`, которая вернёт обёртку хранилища, сохранённую нами ранее.

И мы используем её, чтобы обернуть хранилище, которое хотим отслеживать:

```jsx
<template>
  <!-- template code -->
</template>

<script lang="ts">
import { computed, defineComponent, onMounted, ref } from 'vue'
import { Product, useProductStore } from './store/products'
import { useCartStore } from './store/cart'
import {getStore} from './store/storesManager.js'

export default defineComponent({
  setup() {

    const productStore = useProductStore()
    const loading = ref(true)
    //we get the wrapper here
	  const storeWrapper = getStore()

    onMounted(async () => {
      await productStore.fetchAll()
      loading.value = false
    })

    const cartStore = useCartStore()

		//we wrap the cart store here
    storeWrapper(cartStore)
    return {
      products: computed(() => productStore.list),
      cart: computed(() => cartStore.formattedCart),
      total: computed(() => cartStore.total),
      loading,
      add: (product: Product) => cartStore.add(product),
      remove: (product: Product) => cartStore.remove(product),
    }
  }
})

</script>
```

Чтобы просмотреть изменения состояния во время воспроизведения записи, нажмите на новую вкладку VueX в правом нижнем углу:

![Новая вкладка VueX](/tutorials//pinia/new-tab.png)

Нажав на неё, вы увидите содержимое хранилища и сможете понять, как оно меняется, а также какие данные в нём сохранены:

![Мутации состояния](/tutorials//pinia/state-changes.png)

Обратите внимание на имя хранилища — это то имя, которое мы использовали при первом вызове плагина Pinia.
