Дашборды в OpenReplay служат вашим центральным узлом для мониторинга, анализа и оптимизации производительности продукта и пользовательского опыта. Независимо от того, отлаживаете ли вы проблемы, отслеживаете поведение пользователей или измеряете производительность, дашборды предоставляют сведения, необходимые для принятия решений на основе данных.

Дашборды OpenReplay объединяют важные сведения для каждого участника вашей команды:

- **Для разработчиков:** Отслеживайте технические проблемы, такие как ошибки JavaScript, медленные сетевые запросы и проблемы с производительностью, в реальном времени.
- **Для продакт-менеджеров:** Анализируйте поведение пользователей, отслеживайте воронки конверсии и понимайте пути, по которым пользователи проходят через ваш продукт.
- **Для команд, анализирующих производительность сайта:** Отслеживайте ключевые метрики, такие как наиболее посещаемые страницы, источники трафика и демографические данные пользователей.

## Обзор дашбордов и доступных карточек

Когда вы создаёте дашборд, он начинается как пустое полотно, готовое к настройке. Дашборды состоят из **карточек**, каждая из которых отображает определённую метрику или набор данных. Вы можете добавлять, удалять или перемещать карточки, чтобы организовать дашборд по своему усмотрению.

Карточки делятся на три основные категории:

#### Product analytics

*Product Analytics* помогает отслеживать и анализировать вовлечённость и поведение пользователей в вашем продукте.

| **Карточка**    | **Описание**                                                   |
|-------------------|------------------------------------------------------------------|
| **Trends**        | Отслеживание трендов сессий с течением времени                                |
| **Funnels**       | Визуализация продвижения пользователей через ключевые этапы и выявление отказов для повышения конверсии               |
| **Journeys**      | Понимание путей, по которым пользователи проходят через ваш продукт            |
| **Heatmaps**      | Визуализация горячих зон и паттернов взаимодействия пользователей              |

#### Monitors

*Monitors* помогают выявлять и устранять технические проблемы производительности, влияющие на пользовательский опыт.

| **Карточка**           | **Описание**                                                    |
|--------------------------|-------------------------------------------------------------------|
| **JS Errors**            | Отслеживание ошибок JavaScript, влияющих на пользовательский опыт             |
| **Top Network Requests** | Выявление часто запрашиваемых сетевых ресурсов                    |
| **4xx/5xx Requests**     | Мониторинг клиентских и серверных ошибок для выявления проблем с производительностью           |
| **Slow Network Requests**| Определение задержек, вызванных медленными ответами                          |

#### Web Analytics

*Web analytics* предоставляет сведения о демографии пользователей, источниках трафика и паттернах трафика.


| **Карточка**               | **Описание**                                                   |
|------------------------------|------------------------------------------------------------------|
| **Top Pages**               | Выявление наиболее посещаемых страниц вашего сайта                      |
| **Top Browsers**            | Узнайте, на какие браузеры ваши пользователи полагаются больше всего                        |
| **Top Referrers**           | Понимание того, откуда исходит ваш трафик                          |
| **Top Users**               | Поиск пользователей с наибольшим количеством взаимодействий                             |
| **Top Countries** | Анализ вашей аудитории по местоположению              |
| **Top Devices** | Анализ вашей аудитории по типу устройства             |



## Создание вашего первого дашборда

Шаг 1: Перейдите в раздел Dashboards и нажмите кнопку **Create dashboard**.

Шаг 2: Выберите одну из трёх категорий. Выберите тип карточки, которую хотите добавить, чтобы **начать с нуля**, или нажмите **Add existing Card**, чтобы повторно использовать существующую.

Шаг 3: Нажмите **Create Card**, чтобы добавить её на дашборд

Шаг 4: Чтобы добавить ещё одну карточку, нажмите кнопку «+» рядом с существующей карточкой.

## Распространённые сценарии использования

Дашборды гибкие и могут быть адаптированы для различных целей.

Вот несколько примеров:

- **Анализ поведения пользователей:** Используйте Trends и Journeys, чтобы понять, как пользователи взаимодействуют с вашим продуктом.
- **Выявление проблем:** Отслеживайте JS Errors, 4xx/5xx Requests и Slow Network Requests, чтобы устранять проблемы с производительностью.
- **Понимание трафика:** Отслеживайте Top Pages, Referrers и демографические данные пользователей, чтобы видеть, что работает, и где нужны улучшения.

Создавайте отдельные дашборды для конкретных целей, например для мониторинга маркетинговых кампаний или отслеживания технической производительности. Дашборды могут служить различными представлениями вашей системы и предоставлять целостную сводку происходящего.
