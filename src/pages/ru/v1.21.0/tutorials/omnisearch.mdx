Когда трекер OpenReplay настроен и записывает пользовательские сессии, вы готовы выяснить, что на самом деле происходит в вашем продукте. OmniSearch помогает отлаживать проблемы, выявлять, что идёт не так, и подробно понимать поведение пользователей. Он устраняет разрыв между количественными данными и качественными сведениями, скрытыми в ваших повторах сессий.

## Понимание событий

**События** в OmniSearch представляют взаимодействия пользователя и поведение системы, которые происходят во время сессии.

Они делятся на две основные категории: **автоматически захваченные события** (автоматическое отслеживание взаимодействий пользователя) и **события DevTools** (техническое наблюдение и отладка).

### Автоматически захваченные события
| **Название события**  | **Описание** |
| --- | --- |
| Click | Записывает клики пользователя. |
| Text Input | Отслеживает текст, введённый в поля. |
| Visited URL | Регистрирует посещённые страницы |
| Tagged Element | Отслеживает элементы с пользовательскими тегами. |

### События DevTools

|  **Название события**  | **Описание** |
| --- | --- |
| Network Request | Отслеживает сетевую активность. |
| GraphQL | Регистрирует запросы и ответы GraphQL. |
| State Action | Отслеживает изменения состояния в вашем приложении. |
| Error Message | Захватывает сообщения об ошибках в консоли. |
| Average CPU Load | Измеряет среднюю загрузку CPU. |
| Average Memory Usage | Отслеживает среднее потребление памяти. |

## Понимание фильтров

**Фильтры** в OmniSearch помогают сузить повторы сессий по определённым критериям, позволяя сосредоточиться на наиболее релевантных пользовательских сессиях. Они работают вместе с событиями, создавая точные поисковые запросы по данным ваших сессий.

Фильтры организованы в четыре основные категории:

### Фильтры пользователей

|  **Название фильтра**  | **Описание** |
| --- | --- |
| User ID | Фильтрация конкретных пользователей с использованием их уникальных идентификаторов. |
| Anonymous User ID | Фильтрация сессий пользователей без авторизованной личности. |

### Фильтры сессий

|  **Название фильтра**  | **Описание** |
| --- | --- |
| Country | Фильтрация сессий по стране. |
| City | Фильтрация сессий по городу. |
| State/Province | Фильтрация сессий по штату или провинции. |
| Operating System (OS) | Фильтрация сессий по используемой операционной системе. |
| Browser | Фильтрация сессий по типу браузера. |
| Device | Фильтрация сессий по устройству пользователя. |
| Platform | Фильтрация сессий по типу платформы. |
| Version ID | Фильтрация сессий по версии приложения. |
| Referrer | Фильтрация сессий на основе ссылающегося URL. |
| Duration | Фильтрация повторов сессий по их длительности. |
| UTM Source | Фильтрация сессий по параметру UTM source. |
| UTM Medium | Фильтрация сессий по параметру UTM medium. |
| UTM Campaign | Фильтрация сессий по параметру UTM campaign. |

### Фильтры проблем

|  **Название фильтра**  | **Описание** |
| --- | --- |
| Rage Click, Dead Click, Excessive Scrolling, Mouse Thrashing | Выявляют сессии с разочарованием пользователя, вызванным повторяющимися кликами, неработающими кнопками или ссылками, чрезмерной прокруткой, беспорядочными движениями мыши…. |
| Bad Request | Выявляет сессии с ошибками HTTP 400. |
| Missing Resource | Находит сессии с отсутствующими ресурсами |
| CPU Issues | Выявляет сессии с высокой загрузкой CPU. |
| Slow Resource | Выявляет сессии с медленно загружающимися ресурсами. |
| Slow Page Load | Выявляет сессии с медленным временем загрузки страницы. |
| Crash | Выявляет сессии, в которых произошёл сбой приложения или страницы. |
| Errors | Выявляет сессии с ошибками приложения. |

### Фильтры метаданных

|  **Название фильтра**  | **Описание** |
| --- | --- |
| Metadata key | Используйте определённые пользователем метаданные для фильтрации сессий. Если они не определены, узнайте, как добавить метаданные [здесь](/tutorials/metadata) |
