По умолчанию наш трекер отправляет на платформу много полезной информации, но вся эта информация стандартна. Если какая-либо точка данных является частью Web API, скорее всего, мы её отслеживаем.

Но что произойдёт, если вы также захотите обогатить свои записи сессий данными, специфичными для вашего приложения?

Например, понять, на каком платном тарифе находится ваш пользователь, когда вы просматриваете его повтор? Эта информация может предоставить дополнительный контекст вашим разработчикам, так что если она не стандартна, это не значит, что мы должны её игнорировать, не так ли?

Вот здесь и вступают в игру метаданные.

**Обратите внимание**, что на нашем канале YouTube есть [видеоверсия этого руководства](https://www.youtube.com/watch?v=ITcJCeKWQGU). Поэтому не стесняйтесь ознакомиться с ней, если вы лучше воспринимаете информацию визуально.

## Что такое метаданные?

В контексте повтора сессии метаданные — это вся информация, которую ваш пользователь не генерирует, но которая каким-то образом относится к этому пользователю.

Иными словами, если пользователь нажимает на определённую ссылку или выполняет конкретное действие, это не метаданные, это пользовательское событие (ознакомьтесь с [документацией о Пользовательских событиях](/installation/custom-events), если хотите узнать о них больше).

Но если есть флаг, который сообщает вам, что пользователь находится на бесплатном тарифе или на платном. Или, возможно, код отслеживания, показывающий вам, откуда пришёл этот пользователь? Всё это метаданные, и вы можете легко добавить их к своим сессиям.

## Как добавить метаданные к сессии?

Первое, что нужно сделать, — настроить поля метаданных непосредственно на платформе.

Если вы пропустите этот шаг, вы не сможете записать никакую информацию, так что не пропускайте его!

Для этого перейдите на страницу настроек вашего проекта, нажав на шестерёнку в правом верхнем углу верхнего меню:

![Иконка настроек проекта](/tutorials//metadata/metadata1.png)

Оказавшись там, нажмите на пункт меню Metadata в крайней левой части экрана:

![Опция Metadata в левом меню](/tutorials//metadata/metadata2.png)

Это перенесёт вас на экран настройки метаданных. Убедитесь, что вы выбрали правильный проект из выпадающего списка, и создайте столько полей, сколько вам нужно.

![Добавление новых метаданных](/tutorials//metadata/metadata3.png)

Для этого примера мы создали 3 поля для проекта «e-commerce test»:

- «plan», которое будет содержать текущий тариф пользователя.
- «utm_source», содержащее код, идентифицирующий, откуда пришёл пользователь.
- И, наконец, «items_in_cart» содержит общее количество товаров в корзине пользователя.

Последнее — это данные, которые зависят от действия пользователя, что отличается от того, что мы рекомендуем. Однако в этом примере вы увидите ограничения полей метаданных и почему именно эта информация должна была быть пользовательским событием.

Когда эта настройка готова, следующий шаг — добавить необходимый код, чтобы фактически отправлять данные в составе вашей сессии.

### Добавление кода для отправки метаданных

Чтобы добавить метаданные к сессии, мы будем использовать метод `setMetadata` из трекера. Вы можете использовать этот метод в любой момент в течение сессии пользователя, просто имейте в виду следующие два соображения:

1. Если вы примените несколько значений к одному и тому же полю метаданных, в составе сессии сохранится только последнее.
2. Все значения должны быть строками. В противном случае они не будут сохранены. Трекер не приводит значения к строкам, так что будьте внимательны с этим.

Для этого примера мы используем сайт электронной коммерции, созданный с помощью Next.js, который мы использовали для руководства по NextJs, обязательно [ознакомьтесь с ним, чтобы](https://docs.openreplay.com/tutorials/next) понять базовую архитектуру провайдера контекста, показанного на следующем скриншоте:

![Добавленный код](/tutorials//metadata/metadata4.png)

Красные стрелки показывают, где были внесены соответствующие изменения. Этот провайдер контекста экспортирует функцию под названием `setMetadata`, которая, в свою очередь, вызывает метод `setMetadata` из трекера.

Если вам нужен рабочий пример, вы можете ознакомиться с [этим репозиторием](https://github.com/deleteman/nextjs-commerce-example/tree/redux_metadata/site).

Теперь всё, что нам нужно сделать, — это использовать экспортированную функцию всякий раз, когда мы хотим настроить поля метаданных.

### Установка метаданных из нашего кода

Для целей этого примера мы решим устанавливать поля `utm_source` и `plan` непосредственно с главной страницы, а затем, всякий раз, когда пользователь добавляет товар в корзину, мы будем обновлять последнее оставшееся поле.

В файл `pages/index.tsx` мы добавим следующий код:

```jsx
// inside the main component
const { startTracking, setMetadata } = useContext(TrackerContext)
//...
useEffect(() => {
    async function getProds() {
      await startTracking()
      setMetadata('plan', getPlan()) //addition
      setMetadata('utm_source', getUTMSource()) //addition
      dispatch(getMakeUpProducts() as any)
    }

    getProds()
  }, [dispatch])
```

Большая часть этого хука уже была там, но мы добавили ссылку на функцию `setMetadata`, а затем два вызова к ней. Обе функции, `getPlan` и `getUTMSource`, возвращают случайные данные для этого примера. Для вашего конкретного случая использования вам придётся реализовать их соответствующим образом.

Этот код установит эти значения метаданных при загрузке главной страницы.

Теперь нам нужно добавить последнее значение — количество товаров в корзине. Для этого мы перейдём к компоненту `ProductSidebar`, который отвечает за управление тем, что происходит при нажатии на «Add to cart», и добавим ещё один хук, чтобы реагировать на изменение количества товаров в корзине.

```jsx
useEffect(() => {
    setMetadata('items_in_cart', productsInCart)
  }, [productsInCart])
```

Разумеется, вам также нужно получить функцию `setMetadata` от провайдера контекста, как раньше, но как только вы это сделаете, всё готово.

Теперь данные сохраняются. Мы также можем увидеть это на платформе в списке сессий.

## Использование метаданных

Теперь, когда метаданные сохраняются во всех повторах сессий, что вы можете с ними делать?

Что ж, для начала вы можете визуализировать их непосредственно из повтора:

![Просмотр метаданных в плеере](/tutorials//metadata/metadata5.png)

Вы должны начать видеть поля метаданных в правом верхнем углу экрана вашего плеера. В этом примере мы видим, что у пользователя тариф Enterprise Edition, он пришёл из Reddit и у него добавлен в корзину только один товар.

### Поиск по полям метаданных

Другая мощная вещь, которую вы можете делать с метаданными, — использовать их как параметр поиска. С помощью панели omnisearch вы можете выбрать созданные вами пользовательские поля метаданных в качестве параметров поиска:

![Поиск с использованием ваших полей метаданных](/tutorials//metadata/metadata6.png)

Так что если вы хотите просмотреть только повторы Enterprise Edition, вы можете просто выбрать поле Plan и отфильтровать его по «ee», вот так:

![Поиск по тарифу пользователя](/tutorials//metadata/metadata7.png)

Вы также можете использовать ту же панель omnisearch при создании виджета для вашего дашборда. Итак, продолжая с примером EE, если вы хотите создать дашборд только для клиентов Enterprise, вы можете фильтровать каждый пользовательский виджет с помощью атрибута «Plan».

Например, следующий виджет показывает, сколько клиентов Enterprise нажимают на кнопку «Add to cart»:

![Создание временного ряда с использованием метаданных](/tutorials//metadata/metadata8.png)

В результате получается виджет, который выглядит так:

![Результаты](/tutorials//metadata/metadata9.png)

С помощью полей метаданных вы можете настраивать и расширять аналитику, которую вы получаете из своих повторов сессий, настолько, насколько захотите.

---

Метаданные могут значительно расширить аналитику, которую вы можете получить из своих повторов сессий, а их настройка относительно проста.

Помните, что какую бы информацию вы ни захотели сохранить, она всегда должна быть в виде строк, иначе она будет проигнорирована.
