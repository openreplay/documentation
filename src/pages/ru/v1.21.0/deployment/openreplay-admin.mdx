## CLI

CLI полезен для управления основными аспектами вашего экземпляра OpenReplay, такими как перезапуск или переустановка сервиса, доступ к журналам компонента или просто проверка состояния ваших backend-сервисов.

Запустите CLI с опцией `-h`:

```bash
openreplay -h
```

Backend OpenReplay опирается на следующие компоненты/сервисы:

| Сервис | Описание |
|---------|-------------|
| http | Принимает события и записи сессий |
| sink | Читает данные из конвейера потоковой передачи (Redis или Kafka для версии enterprise) и вставляет их во временное хранилище (NFS) |
| storage | Перемещает временные файлы записей сессий в сервис объектного хранилища |
| assets | Для кэширования ресурсов (CSS, шрифтов и иконок) для корректного отображения записей |
| db | Читает и записывает данные в различные базы данных (Postgres + ClickHouse для версии enterprise) |
| ender | Завершает пользовательские сессии при бездействии или отключении |
| chalice | API для обслуживания frontend |
| alerts | Отправляет уведомления (электронная почта, slack, в приложении, webhook) при достижении установленного пользователем порога по любой из метрик производительности |
| integrations |  Отправляет и получает данные из поддерживаемых сторонних API (Sentry, Elastic, GitHub, Jira и т. д.) |

## Полезные команды

- Проверить состояние работоспособности OpenReplay

```sh
openreplay -s
```

- Проверить журналы сервисов

```sh
openreplay -l <service name>
```

## Увеличение мощности сервиса

Можно увеличить мощность любого сервиса, переопределив значения выделения cpu/памяти по умолчанию. Последние определяются в процессе настройки на основе мощности вашего экземпляра и должны подходить для нужд большинства установок.

 Если у вас большой объём и мощная машина, просто запустите файл `openreplay -e` и переопределите ресурсы сервиса. В приведённом ниже примере мы делаем это для воркера `http`. Но это можно сделать для любого сервиса (т. е. sink, storage, postgresql, redis и т. д.), раскомментировав и обновив блок ниже. Если вам нужно сделать это для более чем одного сервиса, просто скопируйте и переименуйте/обновите тот же блок (следите за дублированием).

```yaml
http:
  resources:
    limits:
      cpu: 4096m
      memory: 8192Mi
    requests:
      cpu: 1024m
      memory: 2056Mi
```

Наконец, переустановите сервис, чтобы новые лимиты вступили в силу (ваши данные не будут потеряны):

```bash
openreplay -R
```

## Обновление PostgreSQL

Ниже приведены инструкции по обновлению PostgreSQL до версии 1.16 или более поздней:

1. Сделайте дамп БД в постоянном томе:

   Если вы используете стандартную (vanilla) установку, выполните следующую команду, чтобы сделать дамп базы данных:

   > Примечание: можно ожидать, что данные будут сжаты в соотношении 3:1 (например: если у вас 15G данных postgres, в дампе они займут 5G). Поэтому убедитесь, что у вас достаточно места на диске.

   ```sh
   kubectl -n db exec -it postgresql-0 -- /bin/bash -c 'PGPASSWORD=$POSTGRES_PASSWORD PGUSER=postgres pg_dumpall -v > /bitnami/postgresql/dumpall.sql'
   ```

2. Удалите существующий каталог данных:

   ```sh
   kubectl -n db exec -it postgresql-0 -- /bin/bash -c 'rm -rf /bitnami/postgresql/data'
   ```

3. Масштабируйте StatefulSet PostgreSQL до нуля, чтобы остановить текущие поды:

   ```sh
   kubectl scale sts postgresql --replicas=0 -n db
   ```

4. Обновите образ PostgreSQL до версии 16.3.0 или более поздней:

   ```sh
   kubectl set image statefulset/postgresql postgresql=bitnamilegacy/postgresql:17.2.0 -n db
   ```

5. Масштабируйте StatefulSet обратно до единицы, чтобы запустить обновлённый под:

   ```sh
   kubectl scale sts postgresql --replicas=1 -n db
   ```

6. Убедитесь, что под запущен и операция масштабирования завершена:

   ```sh
   kg po -n db -l app.kubernetes.io/name=postgresql
   ```

7. Восстановите базу данных из дампа:

   ```sh
   kubectl -n db exec -it postgresql-0 -- /bin/bash -c 'PGPASSWORD=$POSTGRES_PASSWORD PGUSER=postgres psql -f /bitnami/postgresql/dumpall.sql'
   ```

8. После завершения обновления вы можете удалить файл дампа из пода PostgreSQL:

   ```sh
   kubectl -n db exec -it postgresql-0 -- /bin/bash -c 'rm /bitnami/postgresql/dumpall.sql'
   ```

## Удаление OpenReplay

Выполните приведённые ниже команды, чтобы удалить OpenReplay:

```bash
sudo k3s-uninstall.sh
```

Вся конфигурация хранится в `/var/lib/openreplay/`.

```
sudo rm -rf /var/lib/openreplay/openreplay
```
