Установка трекера OpenReplay в проект на базе Remix довольно проста.

Когда ваш проект Remix готов, найдите файл `root.jsx` в папке `app` и добавьте код SSR, предоставленный OpenReplay при создании проекта на платформе.

## Шаг 1. Получите код

Вы вполне можете использовать код, предлагаемый платформой, — он представляет собой необходимый минимум для начала работы. Если вы так делаете, убедитесь, что выбрали вариант «Server-Side-Rendered (SSR)», как показано ниже.

![snippet-code-screenshot.jpg](/using-or/snippet-code-screenshot.jpg)

Но для этого примера мы создадим демонстрационный модуль, который сохраним как `tracker.ts` в папке `app`. Этот модуль будет экспортировать функцию `startTracker`, которая создаст экземпляр трекера и запустит его.

```jsx
import {v4 as uuidV4} from 'uuid'
import Tracker from '@openreplay/tracker'

function defaultGetUserId() {
    return uuidV4()
}

type TrackerConfig = {
    projectKey: string,
    userIdEnabled?: boolean,
    getUserId?: () => string
}

export function startTracker(config: TrackerConfig) {
    let tracker = null;
    let userId = "";

    console.log("Starting tracker...")
    console.log("Custom configuration received: ", config)

    const getUserId = (config?.userIdEnabled && config?.getUserId) ? config.getUserId : defaultGetUserId

    const trackerConfig: TrackerConfig = {
        projectKey: config?.projectKey
    }

    console.log("Tracker configuration")
    console.log(trackerConfig)
    tracker = new Tracker(trackerConfig);

    if(config?.userIdEnabled) {
        userId = getUserId()
        tracker.setUserID(userId)
    }

    tracker.start();

    return {
        tracker,
        userId
    }
}
```

Если у вас возникают проблемы при импорте модуля Tracker из OpenReplay таким способом, рассмотрите возможность использования динамического импорта, например так:

```jsx
import {v4 as uuidV4} from 'uuid'

function defaultGetUserId() {
    return uuidV4()
}

type TrackerConfig = {
    projectKey: string,
    userIdEnabled?: boolean,
    getUserId?: () => string
}

export function startTracker(config: TrackerConfig) {
    let tracker = null;
    let Tracker = null
    let userId = "";

    (async () => {
				//dynamic import
        Tracker  = (await import('@openreplay/tracker')).default
        console.log("Starting tracker...")
        console.log("Custom configuration received: ", config)

        const getUserId = (config?.userIdEnabled && config?.getUserId) ? config.getUserId : defaultGetUserId

        const trackerConfig: TrackerConfig = {
            projectKey: config?.projectKey
        }

        console.log("Tracker configuration")
        console.log(trackerConfig)
        tracker = new Tracker(trackerConfig);

        if(config?.userIdEnabled) {
            userId = getUserId()
            tracker.setUserID(userId)
        }


        tracker.start();

    })()

    return {
        tracker,
        userId
    }
}
```

💡**Примечание для пользователей self-hosted:** если вы используете self-hosted-версию OpenReplay, вам также потребуется указать свойство конфигурации `ingestPoint`. Это свойство задаёт конечную точку приёма данных трекера. Облачным пользователям это не нужно, потому что по умолчанию
трекер знает, где находится SaaS-версия этой конечной точки, но если
вы размещаете его самостоятельно, вам нужно будет указать её (это должно выглядеть примерно так: `https://openreplay.mydomain.com/ingest`)

## Шаг 2.  Добавьте код и трекер в свой проект

После этого вам нужно установить пакет трекера для вашего проекта с помощью:

```markdown
$ npm install @openreplay/tracker
```

Вам также потребуется установить [пакет uuid](https://www.npmjs.com/package/uuid), чтобы приведённый выше код работал как есть.

Когда это сделано, мы почти готовы начать отслеживание, но нам всё ещё нужно настроить **project key** как переменную окружения (ENV) и каким-то образом передать её трекеру через объект `config`, получаемый в функции `startTracker`.

### Загрузка переменных окружения (ENV) в Remix

Переменные окружения (ENV) доступны на back-end, поэтому мы воспользуемся функцией `loader`, которую можно использовать для загрузки любых внешних данных, включая переменные окружения (ENV).

А затем мы используем хук `useLoaderData`, чтобы после гидратации наш компонент имел доступ к загруженным данным.

Внутри нашего файла root.tsx (или jsx, если вы не используете TypeScript) добавьте эту функцию:

```tsx
type LoaderData = {
  ENV: {
    projectKey: string | undefined
  }
};

export const loader: LoaderFunction = async ({ }) => {
  return json<LoaderData>({
    ENV: {
      projectKey: process.env.OPENREPLAY_PROJECT_KEY
    }
  });
};
```

Обратите внимание, как мы задаём наш project key внутри переменной окружения `OPENREPLAY_PROJECT_KEY`.

Возвращая её в рамках выполнения функции `loader`, мы затем можем использовать хук `useLoaderData` внутри основного компонента, например так:

```tsx
export default function App() {

  let loaderData = useLoaderData()

  useEffect(() => {
    startTracker({
      projectKey:  loaderData.ENV.projectKey
    })
  }, [])

  return (
    <html lang="en">
      <head>
        <Meta />
        <Links />
      </head>
      <body>
        <Outlet />
        <ScrollRestoration />
        <Scripts />
        <LiveReload />
      </body>
    </html>
  );
}
```

Содержимое вашего компонента может отличаться в зависимости от того, чего вы пытаетесь достичь, но этот пример обеспечивает запуск трекера с правильным project key после гидратации страницы.

## Шаг 3. Наслаждайтесь процессом отладки!

После этого вы можете приступить к развёртыванию вашего приложения.

Как только вы начнёте его тестировать, вы увидите записи воспроизведения (replays) в списке через несколько минут после закрытия вкладки (это сделано для того, чтобы убедиться, что вкладка не была закрыта случайно).

💡 **Помните**: трекер не будет работать локально, и вы не увидите свои локальные тесты на платформе. Вам нужно будет развернуть приложение, чтобы трекер получил доступ к back-end OpenReplay.
