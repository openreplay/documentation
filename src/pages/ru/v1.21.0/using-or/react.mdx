Запуск трекера в приложении React довольно прост. 

## Добавление кода отслеживания

Хотя вы можете [получить версию кода для копирования и вставки непосредственно с платформы при создании нового проекта](/using-or/snippet#getting-the-snippet), этот код слишком упрощён и значительно усложнит его настройку и добавление дополнительных плагинов.

Если вы создаёте SPA (одностраничное приложение), всё, что вам нужно сделать, — это добавить следующий код в вашу точку входа:

```jsx
import Tracker from '@openreplay/tracker';

const tracker = new Tracker({
  projectKey: "<your project key>",  
});

function YourMainComponent({props}) {
    //some code here...

    useEffect(() => {
        tracker.start();
    }, [])

    return //....
}
```

Другими словами, вы можете создать экземпляр трекера в начале скрипта, и как только страница загрузится, вы можете запустить его с помощью хука `useEffect`.

<Aside type="tip">
**Примечание для пользователей собственного хостинга**: если вы используете самостоятельно размещённую версию OpenReplay, вам также необходимо указать свойство конфигурации `ingestPoint`. 
Это свойство определяет конечную точку приёма данных трекера. Пользователям облачной версии это не нужно, 
потому что по умолчанию трекер уже знает, где находится SaaS-версия этой конечной точки, но если вы размещаете её самостоятельно, вам нужно будет указать её 
(это должно быть что-то вроде `https://openreplay.mydomain.com/ingest`) 
</Aside>

### Обработка "projectKey" в вашем коде

Обратите внимание, что `projectKey`, который вы настраиваете, предоставляется платформой OpenReplay. По соображениям безопасности это значение не должно храниться в коде в явном виде, а лучше — в каком-либо конфигурационном файле/системе.

Существуют разные варианты решения этой проблемы; в зависимости от вашей настройки вы можете использовать что-то вроде [поддержки переменных ENV в `create-react-app`](https://create-react-app.dev/docs/adding-custom-environment-variables/).

Если вы создаёте сложное приложение на React, скорее всего, вы используете [Next](/using-or/next), [Remix](/using-or/remix) или что-то подобное. Обязательно ознакомьтесь с нашим [разделом настройки](/using-or), чтобы найти фреймворк, с которым вы работаете.
