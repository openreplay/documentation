Этот метод позволяет вам сообщать о «функциональном событии». С помощью функциональных событий вы можете отмечать моменты в сессии пользователя, которые важны с функциональной точки зрения и которые по умолчанию наш трекер не может уловить и пометить самостоятельно. Примером функционального события может быть добавление пользователем товара в корзину. Хотя наш трекер сможет записать это действие, он не будет знать, что нужно пометить его как значимое событие в хронологии вашей сессии. Однако благодаря этому методу вы можете сделать это вручную.

Подробнее см. в разделе [пользовательские события](/installation/custom-events#functionalevents).

## Сигнатура

```swift
ORTracker.shared.event(name: String, payload: Encodable?)
```

## Параметры

- `key: string`: Имя ключа, под которым будет сохранена информация. Этот ключ можно использовать для фильтрации и поиска внутри платформы OpenReplay.
- `payload: Encodable`: Содержимое события. Это могут быть любые кодируемые (JSON) данные.
