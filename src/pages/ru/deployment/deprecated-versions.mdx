Если вы хотите обновить ваш экземпляр OpenReplay до последней версии, убедитесь, что вы используете версию, которая выше или равна `v1.4.0`. Если нет, значит, вы находитесь на устаревшей версии, и вам следует сначала обновиться до `v1.4.0`, прежде чем переходить на любую недавнюю версию (v1.8.0+).

## Обновление бэкенда (с v1.3.6 или более ранней) до v1.4.0

Прежде всего, убедитесь, что вы находитесь на версии `v1.3.6` или более ранней. Если это уже так, то проигнорируйте приведённые ниже команды и переходите к шагу 1). Если нет, обновите вашу установку OpenReplay до `v1.3.6`:
   
```bash 
# Copy/backup the current openreplay folder to _version-number
mv openreplay openreplay_v1.3.5
git clone https://github.com/openreplay/openreplay -b v1.3.6
cd openreplay/scripts/helm
# bash upgrade.sh <old openreplay path>
bash upgrade.sh ~/openreplay_v1.3.5
```

Как только вы окажетесь на `v1.3.6`, переходите к приведённым ниже шагам:

1. Скопируйте/создайте резервную копию текущей папки openreplay в `_version-number`, как показано ниже:
   
  ```bash 
  mv openreplay openreplay_v1.3.6
  ```

2. Клонируйте новую версию OpenReplay. В этом примере мы обновляемся до v1.4.0:
   
  ```bash 
  git clone https://github.com/openreplay/openreplay -b v1.4.0
  ```

3. Запустите приведённый ниже скрипт, чтобы автоматически обновить ваш `vars.yaml` до нового формата:
  
  ```bash
  cd openreplay/scripts/helmcharts
  ansible localhost -m template -a "src=vars_template.yaml dest=vars.yaml" -e @~/openreplay_v1.3.6/scripts/helm/vars.yaml
  ```

4. Обновите OpenReplay:

  ```bash
  cd openreplay/scripts/helmcharts
  helm ls -n app | tail -n +2 | awk '{print $1}' | xargs -I{} helm uninstall {} -n app
  helm uninstall nginx-ingress -n nginx-ingress
  helm upgrade --install openreplay ./openreplay -n app -f vars.yaml --atomic --set forceMigration=true
  ```
  
5. Если вы не используете балансировщик нагрузки и сгенерировали ваш SSL-сертификат с помощью скрипта `certmanager.sh`:

  ```
  cd /var/lib/openreplay/openreplay/scripts/helmcharts && bash certmanager.sh
  ```

6. Обновите переменную `fromVersion` в `/openreplay/scripts/helmcharts/vars.yaml`, чтобы она отражала новую версию. Например, если вы переходите с `v1.3.6` на `v1.4.0`, то обновите `fromVersion`, как показано ниже:
  
  ```yaml
  fromVersion: "v1.4.0"
  ```

> **Примечание:** 
Ручные переопределения, внесённые в любой файл конфигурации сервиса (то есть `openreplay/scripts/helm/app/<app>.yaml`), будут сброшены. Поэтому, если у вас есть какие-либо пользовательские переопределения, например использование [внешнего сервиса объектного хранилища](/configuration/recordings-storage) для ваших записей или увеличенная мощность сервиса (cpu/память), обязательно примените их к новой версии (в `/openreplay/scripts/helmcharts/vars.yaml`) перед запуском скрипта обновления (шаг 4).
