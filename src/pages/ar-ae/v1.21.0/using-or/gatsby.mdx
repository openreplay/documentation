إنّ تثبيت متتبع OpenReplay في مشروع قائم على Gatsby أمر بسيط نسبيًا.

![Sesion replay of a Gatsby site](images/gatsby-ss.png)

بالنظر إلى أنّ Gatsby يستخدم React في الأساس، فإنّ كل ما علينا فعله هو إضافة كود المتتبع داخل الصفحة الرئيسية واستدعاء الدالة `start` باستخدام الـ hook `useEffect`. 

لنلقِ نظرة.

## لماذا نُعدّ المتتبع على موقع ثابت؟

غياب المحتوى الديناميكي داخل تطبيقك لا يعني أنك لا ترغب في الاستفادة من بعض الميزات الأخرى في OpenReplay.

ففي حين أنك لن تستخدم إعادة تشغيل الجلسات للعثور على الأخطاء، ستتمكن من التقاط مقاييس مفيدة وإنشاء [لوحات معلومات مخصصة خاصة بك](https://docs.openreplay.com/tutorials/custom-dashboard) حول هذه المقاييس.

يمكنك استخدام مقاييس الأداء، و web vitals، وحتى المقاييس المتعلقة بالموارد للحصول على فكرة عامة عن التجربة التي يمر بها مستخدموك على موقعك.

## إعداد المتتبع داخل Gatsby

بافتراض وجود موقع Gatsby ثابت تكون نقطة الدخول إليه هي الملف `index.tsx` داخل مجلد `pages`، فستحتاج إلى إضافة الكود التالي إليه:

```jsx
import Tracker from '@openreplay/tracker';
const tracker = new Tracker({
  projectKey: process.env.GATSBY_OPENREPLAY_PROJECT_KEY
});

const Index = ({data, location }) => {

  React.useEffect(() => {
    tracker.start();
  }, [])

   //the rest of your code here...
}
```

يتم إنشاء نسخة من المتتبع عند تجميع الصفحة (سواء كان ذلك عبر SSR أو العرض من جهة العميل)، لكن الدالة start لا يمكن استدعاؤها إلا من المتصفح، لذا نحتاج إلى التأكد من استدعائها بعد تركيب المكوّن (ومن هنا يأتي استخدام الـ hook `useEffect` هنا). إنّ المصفوفة الفارغة كوسيط ثانٍ للـ hook `useEffect` تضمن تنفيذ دالة الاستدعاء (callback) مرة واحدة فقط.

💡**ملاحظة لمستخدمي النسخة المستضافة ذاتيًا (self-hosted):** إذا كنت تستخدم النسخة المستضافة ذاتيًا من OpenReplay، فسيتعين عليك أيضًا تحديد خاصية الإعداد `ingestPoint` عند إنشاء نسخة المتتبع. تحدد هذه الخاصية نقطة استقبال بيانات المتتبع. لا يحتاج مستخدمو Cloud إلى ذلك، لأنّ المتتبع سيعرف افتراضيًا أين توجد نسخة SaaS من هذه النقطة، أمّا إذا كنت تستضيفها ذاتيًا، فستحتاج إلى تحديدها (يجب أن تكون شيئًا مثل `https://openreplay.mydomain.com/ingest`)

## التعامل مع متغير ENV في Gatsby

كإجراء أمني، نوصي بعدم كتابة الـ project key مباشرةً داخل الكود (hardcoding). وهذا يعني أنك ستحتاج إلى تصديره كمتغير ENV حتى يتمكن Gatsby من قراءته.

عندئذٍ سيكون الـ project key الخاص بـ OpenReplay موجودًا داخل متغير يُسمى `GATSBY_OPENREPLAY_PROJECT_KEY`

لاحظ كيف يبدأ الاسم بالبادئة `GATSBY_`، وهذا ما سيُخبر Gatsby بأن يضمن توفّر المتغير في كود الواجهة الأمامية أيضًا. وإلّا فلن تتمكن من قراءة القيمة ما لم تكن تنفّذ كود Node.js.
