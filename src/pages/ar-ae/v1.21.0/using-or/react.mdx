جعل الـ tracker يعمل على تطبيق React أمر بسيط للغاية. 

## إضافة كود التتبع

بينما يمكنك [الحصول على نسخة جاهزة للنسخ واللصق من الكود مباشرةً من المنصة عند إنشاء مشروع جديد](/using-or/snippet#getting-the-snippet)، فإن هذا الكود أساسي للغاية وسيجعل تخصيصه وإضافة الإضافات (plugins) الإضافية أكثر صعوبة بكثير.

بافتراض أنك تقوم ببناء تطبيق SPA (تطبيق صفحة واحدة)، فكل ما عليك فعله هو إضافة الكود التالي إلى نقطة الدخول الخاصة بك:

```jsx
import Tracker from '@openreplay/tracker';

const tracker = new Tracker({
  projectKey: "<your project key>",  
});

function YourMainComponent({props}) {
    //some code here...

    useEffect(() => {
        tracker.start();
    }, [])

    return //....
}
```

بمعنى آخر، يمكنك تهيئة الـ tracker في بداية السكربت، وبمجرد تحميل الصفحة، يمكنك تشغيله باستخدام خطاف `useEffect`.

<Aside type="tip">
**ملاحظة للمستخدمين ذاتيي الاستضافة**: إذا كنت تستخدم النسخة ذاتية الاستضافة من OpenReplay، فستحتاج أيضًا إلى تحديد خاصية التكوين `ingestPoint`. 
تحدد هذه الخاصية نقطة الاستقبال (endpoint) لبيانات الـ tracker. لا يحتاجها مستخدمو السحابة، 
لأن الـ tracker بشكل افتراضي سيعرف أين تقع نسخة SaaS من هذه النقطة، ولكن إذا كنت تستضيفها بنفسك، فستحتاج إلى تحديدها 
(ينبغي أن تكون شيئًا مثل `https://openreplay.mydomain.com/ingest`) 
</Aside>

### التعامل مع "projectKey" في كودك

كملاحظة، إن الـ `projectKey` الذي تقوم بتكوينه هو المفتاح المقدَّم من منصة OpenReplay. ولأسباب أمنية، يجب عدم تخزين هذه القيمة بشكل ثابت (hardcoded) في الكود، بل في نوع ما من ملفات/أنظمة التكوين.

هناك خيارات مختلفة لحل هذه المشكلة؛ واعتمادًا على إعدادك، قد تستخدم شيئًا مثل [دعم المتغيرات ENV من `create-react-app`](https://create-react-app.dev/docs/adding-custom-environment-variables/).

## هل تبني شيئًا أكثر تعقيدًا؟
إذا كنت تبني تطبيقًا معقدًا باستخدام React، فمن المحتمل أنك تستخدم [Next](/using-or/next) أو [Remix](/using-or/remix) أو شيئًا مشابهًا. تأكد من الاطلاع على [قسم الإعداد](/using-or) لدينا للعثور على الإطار الذي تعمل معه.

## هل لديك أسئلة؟

إذا واجهت أي مشاكل في إعداد الـ tracker على مشروع React الخاص بك، يرجى التواصل معنا في [مجتمع Slack](https://slack.openreplay.com/) لدينا واسأل مطورينا مباشرةً!
