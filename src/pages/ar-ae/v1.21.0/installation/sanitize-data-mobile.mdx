تشرح هذه الصفحة كيفية تعقيم مكوّنات واجهة المستخدم في تطبيقات الأجهزة المحمولة الخاصة بك باستخدام حزم تطوير برمجيات OpenReplay لنظامي iOS و Android. للحصول على تفاصيل حول تعقيم طلبات الشبكة، راجع [خيارات الشبكة (الأجهزة المحمولة)](/network-options-mobile).

## iOS - تعقيم مكوّنات واجهة المستخدم

تظهر طرق العرض المعقَّمة بشكل ضبابي في إعادة التشغيل الناتجة. تُزال المكوّنات تلقائيًا (للحفاظ على الذاكرة) بمجرد أن تصبح غير مرئية.

### SwiftUI

أضف المُعدِّل `.sensitive()` إلى المكوّن الذي ترغب في تعقيمه.

```swift
import ORTracker

Text("Sensitive Information")
    .sensitive()
```

### UIKit

استخدم الطريقة `addIgnoredView` لتمييز طريقة عرض على أنها حساسة.

```swift
import ORTracker

ORTracker.shared.addIgnoredView(view)
```

### React Native

قم بتغليف المكوّنات الحساسة بالمكوّن `ORSanitizedView` لجعلها ضبابية في إعادة التشغيل.

```js
import OR from '@openreplay/react-native';

<OR.ORSanitizedView>
  <SensitiveComponent />
</OR.ORSanitizedView>
```

## Android - تعقيم مكوّنات واجهة المستخدم

تظهر طرق العرض المعقَّمة بشكل ضبابي في إعادة التشغيل الناتجة وتُزال تلقائيًا من الذاكرة عندما تكون خارج الشاشة.

### View

ميّز طريقة عرض على أنها حساسة باستخدام الطريقة `addIgnoredView`.

```kotlin
import com.openreplay.tracker.OpenReplay

OpenReplay.addIgnoredView(view)
```
