بعد إعداد متعقّب OpenReplay وبدء التقاط جلسات المستخدمين، تكون جاهزًا لاكتشاف ما يحدث فعليًا في منتجك. يساعدك OmniSearch على تصحيح الأخطاء، وتحديد ما يسير على نحو خاطئ، وفهم سلوك المستخدمين بالتفصيل. فهو يسد الفجوة بين البيانات الكمية والرؤى النوعية المخفية داخل عمليات إعادة تشغيل الجلسات لديك.

## فهم الأحداث

تمثّل **الأحداث** في OmniSearch تفاعلات المستخدم وسلوكيات النظام التي تحدث أثناء الجلسة.

وهي مقسّمة إلى فئتين رئيسيتين: **الأحداث الملتقطة تلقائيًا** (التتبّع التلقائي لتفاعلات المستخدم) و**أحداث DevTools** (المراقبة التقنية وتصحيح الأخطاء).

### الأحداث الملتقطة تلقائيًا
| **اسم الحدث**  | **الوصف** |
| --- | --- |
| Click | يسجّل نقرات المستخدم. |
| Text Input | يتعقّب النص المُدخل في الحقول. |
| Visited URL | يسجّل الصفحات التي تمت زيارتها |
| Tagged Element | يتعقّب العناصر الموسومة بوسوم مخصصة. |

### أحداث DevTools

|  **اسم الحدث**  | **الوصف** |
| --- | --- |
| Network Request | يراقب نشاط الشبكة. |
| GraphQL | يسجّل استعلامات GraphQL واستجاباتها. |
| State Action | يتعقّب تغييرات الحالة في تطبيقك. |
| Error Message | يلتقط رسائل الخطأ في وحدة التحكم. |
| Average CPU Load | يقيس متوسط استخدام وحدة المعالجة المركزية. |
| Average Memory Usage | يتعقّب متوسط استهلاك الذاكرة. |

## فهم الفلاتر

تساعدك **الفلاتر** في OmniSearch على تضييق نطاق عمليات إعادة تشغيل الجلسات وفق معايير محددة، مما يتيح لك التركيز على أكثر جلسات المستخدمين صلة. وهي تعمل بالتنسيق مع الأحداث لإنشاء عمليات بحث دقيقة عبر بيانات جلساتك.

تُنظَّم الفلاتر في أربع فئات رئيسية:

### فلاتر المستخدم

|  **اسم الفلتر**  | **الوصف** |
| --- | --- |
| User ID | تصفية مستخدمين محددين باستخدام معرّفاتهم الفريدة. |
| Anonymous User ID | تصفية جلسات المستخدمين الذين ليس لديهم هوية مسجَّل دخولها. |

### فلاتر الجلسة

|  **اسم الفلتر**  | **الوصف** |
| --- | --- |
| Country | تصفية الجلسات حسب البلد. |
| City | تصفية الجلسات حسب المدينة. |
| State/Province | تصفية الجلسات حسب الولاية أو المقاطعة. |
| Operating System (OS) | تصفية الجلسات حسب نظام التشغيل المستخدم. |
| Browser | تصفية الجلسات حسب نوع المتصفح. |
| Device | تصفية الجلسات حسب جهاز المستخدم. |
| Platform | تصفية الجلسات حسب نوع المنصة. |
| Version ID | تصفية الجلسات حسب إصدار التطبيق. |
| Referrer | تصفية الجلسات بناءً على عنوان URL المُحيل. |
| Duration | تصفية عمليات إعادة تشغيل الجلسات حسب مدتها. |
| UTM Source | تصفية الجلسات حسب معامل UTM source. |
| UTM Medium | تصفية الجلسات حسب معامل UTM medium. |
| UTM Campaign | تصفية الجلسات حسب معامل UTM campaign. |

### فلاتر المشكلات

|  **اسم الفلتر**  | **الوصف** |
| --- | --- |
| Rage Click, Dead Click, Excessive Scrolling, Mouse Thrashing | تحديد الجلسات التي يظهر فيها إحباط المستخدم الناتج عن النقرات المتكررة، أو الأزرار أو الروابط المعطّلة، أو التمرير المفرط، أو حركة الماوس المضطربة…. |
| Bad Request | تحديد الجلسات التي تتضمن أخطاء HTTP 400. |
| Missing Resource | العثور على الجلسات التي تحتوي على موارد مفقودة |
| CPU Issues | تحديد الجلسات ذات الاستخدام المرتفع لوحدة المعالجة المركزية. |
| Slow Resource | تحديد الجلسات ذات الموارد البطيئة التحميل. |
| Slow Page Load | تحديد الجلسات ذات أوقات تحميل الصفحات البطيئة. |
| Crash | تحديد الجلسات التي تعطّل فيها التطبيق أو الصفحة. |
| Errors | تحديد الجلسات التي تتضمن أخطاء التطبيق. |

### فلاتر البيانات الوصفية

|  **اسم الفلتر**  | **الوصف** |
| --- | --- |
| Metadata key | استخدم البيانات الوصفية المعرَّفة بشكل مخصص لتصفية الجلسات. إذا لم تكن معرَّفة، فاطّلع على كيفية إضافة البيانات الوصفية [هنا](/tutorials/metadata) |
