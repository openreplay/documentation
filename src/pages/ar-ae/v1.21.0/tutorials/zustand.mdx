Zustand هو بديل سريع جدًا وبسيط لحلول إدارة الحالة مثل Redux. باستخدام [إضافة Zustand](https://www.npmjs.com/package/@openreplay/tracker-zustand) الخاصة بنا، يمكنك تتبع تغييرات الحالة طوال دورة حياة تطبيقك.

## تثبيت الإضافة

يمكن تثبيت الإضافة بسهولة باستخدام الأمر التالي:

```jsx
npm i @openreplay/tracker-zustand
```

## استخدام الإضافة في تطبيق Next.js

في هذا المثال، سنستخدم نفس التطبيق المعروض في [درس Next.js الخاص بالـ tracker لدينا](https://docs.openreplay.com/tutorials/next)، لذا إذا كنت ترغب في الحصول على مزيد من التفاصيل حول مزوّد Context المستخدَم، فاطّلع على ذلك الدرس أولًا.

**ملاحظة**: يوجد الكود المصدري الكامل لهذا المثال في [مستودع Github](https://github.com/deleteman/nextjs-commerce-example/tree/zustand-store) هذا، فاطّلع عليه إذا كنت بحاجة إلى مزيد من التفاصيل.

يتيح مزوّد Context الذي أستخدمه إمكانية تهيئة Tracker Plugins، لذا بعد تثبيت الإضافة، سنعدّل ملف _app.tsx بإضافة عبارة الاستيراد أدناه:

```jsx
import trackerZustand from '@openreplay/tracker-zustand'
```

وبعد ذلك، يجب أن تبدو دالة App الرئيسية على النحو التالي:

```jsx
export default function MyApp({ Component, pageProps }: AppProps) {
  const Layout = (Component as any).Layout || Noop

  useEffect(() => {
    document.body.classList?.remove('loading')
  }, [])

  let plugins = [
    {
      fn: trackerZustand,
      name: 'zustand',
    },
  ]

  return (
    <TrackerProvider
      config={{
        plugins
      }}
    >
      <Head />
      <ManagedUIContext>
        <Layout pageProps={pageProps}>
          <Component {...pageProps} />
        </Layout>
      </ManagedUIContext>
    </TrackerProvider>
  )
}
```

سيؤدي هذا تلقائيًا إلى تهيئة الإضافة مع الـ tracker، كما سيتيح أيضًا البيانات التي يتم إرجاعها عند تنفيذ الدالة الرئيسية للإضافة.

الخطوة التالية هي إنشاء store، وسنضيف بعد لحظات منتجات من موقع التجارة الإلكترونية إلى هذا الـ store.

### إنشاء store باستخدام إضافة Zustand

يتطلب استخدام الإضافة تغيير الطريقة التي تنشئ بها عادةً نسخة من store في Zustand، لذا لا تتردد في نسخ ولصق الكود التالي كقالب:

```jsx
import create from 'zustand'

const productStore = null
export const useProductsStore = (logger) => {
  if (!productStore) {
    //use a singleton pattern to make sure we only create the store the first time around
    productStore = createProductStore(logger)
  }
  return productStore
}

function createProductStore(logger) {
  return create(
    logger((set) => ({
      products: [],
      addProduct: (p) =>
        set((state) => {
          const prodFound = state.products.find((prod) => {
            return p.id == prod.id
          })
          if (prodFound) {
            p.amount = prodFound.amount + 1
          } else {
            p.amount = 1
          }

          return {
            products: [...state.products.filter((prod) => p.id != prod.id), p],
          }
        }),
    }))
  )
}
```

ستنشئ الدالة المُصدَّرة الـ store في المرة الأولى التي يُستخدَم فيها، لكنها بعد ذلك ستُرجع ببساطة الـ store الموجود.

من ناحية أخرى، ستنشئ دالة createProductStore الـ store، وتضبط المنطق الخاص بدالة addProduct، وتستخدم دالة logger التي نزوّدها بها. وهذا الـ logger، كما خمّنت على الأرجح، هو ذاته الذي حصلنا عليه من استخدام إضافة Zustand.

وبعد أن أصبح الـ store جاهزًا، يمكننا البدء بإضافة المنتجات إليه.

## إضافة المنتجات إلى الـ store الخاص بنا

يُصدّر الـ store دالة addProduct، التي تتولى تتبع المنتجات الموجودة في سلة التسوق لدينا.

لاستيراد الدالة إلى الكود الخاص بك، عليك القيام بما يلي:

1. استيراد مزوّد السياق (context provider) الخاص بالـ Tracker واستخدامه.
2. الحصول على السمة pluginsReturnedValue من السياق.
3. استخدامها لاستدعاء دالة zustand (لاحظ أن الدالة مسمّاة وفقًا للسمة name عند تهيئة الإضافة أعلاه)، وتمرير سمة نصية لإعطاء الـ store اسمًا (سيُستخدَم هذا الاسم أثناء إعادة التشغيل).
4. ستُرجع دالة zustand الـ logger الذي سنستخدمه عند استدعاء الـ hook المسمى useProductStore من zustand (وهو الذي أنشأناه للـ store).

يبدو هذا كله أمرًا مبالغًا فيه، لكنه في جوهره يُترجَم إلى ما يلي:

```jsx
const { pluginsReturnedValues } = useContext(TrackerContext)

const useProductStoreTracked = useProductsStore(
  pluginsReturnedValues.zustand('products_store')
)
const { addProduct } = useProductStoreTracked()
```

الآن، يمكنك ببساطة استدعاء دالة addProduct، وستضيف منتجات السلة إلى الـ store.

## قراءة البيانات من الـ store

كما يمكنك أن تتوقع على الأرجح، إذا أردت قراءة البيانات من store الخاص بـ Zustand، فعليك اتباع المجموعة نفسها من الخطوات. لكن بدلًا من أخذ دالة addProduct، سنُفكّك المصفوفة products.

على النحو التالي:

```jsx
const { pluginsReturnedValues } = useContext(TrackerContext)

const useProductStoreTracked = useProductsStore(
  pluginsReturnedValues.zustand('products_store')
)
const { products } = useProductStoreTracked()
```

يمكنك بعد ذلك التكرار على products بشكل اعتيادي:

```jsx
<div>
  {products.map((p: Product, idx: Number) => {
    return (
      <div key={+idx}>
        {p.name}({p.amount})
      </div>
    )
  })}
</div>
```

يكرّر الكود أعلاه على قائمة products ويعرضها على الشاشة. وفي كل مرة تضيف فيها منتجًا آخر إلى الـ store، سيتم تحديثه هنا أيضًا.
