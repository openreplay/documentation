## التثبيت

سيقوم السطر التالي بتثبيت أداة التتبع ومعها حزمة التطوير حتى تتمكن من الاستفادة من جميع ميزات أداة التتبع.

```bash
npm i @openreplay/react-native
```

## التهيئة

عند إنشاء مثيل من أداة تتبع OpenReplay، تتوفر عدة خيارات إعداد يمكنك تقديمها لتخصيص جوانب عديدة من التسجيل وتجربة التسجيل. يجب عليك تعيين الخيار `projectKey` في المُنشئ. يمكنك الحصول على هذه القيمة من لوحة تحكم OpenReplay الخاصة بك ضمن 'Preferences > Projects'.

<Aside type="tip">
	لاحظ أنه إذا كان لديك عدة مشاريع، فيجب أن تنتبه إلى مفتاح المشروع الذي تختاره.
</Aside>


### إضافة كود التتبع

أضف الكود التالي إلى ملفك الجذري:

```js
import Openreplay from '@openreplay/react-native';

// ...
useEffect(() => {
	OpenReplay.tracker.startSession(
		'yourProjectKey',
		options, // explained below, set {} if empty
		'https://local.openreplay.instance/ingest' // not required if you're using managed version
	);
}, [])
```

### تنقية البيانات

راجع [كيفية تنقية البيانات في عمليات إعادة تشغيل React Native](/rn-sdk/sanitization) لمزيد من التفاصيل.

## خيارات التهيئة

افتراضيًا، تكون جميع الخيارات مُعيَّنة على `true`:

- `crashes: Bool` يُفعِّل تحليلات الأعطال (crashlytics).
- `analytics: Bool` يُفعِّل تتبع التحليلات لطرق العرض المُعلَّمة.
- `performances: Bool` يُفعِّل مُستمِع الأداء.
- `logs: Bool` يُفعِّل مُستمِع السجلات.
- `screen: Bool` يُفعِّل مُسجِّل الشاشة.
