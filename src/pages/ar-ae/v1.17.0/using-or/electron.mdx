هناك طريقتان مختلفتان لدمج OpenReplay في تطبيق Electron.

وأيًّا كانت الطريقة التي تختارها، سيتعيّن عليك التحقق من إعدادات CSP الخاصة بك للسماح بواجهات API التالية:
- WebWorkers (تُستخدم لتخزين دفعات الرسائل وترميزها وإرسالها)
- Websockets (تُستخدم في إضافة assist لنقل الجلسات في الوقت الفعلي)
- Peers (تُستخدم في إضافة assist لإجراء المكالمات)

## إضافة كود المقتطف

يمكنك إضافة كود مقتطف OpenReplay مباشرةً إلى جذر ملف HTML الخاص بتطبيقك.
هذه هي الطريقة الأسهل، لكنها تأتي أيضًا بجميع قيود مقتطف JS.

## استخدام مكتبة NPM

إذا كنت تستخدم React أو Angular أو Vue، فيمكنك اتباع النهج نفسه تمامًا وإنشاء نسخة من tracker في عملية
العرض (rendering) داخل تطبيقك. سيتيح لك ذلك استخدام جميع ميزات OpenReplay، بما في ذلك جميع الإضافات.

ولاستخدام واجهة API الخاصة بـ tracker خارج سياق العرض، سيتعيّن عليك استخدام IPC لتشغيل الإجراءات عبر الأحداث
التي ترسلها من العملية الرئيسية.

## الاستخدام مع إضافة Assist


في حالة إضافة Tracker-Assist، سيتعيّن عليك إضافة الخاصية `socketHost` التي ستشير إلى نقطة الاستيعاب (ingest point) الخاصة بك
بدون اللاحقة `/ingest`، وذلك لضمان توجيه WS الافتراضي بشكل صحيح.

بالنسبة لمكتبة NPM:
```js
new Tracker({ ingestPoint: 'https://mywebsite.com/ingest' })

// ...

// or https://app.openreplay.com if you're using SaaS version
tracker.use(trackerAssist({ socketHost: 'https://mywebsite.com' }))
```

بالنسبة لمقتطف الكود:
```js
  var initOpts = {
    projectKey: "...",
		ingestPoint: "https://mywebsite.com/ingest",
    assistSocketHost: "https://mywebsite.com"
  };
```

وإلا فإن WS سيحاول إنشاء الاتصال على `ws://file/ingest`، وهو ما سيفشل.
