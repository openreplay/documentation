إنّ تشغيل أداة التتبع في تطبيق React أمر بسيط للغاية. 

## إضافة شيفرة التتبع

على الرغم من أنه يمكنك [الحصول على نسخة جاهزة للنسخ واللصق من الشيفرة مباشرةً من المنصة عند إنشاء مشروع جديد](/using-or/snippet#getting-the-snippet)، إلا أن تلك الشيفرة بسيطة جدًا وستجعل عملية التخصيص وإضافة الإضافات (plugins) أصعب بكثير.

بافتراض أنك تبني تطبيق صفحة واحدة (SPA - Single Page Application)، فكل ما عليك فعله هو إضافة الشيفرة التالية إلى نقطة الدخول (entry point) الخاصة بتطبيقك:

```jsx
import Tracker from '@openreplay/tracker';

const tracker = new Tracker({
  projectKey: "<your project key>",  
});

function YourMainComponent({props}) {
    //some code here...

    useEffect(() => {
        tracker.start();
    }, [])

    return //....
}
```

بعبارة أخرى، يمكنك إنشاء نسخة من أداة التتبع (instance) في بداية السكربت، وبمجرد تحميل الصفحة يمكنك بدء تشغيلها باستخدام الـ hook المسمى `useEffect`.

<Aside type="tip">
**ملاحظة لمستخدمي الاستضافة الذاتية (self-hosted)**: إذا كنت تستخدم نسخة الاستضافة الذاتية من OpenReplay، فسيتعين عليك أيضًا تحديد خاصية الإعداد `ingestPoint`. 
تحدد هذه الخاصية نقطة استقبال (endpoint) بيانات أداة التتبع. لا يحتاج مستخدمو السحابة (Cloud) إلى ذلك، 
لأن أداة التتبع ستعرف افتراضيًا موقع نسخة الـ SaaS من هذه النقطة، أما إذا كنت تستضيفها ذاتيًا (self-hosting)، فستحتاج إلى تحديدها 
(يجب أن تكون شيئًا مثل `https://openreplay.mydomain.com/ingest`) 
</Aside>

### التعامل مع "projectKey" في شيفرتك

للعلم، فإن الـ `projectKey` الذي تقوم بإعداده هو المفتاح الذي توفّره منصة OpenReplay. ولأسباب أمنية، لا ينبغي تخزين هذه القيمة مكتوبة بشكل ثابت (hardcoded) في شيفرتك، بل في نوع ما من ملفات/أنظمة الإعداد.

توجد خيارات مختلفة لحل هذه المشكلة؛ وبحسب إعداداتك، يمكنك استخدام شيء مثل [دعم `create-react-app` لمتغيرات البيئة (ENV)](https://create-react-app.dev/docs/adding-custom-environment-variables/).

## هل تبني شيئًا أكثر تعقيدًا؟
إذا كنت تبني تطبيقًا معقدًا باستخدام React، فمن المرجّح أنك تستخدم [Next](/using-or/next) أو [Remix](/using-or/remix) أو شيئًا مشابهًا. تأكد من الاطلاع على [قسم الإعداد](/using-or) لدينا للعثور على إطار العمل (framework) الذي تعمل به.
