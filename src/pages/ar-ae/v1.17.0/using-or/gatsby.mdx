تثبيت متتبّع OpenReplay في مشروع قائم على Gatsby أمر بسيط نسبيًا.

![Sesion replay of a Gatsby site](/images/gatsby-ss.png)

بالنظر إلى أن Gatsby يستخدم React في الخلفية، فإن كل ما علينا فعله هو إضافة شيفرة المتتبّع داخل الصفحة الرئيسية واستدعاء الدالة `start` باستخدام الـ hook المسمّى `useEffect`.

دعونا نلقي نظرة.

## لماذا نُعِدّ المتتبّع على موقع ثابت؟

عدم وجود محتوى ديناميكي داخل تطبيقك لا يعني أنك لا ترغب في الاستفادة من بعض الميزات الأخرى في OpenReplay.

ففي حين أنك لن تستخدم إعادة تشغيل الجلسات للعثور على الأخطاء، فإنك ستتمكن من التقاط مقاييس مثيرة للاهتمام وإنشاء [لوحات معلومات مخصّصة خاصة بك](https://docs.openreplay.com/tutorials/custom-dashboard) حول هذه المقاييس.

يمكنك استخدام مقاييس الأداء و web vitals وحتى المقاييس المتعلقة بالموارد للحصول على فكرة عامة عن تجربة المستخدمين على موقعك.

## إعداد المتتبّع داخل Gatsby

إذا كان لديك موقع Gatsby ثابت ونقطة الدخول إليه هي الملف `index.tsx` الموجود داخل مجلد `pages`، فستحتاج إلى إضافة الشيفرة التالية إليه:

```jsx
import Tracker from '@openreplay/tracker';
const tracker = new Tracker({
  projectKey: process.env.GATSBY_OPENREPLAY_PROJECT_KEY
});

const Index = ({data, location }) => {

  React.useEffect(() => {
    tracker.start();
  }, [])

   //the rest of your code here...
}
```

يتم إنشاء كائن المتتبّع عند تجميع الصفحة (سواء كان ذلك عبر SSR أو التصيير من جانب العميل)، لكن الدالة start لا يمكن استدعاؤها إلا من المتصفّح، لذا علينا التأكد من استدعائها بعد تركيب المكوّن (ومن هنا جاء استخدام الـ hook المسمّى `useEffect` هنا). والمصفوفة الفارغة كوسيطة ثانية للـ hook المسمّى `useEffect` تضمن تنفيذ دالة رد النداء مرة واحدة فقط.

💡**ملاحظة لمستخدمي الاستضافة الذاتية (self-hosted):** إذا كنت تستخدم نسخة OpenReplay ذاتية الاستضافة، فسيتعين عليك أيضًا تحديد خاصية الإعداد `ingestPoint` عند إنشاء المتتبّع. تحدّد هذه الخاصية نقطة نهاية استقبال بيانات المتتبّع. لا يحتاج مستخدمو السحابة إلى ذلك، لأن المتتبّع سيعرف افتراضيًا أين توجد نسخة SaaS من نقطة النهاية هذه، ولكن إذا كنت تستضيفه ذاتيًا، فستحتاج إلى تحديدها (ينبغي أن تكون شيئًا مثل `https://openreplay.mydomain.com/ingest`)

## التعامل مع متغير البيئة في Gatsby

كإجراء أمني، نوصي بعدم تضمين مفتاح المشروع بشكل ثابت داخل الشيفرة. وهذا يعني أنه سيتعين عليك تصديره كمتغير بيئة (ENV) حتى يتمكن Gatsby من قراءته.

سيوجد مفتاح مشروع OpenReplay عندئذٍ داخل متغير يُسمّى `GATSBY_OPENREPLAY_PROJECT_KEY`

لاحظ كيف يبدأ الاسم بالبادئة `GATSBY_`، وهذا سيخبر Gatsby بالتأكد من إتاحة المتغير في شيفرة الواجهة الأمامية أيضًا. وإلا فلن تتمكن من قراءة القيمة ما لم تكن تنفّذ شيفرة Node.js.
