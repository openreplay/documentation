‏Pinia هي مكتبة إدارة الحالة البديلة في مشاريع Vue. حيث كنت تستخدم VueX سابقاً، ستستخدم الآن Pinia. باستخدام [إضافة Pinia](/plugins/pinia) الخاصة بنا، يمكنك تتبّع تغيّرات الحالة طوال دورة حياة تطبيقك.

## تثبيت الإضافة

يمكن تثبيت الإضافة بسهولة باستخدام الأمر التالي:

```jsx
npm i @openreplay/tracker-vuex
```

## استخدام الإضافة من داخل تطبيق Vue الخاص بك

في هذا المثال، سنستخدم تطبيق Vue نموذجياً لعرض كيفية إضافة هذه الإضافة والتفاعل معها داخل تطبيق حقيقي.

**ملاحظة**: يوجد الكود المصدري الكامل لهذا المثال في هذا [مستودع GitHub](https://github.com/deleteman/openreplay-vue-pinia-example)، اطّلع عليه إذا كنت بحاجة إلى مزيد من التفاصيل.

بعد تثبيت الإضافة، سنعدّل الملف `main.js` بإضافة عبارة الاستيراد أدناه:

```jsx
import trackerVuex from '@openreplay/tracker-vuex/cjs';
```

ثم سنستخدم الدالة `startTracker` لإنشاء نسخة من المتتبّع واستخدام الإضافة في خطوة واحدة.

ستكون نتيجة استدعاء هذه الدالة كائناً يحتوي على نتائج استخدام الطريقة `use` مع الإضافة.

**لاحظ** كيف أن اسم الخاصية المُفكَّكة هو نفسه قيمة الخاصية `name`. وما دمت تُبقي هاتين القيمتين متطابقتين، فلا مشكلة.

```jsx
const {piniaPlugin} = startTracker({
    projectKey: "<your project key>",
    plugins: [{
        name: 'piniaPlugin',
        fn: trackerVuex
    }]
})
```

الخطوة التالية هي إنشاء غلاف مُسمّى للمخزن (store wrapper) سنستخدمه لاحقاً لإعداد الإضافة على المخزن المطلوب.

```jsx
const piniaStoreWrapper = piniaPlugin("products")
saveStore(piniaStoreWrapper)
```

سيتعيّن استخدام `piniaStoreWrapper` لاحقاً مع المخزن الذي نقرر تتبّعه. غير أن المخزن لم يُنشأ بعد في هذه المرحلة، لذا سنحفظه عبر الدالة `saveStore`.

### الدوال المساعدة

قبل المضي قدماً، دعنا نلقي نظرة على الدالتين `saveStore` و`startTracker`.

الأولى جزء من الوحدة `storesManager`، التي تبدو هكذا:

```jsx
let storeWrapper = null;

export function saveStore(s) {
    storeWrapper = s;
}

export function getStore(store) {
    return storeWrapper;
}
```

ستتيح لنا هذه الوحدة استخدام `piniaStoreWrapper` الذي أنشأناه سابقاً في مكان آخر.

أما بالنسبة للدالة `startTracker`، فإن الكود يبدو هكذا:

```jsx
export function startTracker(config) {
    console.log("Starting tracker...")

    console.log("Project key used: ", config.projectKey)
    const trackerConfig = {
        projectKey: config.projectKey,
         //ingestPoint: config.ingestPoint,
        __DISABLE_SECURE_MODE: true
    }

    const tracker = new Tracker(trackerConfig);

    const pluginReturns = {}
    config?.plugins.forEach( p => {
        console.log("Using plugin...", p.name)
        pluginReturns[p.name] = tracker.use(p.fn())
    })

    tracker.start();
    return {
        tracker,
        ...pluginReturns
    }
}
```

تُنشئ هذه الدالة نسخة من المتتبّع باستخدام الإعدادات التي نمررها إليها. لاحظ أنني تركت الخيار `ingestPoint` كتعليق معطّل؛ وذلك لأننا نتعامل مباشرة مع نسخة SaaS، ولكن إذا كنت تتطلّع إلى استخدام هذا الكود مع نسخة مستضافة ذاتياً، فسيتعيّن عليك تحديده، وينبغي أن يبدو بشكل مشابه لهذا: `<your-instance>.yourhost/ingest`

### تتبّع مخازن Pinia الخاصة بك

بمجرد أن يصبح كود الإعداد بالكامل جاهزاً، انتقل إلى المكان الذي تستخدم فيه المخزن؛ وفي حالة مثالنا، دعنا نعدّل الملف `App.vue` لاستدعاء الدالة `getStore`، التي ستُعيد غلاف المخزن الذي حفظناه سابقاً.

وسنستخدمه لتغليف المخزن الذي نريد تتبّعه:

```jsx
<template>
  <!-- template code -->
</template>

<script lang="ts">
import { computed, defineComponent, onMounted, ref } from 'vue'
import { Product, useProductStore } from './store/products'
import { useCartStore } from './store/cart'
import {getStore} from './store/storesManager.js'

export default defineComponent({
  setup() {

    const productStore = useProductStore()
    const loading = ref(true)
    //we get the wrapper here
	  const storeWrapper = getStore()

    onMounted(async () => {
      await productStore.fetchAll()
      loading.value = false
    })

    const cartStore = useCartStore()

		//we wrap the cart store here
    storeWrapper(cartStore)
    return {
      products: computed(() => productStore.list),
      cart: computed(() => cartStore.formattedCart),
      total: computed(() => cartStore.total),
      loading,
      add: (product: Product) => cartStore.add(product),
      remove: (product: Product) => cartStore.remove(product),
    }
  }
})

</script>
```
