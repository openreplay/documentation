بمجرد [إعدادك](/mcp/setup) لتطبيق MCP وتسجيل الدخول، يمكنك أن تطلب من Claude (أو من مضيف MCP آخر) الاستعلام عن بيانات OpenReplay الخاصة بك بلغة بسيطة. تسرد هذه الصفحة ما يوفّره MCP حاليًا، مع أمثلة على المطالبات لكل حالة.

<Aside type="tip" title="نصيحة" >
  تعمل جميع المطالبات سواء أشرت إلى مشروعك بـ **الاسم** (مثل *"MyApp"*) أو بـ **معرّف الموقع** (مثل *"site 1"*). إذا لم تحدد ذلك، يستخدم MCP أحدث مشروع ذكرته.
</Aside>

## البحث في الجلسات وتصفيتها

يدعم MCP نفس عوامل التصفية المتاحة في واجهة OpenReplay: المتصفح، ونظام التشغيل، والجهاز، والدولة، ومسارات URL، والنقرات، والإدخالات، وطلبات الشبكة، والأخطاء، والبيانات الوصفية، ومعرّفات المستخدمين، والمدة.

```
Show me 20 recent sessions in MyApp
```

```
Sessions from Chrome users in France or Tunisia
```

```
Sessions with failed requests to /api/users in MyApp
```

```
Sessions where the user visited the signup page then clicked "Subscribe"
```

```
Sessions longer than 20 minutes for user tahay@asayer.io
```

تُعرض النتيجة كقائمة تفاعلية داخل المحادثة، بحيث يمكنك النقر للدخول إلى أي جلسة.

## مسارات التحويل

تقيس مسارات التحويل معدل التحويل عبر سلسلة من عناوين URL أو النقرات أو الإدخالات أو الأحداث المخصصة. ويُعرض معدل التسرّب خطوة بخطوة.

```
Show me the checkout funnel last week: /cart → /checkout → /confirm
```

```
Conversion from /pricing to /signup this month
```

```
Visited /pricing then clicked Subscribe
```

```
Page view /checkout followed by a JS exception
```

## الأكثر من X

اعثر على الصفحات الأكثر زيارة، أو المتصفحات الأكثر استخدامًا، أو الدول الأبرز، أو أنظمة التشغيل الأكثر شيوعًا، أو الأجهزة الأبرز، أو طلبات الشبكة الأكثر تكرارًا.

```

```

```

```

```

```

```
Top 5 network requests today in MyApp
```

## رحلات المستخدمين

تعرّف على كيفية تنقّل المستخدمين عبر منتجك — ما الصفحات التي يصلون إليها، وإلى أين يتجهون بعد ذلك، وأين ينسحبون.

```
Show me user journeys this month
```

```
Where do users drop off after the pricing page last week?
```

```
What pages do users visit after /signup?
```

```
Navigation paths for mobile users from France this week
```

## مشاهدة إعادة تشغيل جلسة

تُعرض عمليات إعادة التشغيل في مشغّل مضمّن داخل المحادثة. يمكنك التنقّل عبر الشريط الزمني والإيقاف المؤقت والفحص تمامًا كما في واجهة OpenReplay.

```
Show me the replay of session 7891234567890
```

```
Replay the last session in the list above
```

## المزج والدمج

تأتي القيمة الحقيقية من ربط الأدوات معًا. بعض الأمثلة:

- *"Find sessions where users hit a JS exception on /checkout this week, then show me the replay of the most recent one and tell me what happened."*
- *"Pull the funnel for /pricing → /signup → /dashboard, then for the step with the biggest drop-off show me 5 sessions that dropped off there."*
- *"Plot the timeseries of sessions from France this month. If there's a spike, find a few sessions from that day and summarize what they did."*

ولأن Claude يمكنه التفكير في النتائج بين استدعاءات الأدوات، يمكنك البقاء على مستوى المحادثة بدلًا من التنقّل بين لوحات المعلومات.
