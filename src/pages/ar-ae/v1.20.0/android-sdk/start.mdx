تُستخدم هذه الطريقة لبدء تشغيل الـ tracker. عند بدء تشغيل الـ tracker، يبدأ تسجيل الجلسة.

## التوقيع

```kotlin
OpenReplay.start(applicationContext: Context, projectKey: String, options: OROptions)
	```

### خيارات التهيئة

الخيارات المتاحة:

- `crashes: Bool` يُفعّل crashlytics.
- `analytics: Bool` يُفعّل التحليلات الخاصة بطرق العرض (views).
- `performances: Bool` يُفعّل مُستمِع الأداء (performance listener)
- `logs: Bool` يُفعّل مُستمِع السجلات (logs listener)
- `screen: Bool` يُفعّل مُسجّل الشاشة
- `wifiOnly: Bool` يُجبر الـ tracker على البدء فقط إذا كان لدى المستخدم اتصال wifi

الافتراضي: الكل `true`
