إذا لم تكن المعلومات التي يتتبعها أداة التتبع (tracker) لدينا بشكل افتراضي عند حدوث خطأ JS كافية، فيمكنك استخدام هذه الطريقة. تقوم طريقة `handleError` بالإبلاغ يدويًا عن الاستثناءات الملتقطة أو الوعود المرفوضة أو أحداث الأخطاء. راجع [الإبلاغ عن الأخطاء](/installation/error-reporting#manuallyloggingexceptions) لمزيد من التفاصيل.

## التوقيع

```js 
handleError: (e: Error | ErrorEvent | PromiseRejectionEvent, metadata?: Record<string, any>) => void
```

## المعاملات

- `e: Error | ErrorEvent | PromiseRejectionEvent`: كائن الخطأ الفعلي الذي يتم التقاطه.
- `metadata: Record<string, any>`: **[اختياري]** بنية شبيهة بالقاموس تحتوي على جميع المعلومات الإضافية التي ترغب في حفظها مرتبطة بهذا الخطأ.

## القيمة المُرجَعة

لا تُرجع هذه الطريقة أي قيم، ويمكن عرض المعلومات المحفوظة داخل منصة OpenReplay.
