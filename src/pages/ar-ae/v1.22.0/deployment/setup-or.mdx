البدء مع OpenReplay أمر بسيط. 

إذا كنت ترغب في مشاهدة درس تعليمي سريع حول كيفية البدء، فاستخدم [فيديو YouTube هذا](https://www.youtube.com/watch?v=EWbtMK3qK30)، وإلا فتابع القراءة لمزيد من التفاصيل.

هناك طريقتان لتشغيل OpenReplay وتجهيزه: باستخدام السكربت أو باستخدام حزمة npm.

## استخدام السكربت

انسخ مقتطف الكود والصقه مباشرةً داخل `</head>` في تطبيق الويب الخاص بك. يمكن العثور على السكربت في لوحة تحكم OpenReplay ضمن 'Preferences > Projects' وهو فريد لكل مشروع.

إذا كان موقعك الإلكتروني يدعم Google Tag Manager، فيمكنك اتباع [دليل GTM خطوة بخطوة](/integrations/google-tag-manager).

## استخدام NPM

ثبّت حزمة npm أولاً.

```bash
npm i @openreplay/tracker
```

ثم قم بتهيئة الحزمة من نقطة دخول قاعدة الكود الخاصة بك وابدأ التتبّع (tracker). يجب عليك ضبط الخيار `projectKey` في الدالة البانية (constructor). يمكن العثور على قيمته في لوحة تحكم OpenReplay ضمن 'Preferences > Projects'.

### إذا كان موقعك الإلكتروني تطبيق صفحة واحدة (SPA)

يمكنك استخدام الـ tracker كنسخة وحيدة (singleton)، وهي الطريقة الموصى بها لاستخدامه. 
بهذه الطريقة، يمكنك الوصول إلى الـ tracker من أي مكان في تطبيقك:

```js
import { tracker } from '@openreplay/tracker'

tracker.configure({
	projectKey: PROJECT_KEY,
  ingestPoint: "https://openreplay.mydomain.com/ingest", // when dealing with the self-hosted version of OpenReplay
})

tracker.start() // returns a promise with session info (sessionID, sessionHash, userUUID)
```

أو يمكنك استخدام الفئة `Tracker` مباشرةً للتعامل مع نسختها يدويًا:

```js
import OpenReplay from '@openreplay/tracker';
//...
const tracker = new OpenReplay({
  projectKey: PROJECT_KEY,
  ingestPoint: "https://openreplay.mydomain.com/ingest" // only required if using the self-hosted version of OpenReplay
});

tracker.start();
```

### إذا كان تطبيق الويب الخاص بك مُصيّرًا من جانب الخادم (SSR)

اتبع المقتطف أدناه إذا كان تطبيق الويب الخاص بك مُصيّرًا من جانب الخادم (SSR) (أي NextJS وNuxtJS). 
تأكّد من أن الـ tracker يعمل في بيئة المتصفح وأن `tracker.start()` يُستدعى بمجرد تقديم التطبيق إلى العميل (أي ضمن `useEffect` أو `componentDidMount`).

```js
import OpenReplay from '@openreplay/tracker/cjs';
//...
const tracker = new OpenReplay({
      projectKey: PROJECT_KEY
});
//...
function MyApp() {
  useEffect(() => { // use componentDidMount in case of React Class Component
    tracker.start();
  }, []);
}
```

إذا كنت ترغب في التعمّق أكثر في تهيئة الـ tracker، فراجع [JavaScript SDK](/installation/javascript-sdk#options) للاطلاع على قائمة الخيارات المتاحة.

## الخطوات التالية

الآن وقد قمت بتثبيت الـ tracker ونشر تطبيق الويب الخاص بك، يُفترض أن تظهر تسجيلات الجلسات خلال بضع دقائق. أحسنت!

في غضون ذلك، يمكنك المضي قدمًا في عملية الإعداد لتحقيق أقصى استفادة من OpenReplay. كلما زاد ما تلتقطه، كان ذلك أفضل لإعادة إنتاج المشكلات وإصلاحها بسرعة:
- [تحديد هوية المستخدم](/installation/identify-user): اربط معرّف المستخدم الداخلي الخاص بك بالجلسة التي يجري تسجيلها.
- [البيانات الوصفية (Metadata)](/installation/metadata): يُشار إليها أيضًا بالسمات (traits) أو متغيّرات المستخدم، وتوفّر البيانات الوصفية معلومات إضافية عن المستخدمين أثناء تسجيل الجلسات.
- [الأحداث المخصّصة](/installation/custom-events): أضِف مزيدًا من الذكاء عبر تسجيل أحداث خاصة بمجالك إلى جانب عمليات إعادة تشغيل الجلسات.
- [الإبلاغ عن الأخطاء](/installation/error-reporting): أبلِغ عن الأخطاء والاستثناءات وشاهِدها ضمن علامة التبويب ERRORS.
- [رفع Source-Maps](/installation/upload-sourcemaps): اطّلع على سياق الكود المصدري المستخرَج من تتبّعات المكدّس (stack traces) بصورته الأصلية.
- [الإضافات (Plugins)](/plugins): افهم ما يحدث في الخلفية عبر التقاط المخزن (store) الخاص بتطبيقك ومراقبة استعلامات HTTP وGraphQL لديك.
- [التكاملات](/integrations): اربط OpenReplay بأدوات التسجيل (logging) الخاصة بك وقم بمزامنة أخطاء الواجهة الخلفية (backend) مع عمليات إعادة تشغيل الجلسات.
