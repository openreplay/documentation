## الوضع الخاص

**الوضع الخاص** هو إعداد جديد يمنع تسجيل جميع النصوص والبيانات الحساسة. عند تفعيله، لن يلتقط متعقّب OpenReplay أي نص مرئي أو بيانات شبكة أو سجلات وحدة التحكم. يتم إخفاء كل شيء من المصدر، بحيث لا تغادر أي بيانات حساسة المتصفح أبدًا.

يكون هذا مفيدًا إذا كنت ترغب في رؤية كاملة لسلوك واجهة المستخدم مع الحفاظ على محتوى المستخدم وبياناته خاصة تمامًا.

## كيفية تفعيل الوضع الخاص

يمكنك تفعيل الوضع الخاص على مستوى المتعقّب عند تهيئة OpenReplay.

#### باستخدام NPM

```js
import { tracker } from '@openreplay/tracker'

tracker.configure({
    projectKey: PROJECT_KEY,
    privateMode: true
})
tracker.start();
```

عند ضبط `privateMode` على `true`، سيتم إخفاء ما يلي:

- جميع النصوص المرئية على الصفحة  
- سمات `alt` في الصور  
- قيم `placeholder` في حقول الإدخال  
- نصوص التسميات

لا حاجة إلى أي إعداد إضافي. بمجرد التفعيل، ستسجّل الجلسة التخطيط والتفاعل، ولكن دون أي من المحتوى.


## متى تستخدمه

استخدم `privateMode` إذا كنت:

- بحاجة إلى ضمان خصوصية كاملة لبيانات المستخدم
- تختبر OpenReplay في **بيئة سحابية**، ولكن تخطط **لاستضافته ذاتيًا لاحقًا**، و**لا يمكنك إرسال أي بيانات حقيقية إلى الخارج**
- تعمل في صناعات خاضعة للتنظيم حيث **تأتي الخصوصية أولًا**
