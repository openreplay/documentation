如果你正在使用 VueX 管理基于 Vue 的应用的状态，那么集成 [OpenReplay 的插件](https://docs.openreplay.com/plugins/vuex) 来追踪状态更新非常简单。

如果你想跟着一起操作，请使用此处[发布的代码](https://github.com/deleteman/openreplay-vuex-example)。

## 将 Tracker 集成到你的代码中

我们首先需要关注的是将 OpenReplay 的 tracker 集成到项目中。我们处理的是一个基于 Vue 的项目，这意味着我们的入口点将是 `main.js` 文件。首先使用以下命令安装 tracker：

```tsx
yarn add @openreplay/tracker
```

或者，如果你愿意，也可以使用 npm：

```tsx
npm install @openreplay/tracker
```

完成这一步后，在你于平台内设置好项目之后，获取项目的 KEY，并将其保存在位于项目根目录的 `.env` 文件中。

```bash
VUE_APP_OPENREPLAY_PROJECT_KEY=<your project key here>
```

请注意该环境变量的名称。它以 `VUE_APP_` 开头，因为当我们这样命名时，WebPack 会在我们使用 `process.env.VUE_APP_OPENREPLAY_PROJECT_KEY` 代码的任何地方替换其值，从而免去我们将实际配置文件发送到客户端的麻烦。

现在，在你的 `src` 文件夹中创建一个 `tracker` 文件夹，并在其中创建一个 `index.js` 文件。该文件将导出一个函数：`startTracker`，它会完成所有配置并启动 tracker。

```jsx
import Tracker from '@openreplay/tracker';

import {v4 as uuidV4} from 'uuid'

function defaultGetUserId() {
   return uuidV4() 
}

export function startTracker(config) {

    console.log("Starting tracker...")

    const getUserId = (config?.userIdEnabled && config?.getUserId) ? config.getUserId : defaultGetUserId
    let userId = null;

    const trackerConfig = {
        projectKey: config.projectKey
    }

    const tracker = new Tracker(trackerConfig);

    const pluginReturns = {}
    Object.keys(config?.plugins).forEach( pk => {
        pluginReturns[pk] = tracker.use(config?.plugins[pk]())
    })
 
    if(config?.userIdEnabled) {
        userId = getUserId()
        tracker.setUserID(userId)
    }
    console.log("tracker: user id: ", userId)

    tracker.start();
    return {
        tracker,
        userId,
        ...pluginReturns
    }
}
```

现在你需要关注的是，这个函数已经完成了你所需的一切。你只需在调用它时传入项目密钥即可。就像这样（在 `main.js` 文件中）：

```jsx
import {startTracker} from './tracker/index'

let {vuexTracker} = startTracker({
    projectKey: process.env.VUE_APP_OPENREPLAY_PROJECT_KEY,
})
```

正如我们已经看到的，在渲染页面时，`process.env.VUE_APP_OPENREPLAY_PROJECT_KEY` 这一行将被替换为你的实际项目密钥。

现在，让我们添加 VueX 插件，以便我们也能开始追踪状态变更。

## 添加 VueX 插件

只需一行命令即可轻松安装该插件：

```jsx
yarn add @openreplay/tracker-vuex
```

之后，我们可以将其导入到代码中，并将其作为配置对象的一部分传递给 `startTracker` 函数。

回到该函数的定义，注意我们已经有了处理插件的代码。事实上，配置中的 `plugins` 元素是一个 map，它接收插件（使用我们自己选择的键），并返回我们调用 `use` 方法所得到的内容，同样存放在我们为该插件所用的相同键下。

让我演示给你看，这同样是在 `main.js` 文件中：

```jsx
//your other imports go here...
import trackerVuex from '@openreplay/tracker-vuex';
import store from './store'

let {vuexTracker} = startTracker({
    userIdEnabled: true,
    projectKey: process.env.VUE_APP_OPENREPLAY_PROJECT_KEY,
    plugins:{
      'vuexTracker': trackerVuex
    } 
})

new Vue({
  router,
  store: store([vuexTracker]),
  render: h => h(App)
}).$mount('#app')
```

请注意我们是如何：

1. 使用 `plugins` map 调用 `startTracker` 函数的。字符串 “vuexTracker” 是我当时随手想出来的，但你在这里可以使用任意字符串。
2. 对响应进行解构并获取键 “vuexTracker”（也就是我们为 map 所用的相同键）。只要这两个键相匹配，你就可以使用任意字符串。
3. 将 `vuexTracker` 函数作为参数传递给 `store` 函数的。

vuexTracker 变量本质上包含了我们的 VueX 插件，该插件将被挂接到 VueX store 中，为此我们使用以下代码：

```jsx
import Vue from 'vue'
import Vuex from 'vuex'

import account from './account'
import product from './product'

Vue.use(Vuex)

export default function (plugins = []) {
  return function() {
    const Store = new Vuex.Store({
      modules: {
        account,
        product
      },

      strict: process.env.DEV,
      plugins
    })

    return Store
  }
}
```

上面的代码片段是用于设置 store 的标准代码，我们只是添加了配置插件的能力，以确保我们能够追踪应用中状态所发生的变化。
