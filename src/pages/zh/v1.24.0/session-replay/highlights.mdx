**Highlights** 提供了一种结构化的方式，用于隔离并保存会话回放中有意义的片段。无论是出现的错误、用户的交互，还是值得关注的事件，Highlights 都能确保这些时刻易于访问，便于日后查看、协作或编写报告。

## 使用场景

在查看会话回放时，使用 **Highlights** 可以：

1. 隔离并保存用户遇到错误的片段（例如：失败的 API 调用、控制台错误……）。
2. 识别用户行为暴露出可用性问题或摩擦点的时刻。
3. 提供上下文，为设计改进或未来迭代提供参考。

## 工作原理：创建并保存 highlight

**第 1 步：** 打开一段会话回放，定位关键片段，并点击菜单栏中的 **'Highlights'** 按钮。

## 管理 Highlights

#### 访问已保存的 Highlights

打开 **Highlights** 标签页即可查看所有已保存的条目。Highlights 可按以下条件进行筛选：

- 标签（Issue、Design、Note）。
- 创建者（My Highlights 或 Team Highlights）。
- 搜索词（例如，备注内容）。

<div style={{ display: 'flex', justifyContent: 'center', alignItems: 'center' }}>
  <img src="/images/products/session-replay/highlights/highlight_view.png" alt="highlights" style={{ width: '700px', height: 'auto' }} />
</div>

#### 从高亮片段开始播放回放

从已保存的片段开始重播该会话。使用 **Navigate to Full Session** 按钮，从相关时间戳开始查看完整会话。

<div style={{ display: 'flex', justifyContent: 'center', alignItems: 'center' }}>
  <img src="/images/products/session-replay/highlights/highlight_manage.gif" alt="highlights" style={{ width: '800px', height: 'auto' }} />
</div>

<Aside type="tip">
    已保存的 highlights 会在会话回放的_**时间轴**_上标记出来，并可在 _**Activity**_ 标签页中访问。
</Aside>

## 协作与共享

- **调整可见性：**

    定义某个 highlight 是仅对你自己可见（Private），还是对你的团队可见（Team）。
    
- **共享 Highlights：**
    
    使用 **Copy Link** 选项与同事共享某个特定的 highlight。
    
- **编辑或删除 Highlights：**
    
    修改备注，或在 highlight 不再相关时将其删除。

## **充分发挥 Highlights 的作用**

1. 专注于只保存会话回放中最相关的片段。
2. 使用备注和标签提供上下文并对 highlights 进行分类，以便快速查阅。
3. 与团队共享 highlights，以便快速解决问题或为设计改进提供参考。
