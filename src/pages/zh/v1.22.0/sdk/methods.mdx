## 方法

- [Constructor](/sdk/constructor): 了解 tracker 的所有初始化选项。
- [event()](/sdk/methodsevent): 使用此方法发送自定义事件，将其记录为回放的一部分。
- [getSessionToken()](/sdk/methods/get-session-token): 获取当前活动会话的令牌。
- [getSessionID()](/sdk/methods/get-session-id): 获取当前活动会话的 ID。
- [getSessionURL()](/sdk/methods/get-session-url): 获取用于回放当前会话的 URL。 
- [handleError()](/sdk/methods/handle-error): 捕获自定义错误信息并将其保存为录制的一部分。
- [isActive()](/sdk/methods/is-active): 获取活动会话的当前状态。
- [issue()](/sdk/methods/issue): 捕获自定义技术事件并将其保存为回放的一部分。
- [setUserID()](/sdk/methods/set-user-id): 使用此方法识别您的用户。
- [setMetadata()](/sdk/methods/set-metadata): 设置元数据字段并将其保存为录制的一部分。
- [start()](/sdk/methods/start): 开始录制会话。
- [stop()](/sdk/methods/stop): 使用此方法停止会话的录制。
- [clearPersistFlag()](/sdk/methods/clear-persist-flag): 从浏览器的 sessionStorage 中移除持久化的 Feature Flags。
- [getFeatureFlag()](/sdk/methods/get-feature-flag): 如果存在，则按键返回一个 `IFeatureFlag`。
- [onFlagsReload()](/sdk/methods/on-flags-reload): 设置在加载 Feature Flags 时使用的回调。
- [isFlagEnabled()](/sdk/methods/is-flag-enabled): 如果具有此键的标志存在且处于活动状态，则返回 `true`。
- [reloadFlags()](/sdk/methods/reload-flags): 重新加载所有标志（如果用户会话的元数据、userId 或某些标志条件可能需要的任何其他信息发生了变化，此方法会很有用）。
- [getAllFeatureFlags()](/sdk/methods/get-all-feature-flags): 返回当前处于活动状态的所有标志（`IFeatureFlag[]`）。
- [coldStart()](/sdk/methods/cold-start): 开始缓冲消息（刷新缓冲区），并可以调用 `start()` 来启用会话录制并发送缓冲区内容。
- [startOfflineRecording()](/sdk/methods/start-offline-recording): 启动离线会话录制，可以通过 `uploadOfflineRecording` 发送。
- [uploadOfflineRecording()](/sdk/methods/upload-offline-recording): 将存储的会话缓冲区上传到后端。
- [forceFlushBatch()](/sdk/methods/force-flush-batch): 强制发送当前的消息批次，而无需等待其完成。
- [trackWs()](/sdk/methods/track-ws): 创建一个钩子来捕获 WebSocket 消息。
