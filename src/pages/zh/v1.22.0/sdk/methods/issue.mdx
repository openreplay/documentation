此方法可用于报告"[技术事件](/installation/custom-events)"，也称为"issue"（问题）。这些问题可用于报告有关 tracker 默认无法捕获的某个问题的额外信息。一个很好的例子是用户在我们的电商网站上尝试支付一笔订单时，收到了支付网关返回的错误。虽然 tracker 可以记录包含该错误的 payload，但它不会将其识别为一个问题。此方法让你能够将这种行为标记为一个 issue，以便进一步筛选会话。

<Aside type="tip">
  我们还提供了一篇关于如何追踪自定义事件的详细教程，你可以在[这里](/tutorials/custom-events)查看。
</Aside>

## 方法签名

```js
issue(key: string, payload?: any): void
```

## 参数

- `key: string`：用于保存事件数据的键名。你可以在 OpenReplay 平台的筛选字段中使用该值。
- `payload: any`：与所追踪事件相关的数据。你可以在此保存任何可序列化的对象。

## 返回值

此方法不返回任何值，你将能够在 OpenReplay 平台中查看所追踪的信息。
