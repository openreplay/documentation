此方法允许你上报一个"功能性事件"。通过功能性事件，你可以标记用户会话中从功能角度来看具有重要意义的时刻，而这些时刻默认情况下我们的追踪器无法自行捕获和标记。功能性事件的一个例子是用户将某件商品加入购物车。虽然我们的追踪器能够记录该操作，但它不会知道要将其标记为会话时间线中的相关事件。不过，借助此方法，你可以手动完成标记。

更多详情请参阅 [自定义事件](/installation/custom-events#functionalevents)。

## 签名

```kotlin
data class User(val name: String, val age: Int)

OpenReplay.event("Test Event", User("John", 25))
```

## 参数

- `key: string`：保存信息所用的键的名称。该键可在 OpenReplay 平台内用于过滤和搜索。
- `payload: Encodable`：事件的内容。可以是任何可编码（JSON）的数据。
