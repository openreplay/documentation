本页介绍 OpenReplay JavaScript SDK [选项](/sdk/constructor)中的 `network` 命名空间。它涉及捕获诸如 `fetch` 和 `XHR` 等网络请求（必要时包括有效负载），并在稍后回放会话录制时检查它们。这对于理解和修复问题非常有用。

## 选项

```js
network?: {
  failuresOnly: boolean;
  sessionTokenHeader: string | boolean;
  ignoreHeaders: Array<string> | boolean;
  capturePayload: boolean;
  sanitizer: (RequestResponseData) => RequestResponseData | null;
  captureInIframes: boolean;
  axiosInstances: AxiosInstance[];
  useProxy?: boolean;
	tokenUrlMatcher?: (url: string) => boolean;
}
```

- `useProxy`：将追踪器从修补 window.fetch/window.XMLRequest 切换为使用代理对象。默认值：`true`。
- `failuresOnly`：捕获具有 4xx-5xx HTTP 状态码的请求。默认值：`false`。
- `sessionTokenHeader`：如果你已启用我们的某些后端[集成](/integrations)（例如 Sentry），你可以使用此选项指定请求头名称（或通过将选项设置为 `true` 来激活默认的 'X-OpenReplay-SessionToken'）。后者会自动附加到每个 fetch/XHR 请求中，以包含 OpenReplay sessionToken 的值。默认值：`false`。
- `tokenUrlMatcher`：获取当前追踪器请求的 url 并返回一个 `boolean`。如果存在，则仅当此函数返回 `true` 时才会应用 `sessionTokenHeader`。默认值：`undefined`。
- `ignoreHeaders`：帮助定义你不希望捕获的请求头列表。将其值设置为 `false` 以捕获所有请求头（设置为 `true` 则不捕获任何请求头）。默认值：`['Cookie', 'Set-Cookie', 'Authorization']`，因此敏感请求头不会被捕获。
- `capturePayload`：存储请求/响应正文的有效负载。这对于故障排除非常有用，但如果你的应用程序大量使用 Fetch/XHR 请求，可能会对网络使用造成较大负担。默认值：`false`。

<Aside type="caution">
	捕获 HTTP 有效负载可能会记录高度敏感的数据（例如密码）或你的应用程序可能在 Fetch/XHR 请求/响应中发送/接收的私人信息，因此请务必使用下面的 `sanitizer` 选项对这些数据进行清理。
</Aside>

- `captureInIframes`：用于禁用 iFrame 内部的网络追踪。默认值：`true`。
- `axiosInstances`：在特定情况下用于追踪自定义的 axios 实例，如果常规配置（proxy api）不起作用的话。默认值：`undefined`。
- `sanitizer`：清理 Fetch/XHR 请求/响应中的敏感数据，或完全忽略该请求。你可以通过修改请求对象然后从函数中返回它来编辑其中的字段（参见下面的示例）。清理在其他选项引发的所有过滤和修改**之后**应用（也就是说，如果 `capturePayload` 未明确设置为 `true`，则参数不会包含 `body` 有效负载）。

```typescript
interface RequestData {
  body: BodyInit | null | undefined; // whatewer you've put in the init.body in fetch(url, init) or xhr.send(body)
  headers: Record<string, string>;
}

interface ResponseData {
  body: string | Object | null;  // Object if response is of JSON type
  headers: Record<string, string>;
}

interface RequestResponseData {
  readonly status: number;
  readonly method: string;
  url: string;
  request: RequestData;
  response: ResponseData;
}

sanitizer: (data: RequestResponseData) => { // sanitise the body or headers
  if (data.url === "/auth") {
    data.request.body = null
  }

  if (data.request.headers['x-auth-token']) { // can also use ignoreHeaders option instead
    data.request.headers['x-auth-token'] = 'SANITISED';
  }

  // Sanitize response
  if (data.status < 400 && data.response.body.token) {
    data.response.body.token = "<TOKEN>"  
  }

  return data
}

// OR

sanitizer: data => { // ignore payloads for calls that start with /secure
  if (data.url.startsWith("/secure")) {
    data.request.body = null;
    data.response.body = null;
  }
  return data
}

// OR

sanitizer: data => { // sanitize request url: replace all numbers
  data.url = data.url.replace(/\d/g, "*")
  return data
}
```

## 示例

以下是有关如何使用 `network` 选项的示例。 

### 使用 NPM

```js
const tracker = new OpenReplay({
  projectKey: PROJECT_KEY,
  network: {
    capturePayload: true // Capture HTTP payload
  }
});
```

### 使用 JavaScript 代码片段

注意用于设置 `network` 选项的 `initOpts` 变量/行。

```js
<!-- OpenReplay Tracking Code -->
<script>
var initOpts = { 
  projectKey: "GxPpaDARdn2345fgt321",

</script>
```
