从 v1.10.0 开始，我们发布了我们的 CLI 实用工具。借助它，管理你自托管版本的 OpenReplay 时最常运行的任务现在只需一个简单的命令即可完成。


## 安装 CLI

每个自托管版本的 OpenReplay 都自带 CLI。

## 使用 CLI

CLI 的语法如下：

```bash 
openreplay [command] <service>
```

可用命令列表如下：

- `-h or --help`：此命令将显示可用的命令和服务列表。
- `-s or --status`：如果你想了解已部署服务的状态，此命令会告诉你需要了解的一切。
- `-u or --upgrade`：用于将你的安装升级到最新的可用版本。
- `-U or --deprecated-upgrade /path/to/old_vars.yaml`
- `-r or --restart`：允许你重启所有服务。这样做可以修复无响应的服务。
- `-R or --Reload`：重新加载某个服务的配置，因此仅在你更改某些内容时使用它。
- `-p or --install-packages`：它会更新所有第三方依赖项（如 `kubectl`、`helm` 等）。
- `-i or --install domain.name.com`：将 OpenReplay 安装到机器上。
- `-e or --edit`：编辑已安装的 OpenReplay 配置并重新加载。
- `-l or --logs SERVICE`：显示某个特定服务的日志。

CLI 可用的所有服务列表如下：

- alerts
- assets
- assist
- chalice
- db
- ender
- frontend
- heuristics
- http
- integrations
- ngix-controller
- peers
- sink
- sourcemapreader
- storage
- image-storage
- video-storage

## 示例

要获取部署状态，请运行以下 CLI 命令：

```bash 
openreplay -s
```
