本页介绍如何使用 OpenReplay 的 iOS 和 Android SDK 对移动应用中的 UI 组件进行脱敏。有关脱敏网络请求的详细信息，请参阅 [网络选项（移动端）](/network-options-mobile)。

## iOS - 对 UI 组件进行脱敏

经过脱敏的视图将在生成的回放中被模糊处理。组件一旦不再可见，便会被自动移除（以节省内存）。

### SwiftUI

将 `.sensitive()` 修饰符添加到你想要脱敏的组件上。

```swift
import ORTracker

Text("Sensitive Information")
    .sensitive()
```

### UIKit

使用 `addIgnoredView` 方法将视图标记为敏感视图。

```swift
import ORTracker

ORTracker.shared.addIgnoredView(view)
```

### React Native

使用 `ORSanitizedView` 组件包裹敏感组件，以在回放中将其模糊处理。

```js
import OR from '@openreplay/react-native';

<OR.ORSanitizedView>
  <SensitiveComponent />
</OR.ORSanitizedView>
```

## Android - 对 UI 组件进行脱敏

经过脱敏的视图会在生成的回放中被模糊处理，并在移出屏幕时自动从内存中移除。

### View

使用 `addIgnoredView` 方法将视图标记为敏感视图。

```kotlin
import com.openreplay.tracker.OpenReplay

OpenReplay.addIgnoredView(view)
```
