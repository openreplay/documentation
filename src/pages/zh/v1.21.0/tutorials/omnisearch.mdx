## 了解事件

OmniSearch 中的**事件**表示会话期间发生的用户交互和系统行为。

它们分为两大类：**自动捕获的事件**（自动跟踪用户交互）和 **DevTools 事件**（技术监控和调试）。

### 自动捕获的事件
| **事件名称**  | **描述** |
| --- | --- |
| Click | 记录用户点击。 |
| Text Input | 跟踪字段中输入的文本。 |
| Visited URL | 记录访问过的页面 |
| Tagged Element | 跟踪自定义标记的元素。 |

### DevTools 事件

|  **事件名称**  | **描述** |
| --- | --- |
| Network Request | 监控网络活动。 |
| GraphQL | 记录 GraphQL 查询和响应。 |
| State Action | 跟踪应用程序中的状态变化。 |
| Error Message | 捕获控制台中的错误消息。 |
| Average CPU Load | 测量平均 CPU 使用率。 |
| Average Memory Usage | 跟踪平均内存消耗。 |

## 了解筛选器

OmniSearch 中的**筛选器**帮助您按特定条件缩小会话回放的范围，让您能够专注于最相关的用户会话。它们与事件配合使用，从而在您的会话数据中创建精确的搜索。

筛选器分为四大类：

### 用户筛选器

|  **筛选器名称**  | **描述** |
| --- | --- |
| User ID | 使用唯一标识符筛选特定用户。 |
| Anonymous User ID | 筛选没有登录身份的用户的会话。 |

### 会话筛选器

|  **筛选器名称**  | **描述** |
| --- | --- |
| Country | 按国家/地区筛选会话。 |
| City | 按城市筛选会话。 |
| State/Province | 按州或省筛选会话。 |
| Operating System (OS) | 按所使用的操作系统筛选会话。 |
| Browser | 按浏览器类型筛选会话。 |
| Device | 按用户的设备筛选会话。 |
| Platform | 按平台类型筛选会话。 |
| Version ID | 按应用程序的版本筛选会话。 |
| Referrer | 根据来源 URL 筛选会话。 |
| Duration | 按会话回放的时长筛选。 |
| UTM Source | 按 UTM source 参数筛选会话。 |
| UTM Medium | 按 UTM medium 参数筛选会话。 |
| UTM Campaign | 按 UTM campaign 参数筛选会话。 |

### 问题筛选器

|  **筛选器名称**  | **描述** |
| --- | --- |
| Rage Click, Dead Click, Excessive Scrolling, Mouse Thrashing | 识别由于重复点击、失效的按钮或链接、过度滚动、不规则的鼠标移动等造成用户挫败感的会话……。 |
| Bad Request | 识别存在 HTTP 400 错误的会话。 |
| Missing Resource | 查找存在缺失资源的会话 |
| CPU Issues | 识别 CPU 使用率高的会话。 |
| Slow Resource | 识别资源加载缓慢的会话。 |
| Slow Page Load | 识别页面加载时间缓慢的会话。 |
| Crash | 识别应用程序或页面崩溃的会话。 |
| Errors | 识别存在应用程序错误的会话。 |

### 元数据筛选器

|  **筛选器名称**  | **描述** |
| --- | --- |
| Metadata key | 使用自定义的元数据筛选会话。如果尚未定义，请参阅[此处](/tutorials/metadata)了解如何添加元数据 |
