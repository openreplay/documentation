从[版本 1.7.0](https://github.com/openreplay/openreplay/releases/tag/v1.7.0) 开始，OpenReplay 允许你将自定义指标添加到你自己的仪表板中。这种灵活性使你能够从你和你的团队所需的任何角度监控你的应用程序（因为仪表板可以在团队成员之间共享）。

同时，作为这个新版本的一部分，你可以创建一个自定义指标，在其中列出会话期间跟踪到的所有 JavaScript 错误。

所以今天我们将探索这个指标。

## 使用 tracker 捕获 JavaScript 错误

JavaScript 错误会被我们的 tracker 自动捕获，因此你实际上不必担心任何事情。

在真实会话期间出现在开发者控制台中的所有异常，都会被跟踪到回放的 “Exceptions” 标签页中。

![JavaScript 错误标签页](/tutorials//js-error-dashboard/js-errors-tab.png)

我们可以使用这些信息围绕它创建一个指标，所以让我们来看看吧。

## 使用你的 JavaScript 错误创建自定义指标

按照以下步骤创建你的自定义指标。指标准备好后，我们将创建一个包含它的自定义仪表板。

1. 转到顶部导航栏中的 Dashboards 部分。
2. 点击左侧菜单中的 “Metrics”。
3. 然后点击 “Create metric”
4. 为你的指标命名，比如 “JavaScript errors”。
5. 在 Metric Type 中，选择 “Table”，然后选择 “JS Errors”

![错误列表](/tutorials//js-error-dashboard/metric-type.png)

或者，你也可以筛选出含有 JS 错误的会话。例如，我们可以将这个指标设为 “**产品页面的 JS 错误列表**”。因此你可以基于 Path 添加一个筛选条件，如下所示：

![按 path 筛选](/tutorials//js-error-dashboard/filter-by-path.png)

对于我们的示例应用程序，产品页面位于路径 `/es/product/[SLUG]` 下，因此将筛选条件设置为 “**以……开头**” `/es/product`，我们将只考虑发生在此页面内的 JS 错误。

此外，为了让仪表板更有趣，你可以创建另一个指标来跟踪同一页面内的 “Rage clicks”。这是一个显示用户在没有明显原因的情况下多次点击同一位置的时刻的指标。

像这样设置的指标即可达到目的：

![Click rage 筛选](/tutorials//js-error-dashboard/click-rage-filter.png)

统计会话，按 URL 筛选，并确保它们内部检测到了 “Click Rage” 类型的问题。

在你设置好筛选条件和指标本身之后，只需点击 “Create” 即可。

### 创建你的仪表板

指标准备好后，你可以点击左侧菜单中的 “**+Create**” 链接（在 Dashboards 部分内）。

![创建仪表板的模态框](/tutorials//js-error-dashboard/create-dashboard.png)

当 “Create Dashboard” 模态框出现后，为它命名，然后：

1. 点击 “Custom” 类别。
2. 选择我们刚刚创建的新指标。
3. 点击左下角的 “Create” 按钮。

这样应该会创建一个向你显示产品页面上发现的问题的仪表板：

![仪表板已就绪](/tutorials//js-error-dashboard/dashboard-ready.png)

如果你需要更多详细信息，可以点击 “**Click rage on products**” 图表，查看存在此问题的会话列表。

如果你点击 “**JS Errors on product”** 列表中的某一行，你将获得有关该问题的额外详细信息：

![JavaScript 错误详情](/tutorials//js-error-dashboard/js-error-details.png)

你会获得错误的 stacktrace，以及有关受其影响的最近会话的额外信息。在这种情况下，我们处理的是一个变量名称中的拼写错误，但它可能是任何 JS 异常。

就是这样，你已成功使用你自己的自定义指标创建了一个新的仪表板。现在继续为你自己的应用程序定义你所需的指标吧。

如果你想阅读有关 OpenReplay 提供的内置指标的更多详细信息，请查看[这里的另一篇教程](https://docs.openreplay.com/tutorials/custom-dashboard)。
