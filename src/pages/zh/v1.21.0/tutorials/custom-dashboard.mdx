OpenReplay 中的仪表盘是你监控、分析和优化产品性能与用户体验的中心枢纽。无论你是在调试问题、跟踪用户行为还是衡量性能，仪表盘都能为你提供做出数据驱动决策所需的洞察。

OpenReplay 仪表盘为团队中的每一位成员汇集了关键洞察：

- **对于开发者：** 实时跟踪 JavaScript 错误、缓慢的网络请求和性能问题等技术问题。
- **对于产品经理：** 分析用户行为、监控转化漏斗，并了解用户在产品中所走过的路径。
- **对于分析网站性能的团队：** 跟踪关键指标，例如访问量最高的页面、流量来源和用户人口统计数据。

## 仪表盘及可用卡片概览

当你创建仪表盘时，它最初是一块空白画布，等待你进行自定义。仪表盘由**卡片**组成，每张卡片显示特定的指标或一组数据。你可以添加、删除或移动卡片，根据需要组织你的仪表盘。

卡片分为三大类：

#### Product analytics

*Product Analytics* 帮助你跟踪和分析用户在产品中的参与度和行为。

| **卡片**    | **描述**                                                   |
|-------------------|------------------------------------------------------------------|
| **Trends**        | 监控会话随时间变化的趋势                                |
| **Funnels**       | 可视化用户在关键步骤中的推进过程，识别流失点以提升转化               |
| **Journeys**      | 了解用户在产品中所走过的路径            |
| **Heatmaps**      | 可视化用户交互的热点区域和模式              |

#### Monitors

*Monitors* 帮助你识别和排查影响用户体验的技术性能问题。

| **卡片**           | **描述**                                                    |
|--------------------------|-------------------------------------------------------------------|
| **JS Errors**            | 跟踪影响用户体验的 JavaScript 错误             |
| **Top Network Requests** | 识别频繁访问的网络资源                    |
| **4xx/5xx Requests**     | 监控客户端和服务器错误，以发现性能问题           |
| **Slow Network Requests**| 定位由响应缓慢造成的延迟                          |

#### Web Analytics

*Web analytics* 提供有关用户人口统计、流量来源和流量模式的洞察。


| **卡片**               | **描述**                                                   |
|------------------------------|------------------------------------------------------------------|
| **Top Pages**               | 识别你网站上访问量最高的页面                      |
| **Top Browsers**            | 查看你的用户最常使用哪些浏览器                        |
| **Top Referrers**           | 了解你的流量来自何处                          |
| **Top Users**               | 找出交互最多的用户                             |
| **Top Countries** | 按地理位置分析你的受众              |
| **Top Devices** | 按设备类型分析你的受众             |



## 创建你的第一个仪表盘

第 1 步：进入 Dashboards，然后点击 **Create dashboard** 按钮。

第 2 步：选择三大类别中的一种。选择你想添加的卡片类型以**从头开始**，或点击 **Add existing Card** 来复用已有的卡片。

第 3 步：点击 **Create Card** 将其添加到仪表盘

第 4 步：要添加另一张卡片，请点击现有卡片旁边的“+”按钮。

## 常见使用场景

仪表盘非常灵活，可适应各种用途。

以下是一些示例：

- **分析用户行为：** 使用 Trends 和 Journeys 来了解用户如何与你的产品交互。
- **识别问题：** 监控 JS Errors、4xx/5xx Requests 和 Slow Network Requests，以解决性能问题。
- **了解流量：** 跟踪 Top Pages、Referrers 和用户人口统计数据，以查看哪些有效以及哪些需要改进。

为特定目标创建单独的仪表盘，例如监控营销活动或跟踪技术性能。仪表盘可以作为系统的不同视图，并提供对当前情况的整合概览。
