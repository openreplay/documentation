OpenReplay 技术栈可以安装在单个实例上，而 Scaleway Elements 正是一个理想的选择。下面介绍具体操作方法。

## 启动实例

1. 进入 Scaleway 控制台
2. 导航到 'Compute > Instances'，然后点击 'Create an instance'
3. 选择你偏好的可用区
4. 选择一个镜像。在本指南中，我们将使用 *Ubuntu Server 20.04 Focal Fossa*
5. 选择你的实例类型。最低配置为 `2 vCPUs, 8 GB of RAM, 50 GB of storage`，且需采用 x86 架构，否则 OpenReplay 后端服务根本无法启动。因此，我们建议**至少**选择 `DEV1-L`（或同等配置），这足以应对低/中等流量。如果你预计会有高流量，则应在此基础上进行扩展。
6. 添加存储卷：将大小设置为至少 50 GB（无论是本地存储还是块存储）
7. 给你的实例起一个好听的名字（例如 openreplay）
8. SSH 密钥：确保你已经有一个与项目关联的密钥，以便可以连接到你的实例
9. 点击 'Create a new instance'

> **注意：** SMTP 端口（25、465、587）默认被 Scaleway 屏蔽。除非你从安全组配置中启用 SMTP，否则你的 OpenReplay 实例将无法发送电子邮件。要启用它，请查看这篇[快速教程](https://www.scaleway.com/en/faq/why-can-i-not-send-any-email/)。

## 部署 OpenReplay

1. 确保你的实例处于 `Started` 状态，然后连接到它：

```bash
## From your terminal
SSH_KEY=~/Downloads/openreplay-key.pem #! wherever you've saved the SSH key
INSTANCE_IP=REPLACE_WITH_INSTANCE_PUBLIC_IP
chmod 400 $SSH_KEY
ssh -i $SSH_KEY root@$INSTANCE_IP
```

2. 通过提供 OpenReplay 将要运行的域名来安装它（例如 DOMAIN_NAME=openreplay.mycompany.com）：

```bash
sudo wget https://raw.githubusercontent.com/openreplay/openreplay/main/scripts/helmcharts/openreplay-cli -O /bin/openreplay 
sudo chmod +x /bin/openreplay
openreplay -i DOMAIN_NAME
```

## 配置 TLS/SSL

OpenReplay 处理敏感的用户数据，因此运行时需要 HTTPS。这是强制要求，否则 tracker 根本不会开始录制。dashboard 也是如此，没有 HTTPS 你将无法回放用户会话。

因此，你必须自带（或生成）你自己的 SSL 证书。

1. 首先，进入 'Network' > 'DNS'（或你的其他 DNS 服务提供商）并创建一条 `A Record`。使用你在安装步骤中先前提供的域名，并通过实例的公网 IP（可在 'Compute' > 'Instances' 中找到）将其指向该实例。

2. 如果你要自带证书，请使用以下命令创建一个 SSL secret：`kubectl create secret tls openreplay-ssl -n app --key="private_key_file.pem" --cert="certificate.crt"`。

> **注意：** 如果你没有证书，请使用 Let's Encrypt 为你的子域名（即安装期间提供的那个）生成一个可自动续期的证书。只需连接到 OpenReplay 实例，运行 `cd /var/lib/openreplay/openreplay/scripts/helmcharts && bash certmanager.sh` 并按照步骤操作即可。

3. 如果你希望启用 http 到 https 的重定向（推荐），请在 `/var/lib/openreplay/vars.yaml` 中 `ingress-nginx` 部分下取消注释以下代码块：
   
```yaml
ingress-nginx: &ingress-nginx
  controller:
    config:
      ssl-redirect: true
      force-ssl-redirect: true
```

值得一提的是，我们的 `ingress-nginx` 默认运行在端口 `80|443` 上，但如有需要，可以在 `vars.yaml` 中轻松更改：

```yaml
ingress-nginx: &ingress-nginx
  controller:
    service:
      ports:
        http: 80
        https: 443
```

4. 最后，重新安装 OpenReplay NGINX：

```bash
openreplay -R
```

现在一切就绪，OpenReplay 应该可以通过你的子域名访问了。你可以访问 `/signup` 页面（例如 openreplay.mycompany.com/signup）来创建账户。
