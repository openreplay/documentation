## 缺少时序信息

当 CORS 生效时，许多时序属性的值会返回为零，除非服务器的访问策略允许共享这些值。这要求提供资源的服务器发送 `Timing-Allow-Origin` HTTP 响应头，其值需指定允许获取受限时间戳值的来源（origin）。

这对于跨域资源尤为重要，例如从与提供主页面不同的域加载的资源。如果未正确设置 `Timing-Allow-Origin` 头，OpenReplay 将无法记录这些资源的时序信息。

例如，要允许 https://application.example.com 在请求位于 https://data.example.com 的后端时查看资源的时序信息，后端应包含以下响应头：

```http
Timing-Allow-Origin: https://application.example.com
```
