OpenReplay 中的数据管理允许你跟踪自定义事件和用户属性，为产品分析提供支持。本页同时介绍用于发送数据的 SDK 集成以及用于管理数据的用户界面。

有关完整的 SDK 文档，请参阅 [Analytics SDK 参考](/sdk/analytics)。

## 事件概览

OpenReplay 捕获两种类型的事件：

- **自动捕获的事件**：自动跟踪的交互，例如页面浏览、点击和表单提交
- **自定义事件**：由你定义的特定领域事件，例如"购买完成"或"功能已启用"

自定义事件让你能够跟踪对业务至关重要的特定用户操作，并在 [趋势](/product-analytics/trends)、[漏斗](/product-analytics/funnels) 和 [旅程](/product-analytics/journeys) 等分析功能中使用它们。

## 捕获事件

要跟踪自定义事件，请使用 SDK 的 analytics 事件模块。有关完整的 API 文档，请参阅 [Events SDK 参考](/sdk/analytics/events)。

### 基础示例

```ts
// Track a simple event
tracker.analytics.events.sendEvent("Button Clicked", {
  button_text: "Start",
  page: "home",
})
```

### 超级属性

超级属性是持久化的键/值对，会自动附加到每一个被跟踪的事件上：

```ts
// Set super properties that apply to all events
tracker.analytics.events.setProperty({
  app_version: "1.42.0",
  env: "prod",
})

// This event will include app_version and env automatically
tracker.analytics.events.sendEvent("Signup Started", {
  plan: "pro",
})
```

## 丰富用户信息

用户属性允许你存储有关用户的属性，用于细分和分析。有关完整的 API 文档，请参阅 [People SDK 参考](/sdk/analytics/people)。

### 识别用户

```ts
// Identify the current user
tracker.analytics.people.identify("user_123")

// Set user properties
tracker.analytics.people.setProperties({
  email: "user@example.com",
  plan: "pro",
})
```

### 属性类型

OpenReplay 支持多种属性类型：

| 类型 | 描述 | 示例 |
|------|-------------|----------|
| **自动跟踪** | 由 SDK 自动捕获 | 设备、浏览器、操作系统、UTM 参数、引荐来源 |
| **特殊** | 具有特定行为的特殊属性 | `$user_id`、`$email`、`$name` |
| **自定义** | 用户定义的属性 | `plan type`、`company`、`signup date` |

## 在用户界面中管理数据

OpenReplay 提供了一个数据管理界面，用于浏览、筛选和组织你的用户、事件及其各自的属性。

### People

<img src="/data-management/people-page.png" alt="事件页面" style={{ width: '700px', height: 'auto' }} />

People 选项卡提供已识别用户的视图：

- 浏览用户及其属性
- 按用户 ID 或属性值搜索
- 查看用户活动和会话历史
- 访问单个用户的个人资料

### Events

<img src="/data-management/events-page.png" alt="事件页面" style={{ width: '700px', height: 'auto' }} />

Events 选项卡显示项目中所有被跟踪的事件类型：

- 按名称浏览和搜索事件
- 按事件类型筛选（自动捕获与自定义）
- 编辑事件元数据（显示名称、描述）
- 控制事件在仪表板和筛选器中的可见性

### Properties

<img src="/data-management/props-page.png" alt="事件页面" style={{ width: '700px', height: 'auto' }} />

Properties 选项卡让你能够同时管理用户属性和事件属性：

- 在用户属性和事件属性之间切换
- 查看属性类型和值分布
- 编辑属性的显示名称
- 控制属性在仪表板和筛选器中的可见性 

### Activity

<img src="/data-management/activity-page.png" alt="事件页面" style={{ width: '700px', height: 'auto' }} />

Activity 选项卡显示事件的实时流：

- 在事件被摄取时进行监控
- 按事件类型或用户筛选
- 验证事件是否被正确捕获

### 兼容 Mixpanel 的别名

为了方便使用，所有方法都支持提供的兼容层：

```ts
tracker.analytics.people.set            // setProperties
tracker.analytics.people.set_once       // setPropertiesOnce
tracker.analytics.people.append         // appendValues
tracker.analytics.people.union          // appendUniqueValues
tracker.analytics.people.incrementBy    // increment
tracker.analytics.events.register       // setProperty
tracker.analytics.events.register_once  // setPropertiesOnce
tracker.analytics.events.unregister     // unsetProperties
tracker.analytics.events.track          // sendEvent
```
