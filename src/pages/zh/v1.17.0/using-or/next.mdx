让 tracker 在 Next.js 应用中运行是相当简单的。由于 Next.js 构建在 React 之上，因此整个流程非常类似，主要的难点来自 Next.js 的服务端渲染（Server Side Rendering）能力。

但是一旦你理解了它们的工作原理以及如何组织你的仅客户端代码，你很快就能开始追踪你的用户了。

## 添加追踪代码

虽然在创建新项目时，你可以直接从平台获取一份可复制粘贴的代码版本，但那份代码过于简陋，可能无法遵循你的团队或公司所期望的最佳实践。

### Tracker 组件

此方法提供了一种极简的做法，我们将 tracker 作为一个组件注入到应用中，它会动态加载该库。
 
#### App Router (v14+) - use client

```jsx
'use client'
import {useEffect} from "react"
import Tracker from '@openreplay/tracker'

const tracker = new Tracker({
  projectKey: "",
  ingestPoint: "",
})

const Openreplay = () => {
  useEffect(() => {
    if (typeof window !== 'undefined') {
      tracker.start()
    }
  }, [])

  return null
}

export default Openreplay 
```

#### 常规组件（Next 直到 v13）

```jsx
[...]

function TrackerComponent() {
const [tracker, setTracker] = React.useState(null);

  React.useEffect(() => {
    (async function () {
      const Tracker = await import("@openreplay/tracker");
      const openRelayTracker = new Tracker.default({
        projectKey: PROJECT_KEY,
        __DISABLE_SECURE_MODE: true,
      });

      setTracker(openRelayTracker);
    })();
  }, []);
[...]
```

### Tracker Context

此方法让你可以创建一个 context，进而更容易地控制 tracker 的状态并调用它的各种 API。

```jsx
import { createContext } from "react"
import {v4 as uuidV4} from 'uuid'
import { useReducer } from "react"

export const TrackerContext = createContext()

function defaultGetUserId() {
   return uuidV4() 
}

async function newTracker(config) {
    // we use dynamic import to make sure that tracker is started inside browser env
    const Tracker = await import("@openreplay/tracker");
    const getUserId = (config?.userIdEnabled && config?.getUserId) ? config.getUserId : defaultGetUserId
    let userId = null;

    const trackerConfig = {
        projectKey: config?.projectKey || process.env.NEXT_PUBLIC_OPENREPLAY_PROJECT_KEY
    }

    const tracker = new Tracker(trackerConfig);
    
    if(config?.userIdEnabled) {
        userId = getUserId()
        tracker.setUserID(userId)
    }
    return tracker

}

function reducer(state, action) {
        switch(action.type) {
            case 'init': {
                if(!state.tracker) {
                    console.log("Instantiaing the tracker for the first time...")
                    return {...state, tracker: newTracker(state.config)}
                }
                return state
            }
            case 'start': {
                console.log("Starting tracker...")
                state.tracker.start()
                return state
            }
        }
    }

export default function TrackerProvider({children, config = {}}) {
    let [state, dispatch] = useReducer(reducer, {tracker: null, config})
    let value = {
        startTracking: () => dispatch({type: 'start'}),
        initTracker: () => dispatch({type: 'init'})
    }

    return <TrackerContext.Provider value={value}>{children}</TrackerContext.Provider>
}
```

这个 provider 向用户提供了 2 个方法：

- `initTracker`：使用提供的配置来实例化 tracker。
- `startTracking`：触发追踪流程的启动。

这个 provider 还接受一个“配置”对象，你可以为将来的功能对其进行扩展。本示例向你展示如何以可选的方式启用唯一识别用户的能力。你可以使用一个默认的 UUID，它会在用户每次访问页面时为每位用户生成一个唯一值。你也可以在配置中提供一个自定义的 `getUserId` 函数。 

借助你的自定义函数，你可以提供你商店的客户 ID、用户的电子邮箱地址，或任何能在你的应用中唯一识别用户的内容。这样，你就可以跨时间追踪他们的会话回放，以防他们遇到其他人没有报告的特定问题。

💡**给 self-hosted 用户的说明**：如果你使用的是 OpenReplay 的 self-hosted 版本，你还需要指定 `ingestPoint` 配置属性。该属性指定了 tracker 数据的接收端点。Cloud 用户不需要这一项，因为默认情况下 tracker 已经知道该端点的 SaaS 版本在哪里，但如果你是自托管的，你就需要指定它（它应该类似于 `https://openreplay.mydomain.com/ingest`） 

### 在代码中处理 "projectKey"

需要说明的是，你所配置的 `projectKey` 是由 OpenReplay 平台提供的。出于安全原因，该值不应被硬编码地存储在你的代码中，而应存放在某种配置文件/系统中。

在本示例中，我利用了 Next 的 `.env` 文件。你需要理解的是，如果你不给环境变量加上 `NEXT_PUBLIC` 前缀，你的客户端代码（即 tracker）将无法访问它。 

如果这无法接受，你可以在用户访问的第一个页面中使用 `getStaticProps`，在实例化 tracker 之前获取该值。下面你可以看到这两个示例。

## 使用 tracker context

像下面这样，导入并用我们之前创建的 `TrackerProvider` 包裹你的应用：

```jsx
import '../styles/globals.css'
import TrackerProvider from '../context/trackerContext'

function MyApp({ Component, pageProps }) {

  return <TrackerProvider>
        <Component {...pageProps} />
    </TrackerProvider>
}

export default MyApp
```

这是 `_app.js` 文件，它充当整个应用的入口点。  

然后，你可以从任何你希望的组件中触发追踪，如下所示：

```jsx
import { useContext, useEffect } from 'react'
import { TrackerContext } from '../context/trackerContext'
import styles from '../styles/Home.module.css'

export default function YourComponent({projectKey}) {
    
    const {initTracker, startTracking} = useContext(TrackerContext)
    
    useEffect( () => {
       initTracker()
       startTracking()
    }, [])
//.... rest of your code
}
```

如果你不需要这么多的控制，你可以将两者——“init 消息”和“start tracking 消息”——合并起来，这样你只需调用一个函数即可启动追踪。
