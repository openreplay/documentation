让追踪器在 React 应用中运行非常简单。 

## 添加追踪代码

虽然你可以[在创建新项目时直接从平台获取一份可复制粘贴的代码](/using-or/snippet#getting-the-snippet)，但那段代码过于简陋，会让你在定制和添加额外插件时困难得多。

假设你正在构建一个 SPA（单页应用，Single Page Application），你只需要将以下代码添加到你的入口文件中：

```jsx
import Tracker from '@openreplay/tracker';

const tracker = new Tracker({
  projectKey: "<your project key>",  
});

function YourMainComponent({props}) {
    //some code here...

    useEffect(() => {
        tracker.start();
    }, [])

    return //....
}
```

换句话说，你可以在脚本开始时实例化追踪器，并在页面加载后使用 `useEffect` 钩子来启动它。

<Aside type="tip">
**给自托管（self-hosted）用户的提示**：如果你使用的是 OpenReplay 的自托管版本，你还需要指定 `ingestPoint` 配置属性。 
该属性指定了追踪器数据的摄取端点（ingestion endpoint）。云端用户不需要它， 
因为默认情况下追踪器会知道此端点的 SaaS 版本在哪里；但如果你自行托管（self-hosting），就需要指定它 
（它应该类似于 `https://openreplay.mydomain.com/ingest`） 
</Aside>

### 在代码中处理 "projectKey"

需要注意的是，你配置的 `projectKey` 是由 OpenReplay 平台提供的。出于安全原因，这个值不应硬编码（hardcoded）存储在你的代码中，而应存放在某种配置文件/系统中。

针对这个问题有多种解决方案；根据你的具体配置，你可以使用类似 [`create-react-app` 对环境变量（ENV）的支持](https://create-react-app.dev/docs/adding-custom-environment-variables/) 的方式。

## 正在构建更复杂的应用？
如果你正在用 React 构建一个复杂的应用，你很可能正在使用 [Next](/using-or/next)、[Remix](/using-or/remix) 或类似的框架。请务必查看我们的[安装设置部分](/using-or)，找到你正在使用的框架。
