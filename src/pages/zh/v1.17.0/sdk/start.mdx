此方法用于启动 tracker。启动 tracker 后，会话录制随即开始。

## 签名

```js
start(startOpts?: Partial<StartOptions>): Promise<StartPromiseReturn>
```

### 参数
`start` 方法接受一个可选参数，你可以借此自定义录制的各个方面：
- `userID: string`：用于手动设置用户 ID，以便跨会话追踪该用户。这是一个字符串值，可以是你想要的任意内容。
- `metadata: Record<string, string>`：手动设置元数据值。请查看[元数据部分](/installation/metadata/)以了解有关此功能的更多信息。
- `forceNew: boolean`：用于在页面刷新后强制开启一个新会话。默认值为 `false`，因此刷新后会话会被保留。
- `sessionHash: string`：用于粘性会话。如果你有一个多站点应用，或者需要将用户重定向到站点外部然后再返回站点（例如跳转到支付网关），此参数会很有用。

### 返回值

启动后，该方法将返回一个 promise，其中包含一个会话信息对象，内容如下：
- `sessionID: string`：表示已启动会话 ID 的字符串值。
启动 tracker（以及录制过程），然后返回一个包含会话信息（sessionID、sessionHash、userUUID）的 promise。
