此方法允许你上报一个“功能性事件”。通过功能性事件，你可以标记用户会话中那些从功能角度来看具有相关性的时刻——这些时刻默认情况下我们的追踪器无法自行捕捉并标记出来。功能性事件的一个示例是用户将某个商品添加到购物车。虽然我们的追踪器能够记录该操作，但它无法知道要将其标记为你会话时间轴中的一个相关事件。然而，借助此方法，你可以手动完成这一标记。

<Aside type="tip">
  我们还提供了一份关于如何发送自定义事件的详细教程，你可以在[这里](/tutorials/custom-events)查看。
</Aside>

更多详情请参阅[自定义事件](/installation/custom-events#functionalevents)。

## 签名

```js 
event(key: string, payload?: any, issue?: boolean): void
```

## 参数

- `key: string`：用于保存信息的键名。该键可在 OpenReplay 平台内用于过滤和搜索。
- `payload: any`：事件的内容。可以是任何可序列化的数据。
- `issue: boolean` ：指示这是否为功能性事件（如果设置为 `false`），还是“技术性事件”（如果设置为 `true`）。其默认值为 `false`。
