Pinia 是 Vue 项目中用于替代旧方案的状态管理库。过去你使用 VueX 的地方，现在将改用 Pinia。借助我们的 [Pinia 插件](/plugins/pinia)，你可以在应用的整个生命周期中跟踪状态的变化。

## 安装插件

可以使用以下命令轻松安装该插件：

```jsx
npm i @openreplay/tracker-vuex
```

## 在你的 Vue 应用中使用该插件

在本示例中，我们将使用一个示例 Vue 应用来展示如何在真实应用中添加该插件并与之交互。

**注意**：本示例的完整源代码位于这个 [GitHub 仓库](https://github.com/deleteman/openreplay-vue-pinia-example) 中，如果你需要更多细节，可以去查看一下。

安装插件后，我们将通过添加下面的导入语句来修改 `main.js` 文件：

```jsx
import trackerVuex from '@openreplay/tracker-vuex/cjs';
```

然后，我们将使用 `startTracker` 函数来实例化 tracker，并一次性使用该插件。

调用此函数的结果将是一个对象，其中包含对该插件使用 `use` 方法的结果。

**请注意**，解构出的属性名与 `name` 属性的值相同。只要你保持这两者一致，就不会有问题。

```jsx
const {piniaPlugin} = startTracker({
    projectKey: "<your project key>",
    plugins: [{
        name: 'piniaPlugin',
        fn: trackerVuex
    }]
})
```

接下来要做的是创建一个具名的 store 包装器，稍后我们将用它在所需的 store 上设置插件。

```jsx
const piniaStoreWrapper = piniaPlugin("products")
saveStore(piniaStoreWrapper)
```

`piniaStoreWrapper` 稍后需要与我们决定跟踪的 store 一起使用。然而，此时 store 尚未创建，因此我们将通过 `saveStore` 函数将其保存起来。

### 实用函数

在继续之前，让我们来看一下 `saveStore` 和 `startTracker` 函数。

第一个函数是 `storesManager` 模块的一部分，它看起来像这样：

```jsx
let storeWrapper = null;

export function saveStore(s) {
    storeWrapper = s;
}

export function getStore(store) {
    return storeWrapper;
}
```

这个模块可以让我们在其他地方使用之前创建的 `piniaStoreWrapper`。

至于 `startTracker` 函数，代码看起来像这样：

```jsx
export function startTracker(config) {
    console.log("Starting tracker...")

    console.log("Project key used: ", config.projectKey)
    const trackerConfig = {
        projectKey: config.projectKey,
         //ingestPoint: config.ingestPoint,
        __DISABLE_SECURE_MODE: true
    }

    const tracker = new Tracker(trackerConfig);

    const pluginReturns = {}
    config?.plugins.forEach( p => {
        console.log("Using plugin...", p.name)
        pluginReturns[p.name] = tracker.use(p.fn())
    })

    tracker.start();
    return {
        tracker,
        ...pluginReturns
    }
}
```

这个函数会使用我们提供的配置来实例化 tracker。请注意我把 `ingestPoint` 选项注释掉了，这是因为我们直接使用的是 SaaS 版本；但如果你想在自托管版本中使用这段代码，就必须指定它，它应该类似于这样：`<your-instance>.yourhost/ingest`

### 跟踪你的 Pinia store

当所有设置代码都准备好后，前往你使用 store 的地方；以我们的示例为例，让我们编辑 `App.vue` 文件以调用 `getStore` 函数，它会返回我们之前保存的 store 包装器。

然后我们将用它来包装我们想要跟踪的 store：

```jsx
<template>
  <!-- template code -->
</template>

<script lang="ts">
import { computed, defineComponent, onMounted, ref } from 'vue'
import { Product, useProductStore } from './store/products'
import { useCartStore } from './store/cart'
import {getStore} from './store/storesManager.js'

export default defineComponent({
  setup() {

    const productStore = useProductStore()
    const loading = ref(true)
    //we get the wrapper here
	  const storeWrapper = getStore()

    onMounted(async () => {
      await productStore.fetchAll()
      loading.value = false
    })

    const cartStore = useCartStore()

		//we wrap the cart store here
    storeWrapper(cartStore)
    return {
      products: computed(() => productStore.list),
      cart: computed(() => cartStore.formattedCart),
      total: computed(() => cartStore.total),
      loading,
      add: (product: Product) => cartStore.add(product),
      remove: (product: Product) => cartStore.remove(product),
    }
  }
})

</script>
```
