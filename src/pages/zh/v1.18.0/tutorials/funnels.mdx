查找问题有时并不像查看单个回放那么简单。有时问题出现在用户经历一系列步骤的过程中。

当这些步骤将用户引导至期望的结果（例如购买你的某个产品，或订阅你的新闻通讯）时，它们被称为“漏斗”。

OpenReplay 允许你分析和跟踪自定义漏斗，以了解你的用户为什么会从中流失而没有完成转化。

在本教程中，我们将了解如何设置一个漏斗，以弄清楚我们的电商网站在用户从主页到将产品加入购物车的过程中流失了多少用户。

## 创建你的第一个漏斗

**请注意**，捕获所需数据不需要任何特殊的插件或附加组件。只需基本的 tracker 设置，你就拥有所需的一切。

漏斗是一种你可以在顶部导航的 Dashboards 部分中创建的指标：

![找到指标部分](/tutorials/funnels/getting-to-metrics.png)

进入此部分后，点击 “Create Metric” 并选择 “Funnel” 指标类型。

然后使用下方的筛选条件开始添加步骤。以下是我们的漏斗如何跟踪用户从其他主页一路到将产品加入购物车的过程：

![用于创建漏斗的筛选条件](/tutorials/funnels/funnel-filters.png)

注意 “Series 1” 筛选条件中的事件：

1. 首先，确保路径为 “/es”，它对应于我们网站的主页。
2. 然后检查路径是否以 “/es/product” 开头，它对应于我们网站上的每一个产品。
3. 最后，获取那些点击 “Add to Cart” 按钮的会话。

由于我们定义的是一个漏斗，每个新的筛选条件都会更加严格。

在筛选条件下方，你会看到每个步骤的结果，如下所示：

![漏斗的可视化呈现](/tutorials/funnels/funnel-visual.png)

你可以看到，我们在从第 2 步到第 3 步的过程中流失了 5 个用户。

而在所有进入第 3 步的用户中，只有 50% 的人确实点击了按钮。那么，是什么导致了转化率的这种突然下降呢？我们在漏斗下方有一些线索。

![漏斗内的错误](/tutorials/funnels/funnel-errors.png)

我们遇到了高 CPU 问题、无效点击，甚至还有一些自定义事件，它们或许能为我们提供一些关于正在发生什么的线索。

这些问题提供了它们在漏斗内所产生影响的一些洞察，因此你可以查看并专注于最严重的那些。

如果你点击其中一个错误，你会得到漏斗内受其影响的回放列表，这样你就可以分析正确的回放：

![漏斗错误内的详细信息](/tutorials/funnels/funnel-details.png)

现在你可以将该指标添加到一个或多个 dashboard 中来使用它。

## 将你的指标添加到 dashboard

要将你全新的指标添加到 dashboard，请前往 Dashboards 部分，选择你想要处理的那个，然后点击 “Add Metric”。


![添加指标](/tutorials/funnels/add-metric-button.png)
从 Custom 部分选择新的指标，然后点击 “Add Selected to Dashboard”：

![将指标添加到 dashboard](/tutorials/funnels/adding-metric-to-dashboard.png)
