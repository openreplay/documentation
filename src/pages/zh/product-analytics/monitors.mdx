OpenReplay 中的**监控器**提供了一个聚焦视图，用于关注影响应用稳定性和性能的关键问题。借助预配置的卡片，你可以监控 JavaScript 错误、网络请求以及其他关键指标，从而快速识别并解决问题。

## 你能从监控器中了解到什么？

借助**监控器**，你可以解答以下问题：

- 哪些 JavaScript 错误正在影响用户体验？
- 你的应用中最频繁的网络请求是哪些？
- 是否存在反复出现的客户端（4xx）或服务端（5xx）网络请求？
- 哪些网络请求导致了最显著的延迟？

## 设置你的监控器仪表盘

创建一个专门的监控器仪表盘有助于将关键指标集中到一处，让你更轻松地掌握技术问题。具体方法如下：

#### 1. **创建仪表盘**：
    1. 进入 **Dashboards** 菜单。
    2. 点击 **Add Dashboard** 并将其命名为 **Monitors**。

#### 2. **添加卡片**：
    1. 进入 **Monitors** 类别。
    2. 将以下每张卡片添加到你的仪表盘中：
        - **Table of JS Errors：** 监控影响用户体验的 JavaScript 错误，并快速识别错误类型、出现频率及受影响的用户。
        - **Top Network Requests：** 查看最频繁的网络请求，并识别高流量的 API 端点。
        - **Sessions with 4xx/5xx Requests：** 跟踪客户端（4xx）和服务端（5xx）错误。
        - **Sessions with Slow Network Requests：** 定位导致延迟的网络请求，并优先修复缓慢的 API 或过载的服务。

## 深入挖掘监控器的洞察

监控器仪表盘上的每张卡片都是可交互的，让你能够更详细地探究问题：

### **Table of JS Errors**：

1. 点击某个错误以查看其 **stack trace**。
2. 使用 **‘Find all sessions with this error’** 按钮定位所有受影响的会话。
3. 通过**概览**分析趋势，它会显示过去 24 小时和 30 天内的出现情况。


### Sessions with 4xx/5xx and Slow Network Requests：

1. 在图表上选择一个点，以查看相关的会话回放。
2. 在会话回放中打开 **Network** 标签页，以分析请求的详细信息，包括持续时间和状态。
3. 使用下拉菜单调整图表的时间范围，以聚焦于特定时间段。


### **Top Network Requests**：

- 查看应用中最频繁的请求。
- 点击某个具体请求，以筛选包含该请求的会话并做进一步分析。

### **调整时间范围**

仪表盘的默认时间范围为**过去 24 小时**，但你可以根据需要进行更改：

1. 点击仪表盘顶部的**时间范围下拉菜单**。
2. 选择一个预设范围（例如 "Last 7 days"）或设置自定义时间段。

## **创建卡片的替代方式**

如果你愿意，也可以直接从 **Cards View** 创建并添加卡片：

1. 在 Analytics 模块中进入 **Cards View**。
2. 为任意监控器指标（例如 JS 错误或缓慢的网络请求）创建一张卡片。
3. 使用 **Add to Dashboard** 选项将该卡片添加到你的监控器仪表盘。

## **充分利用**监控器

**监控器**让你能够跟踪并处理关键问题，例如 JavaScript 错误、网络性能和缓慢的请求。利用其交互功能、会话回放和 Network 标签页深入剖析问题、找出根本原因，并提升应用的稳定性和性能。
