在 React 应用程序上使 tracker 工作相当简单。 

## 添加跟踪代码

虽然您可以[在创建新项目时直接从平台获取复制粘贴版本的代码](/using-or/snippet#getting-the-snippet)，但该代码过于基础，会使自定义和添加额外插件变得更加困难。

假设您正在构建一个 SPA（单页应用程序），您只需要将以下代码添加到您的入口点：

```jsx
import Tracker from '@openreplay/tracker';

const tracker = new Tracker({
  projectKey: "<your project key>",  
});

function YourMainComponent({props}) {
    //some code here...

    useEffect(() => {
        tracker.start();
    }, [])

    return //....
}
```

换句话说，您可以在脚本开始时实例化 tracker，一旦页面加载，您就可以使用 `useEffect` 钩子启动它。

<Aside type="tip">
**给自托管用户的说明**：如果您正在使用 OpenReplay 的自托管版本，您还需要指定 `ingestPoint` 配置属性。 
此属性指定 tracker 数据的摄取端点。云用户不需要这个， 
因为默认情况下 tracker 会知道该端点的 SaaS 版本在哪里，但如果您是自托管的，您就需要指定它 
（它应该是像 `https://openreplay.mydomain.com/ingest` 这样的东西） 
</Aside>

### 在代码中处理 "projectKey"

需要说明的是，您配置的 `projectKey` 是 OpenReplay 平台提供的。出于安全原因，此值不应在代码中硬编码存储，而应存放在某种配置文件/系统中。

有多种解决此问题的方案；根据您的设置，您可以使用类似 [`create-react-app` 对 ENV 变量的支持](https://create-react-app.dev/docs/adding-custom-environment-variables/) 的方式。

## 正在构建更复杂的东西？
如果您正在用 React 构建一个复杂的应用程序，很可能您正在使用 [Next](/using-or/next)、[Remix](/using-or/remix) 或类似的框架。请务必查看我们的[设置部分](/using-or)以找到您正在使用的框架。
