错误报告对于监控应用的稳定性、提前发现回归问题以及了解哪些问题对用户来说发生得最频繁都非常重要。默认情况下，任何未捕获的异常都会记录在 'Errors' 标签页下：

![Errors Tab](/static/errors-tab.jpg#center)

此外，记录的错误还会出现在会话回放中 DevTools 的 'Console' 标签页下：

![Exceptions Tab](/static/exceptions-tab-in-recording.jpg#center)

## 手动记录异常

OpenReplay 跟踪器支持记录 3 种类型的错误：
* 已捕获的异常 - [Error](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Error)
* 被拒绝的 Promise - [PromiseRejectionEvent](https://developer.mozilla.org/en-US/docs/Web/API/PromiseRejectionEvent)
* 错误事件 (onError) - [ErrorEvent](https://developer.mozilla.org/en-US/docs/Web/API/ErrorEvent)

以上所有类型都可以使用单个方法进行报告：

```js
tracker.handleError(error, metaObject); // metaObject is optional and is a flat object
```

让我们来看一下每种错误类型的示例：

### 已捕获的异常 (Error)

默认情况下，已捕获的异常不会记录到 'Console' 标签页。为了报告错误，我们可以这样做：

```js
try {
    // application code
}
catch(err) {
    // application code that handles the error
    tracker.handleError(error, metaObject); // metaObject is optional and is a flat object
}
```
### 被拒绝的 Promise (PromiseRejectionEvent)

如果某个 Promise 被拒绝，而我们希望将该错误报告给 OpenReplay，可以这样做：

```js
function myFunc() {
    doSomeAsyncStuff()
    .then((result) => {
        // application code
    })
    .catch(promiseRejectionErr => {
        // application code to handle the error
        tracker.handleError(promiseRejectionErr, metaObject); // metaObject is optional and is a flat object     
    })
}
```

### onError (ErrorEvent)

此示例将展示如何在 ```<img>``` 元素未能加载时报告错误：  

```html
<img id="myImg" src="image.gif">
<p id="demo"></p>
```

```js
document.getElementById("myImg").addEventListener("error", myFunction);

function myFunction(errorEvent) {
  document.getElementById("demo").innerHTML = "The image could not be loaded.";
  tracker.handleError(errorEvent, { context: "demo" });
}
```
