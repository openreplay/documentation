## Información de tiempos faltante

Cuando CORS está en vigor, muchos de los valores de las propiedades de tiempo se devuelven como cero, a menos que la política de acceso del servidor permita compartir estos valores. Esto requiere que el servidor que proporciona el recurso envíe el encabezado de respuesta HTTP `Timing-Allow-Origin` con un valor que especifique el origen u orígenes a los que se les permite obtener los valores de marca de tiempo restringidos.

Esto es especialmente importante para los recursos de origen cruzado (cross-origin), como los que se cargan desde un dominio distinto al que sirve la página principal. Si el encabezado `Timing-Allow-Origin` no está configurado correctamente, OpenReplay no podrá registrar la información de tiempos de estos recursos.

Por ejemplo, para permitir que https://application.example.com vea la información de tiempos de los recursos al solicitar el backend en https://data.example.com, el backend debe incluir los siguientes encabezados:

```http
Timing-Allow-Origin: https://application.example.com
```
