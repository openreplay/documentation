Este método parchea las APIs de XHR y Fetch para interceptar las solicitudes de red.

## Firma

```js
import OR from '@openreplay/react-native'

OR.tracker.patchNetwork(
    global,
		(url) => url.includes('openreplaydomain'),
    { mode: 'fetch' }
)
```

## Parámetros

- `context: globalThis`: contexto actual de la app (para obtener fetch/xhr)
- `isServiceURL: (url: string) => boolean`: función que comprueba si la solicitud actual debe rastrearse (false) o ignorarse (true)
- `options: Partial<Options>`: objeto de opciones

## Opciones

- `ignoreHeaders: Array<string> | boolean`: encabezados a ignorar (predeterminado: `['cookie', 'set-cookie', 'authorization']`)
- `capturePayload: boolean`: si se debe capturar el payload de la solicitud (predeterminado: `false`)
- `sanitizer: data: RequestResponseData) => RequestResponseData`: función que sanea la solicitud/respuesta (predeterminado: `undefined`)
- `mode: 'fetch' | 'xhr' | 'all'`: qué API parchear (predeterminado: `'fetch'`)

> Nota: las solicitudes fetch de React-Native usan XHR por debajo (en el momento de escribir esto), por lo que usar "all" puede provocar solicitudes duplicadas en la grabación
