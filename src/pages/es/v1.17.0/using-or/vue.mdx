Poner en marcha el tracker en una aplicación Vue es bastante sencillo. 

## Añadir el código de seguimiento

Aunque puedes [obtener una versión para copiar y pegar del código directamente desde la plataforma cuando creas un nuevo proyecto](/using-or/snippet#getting-the-snippet), ese código es demasiado básico y hará que sea mucho más difícil personalizarlo y añadir complementos adicionales.

Suponiendo que estás creando una SPA (Single Page Application), todo lo que tienes que hacer es añadir el siguiente código a tu script de setup en tu archivo principal.

```html
<script setup>
//... your code here
const tracker = new Tracker({
  projectKey: "<your project key>"
});

tracker.start();
//... more code here
</script>
```


<Aside type="tip">
**Una nota para los usuarios self-hosted**: si estás usando la versión self-hosted de OpenReplay, también tendrás que especificar la propiedad de configuración `ingestPoint`. 
Esta propiedad especifica el endpoint de ingesta para los datos del tracker. Los usuarios de la nube no la necesitan, 
porque por defecto el tracker sabrá dónde se encuentra la versión SaaS de este endpoint, pero si lo alojas tú mismo, tendrás que especificarlo 
(debería ser algo como `https://openreplay.mydomain.com/ingest`) 
</Aside>

### Manejo del "projectKey" en tu código

A modo de nota, el `projectKey` que configuras es el que proporciona la plataforma de OpenReplay. 
Por razones de seguridad, este valor no debería almacenarse codificado de forma fija en tu código, sino más bien en algún tipo de archivo/sistema de configuración.

Una opción sería añadir una variable de entorno en un archivo `.env` en la raíz de tu proyecto y prefijar la variable con `VITE_`, lo que la hará disponible en tu código Vue.
Luego puedes usarla mediante `import.meta.env.<your variable name>`.
Así:

```javascript
const tracker = new Tracker({
  projectKey: import.meta.env.VITE_VUE_APP_PROJECT_KEY,
});
 ```

## ¿Estás creando algo más complejo?
Si estás creando una aplicación compleja con Vue, lo más probable es que estés usando [Nuxt](/using-or/nuxt) o algo similar. Asegúrate de consultar nuestra [sección de Configuración](/using-or) para encontrar el framework con el que estás trabajando.
