Poner en funcionamiento el rastreador en una aplicación React es bastante sencillo. 

## Añadir el código de seguimiento

Aunque puedes [obtener una versión para copiar y pegar del código directamente desde la plataforma cuando creas un nuevo proyecto](/using-or/snippet#getting-the-snippet), ese código es demasiado básico y hará que sea mucho más difícil personalizarlo y añadir plugins adicionales.

Suponiendo que estás construyendo una SPA (Single Page Application), todo lo que tienes que hacer es añadir el siguiente código a tu punto de entrada:

```jsx
import Tracker from '@openreplay/tracker';

const tracker = new Tracker({
  projectKey: "<your project key>",  
});

function YourMainComponent({props}) {
    //some code here...

    useEffect(() => {
        tracker.start();
    }, [])

    return //....
}
```

En otras palabras, puedes instanciar el rastreador al inicio del script y, una vez que la página cargue, puedes iniciarlo usando un hook `useEffect`.

<Aside type="tip">
**Una nota para usuarios self-hosted**: si estás usando la versión self-hosted de OpenReplay, también tendrás que especificar la propiedad de configuración `ingestPoint`. 
Esta propiedad especifica el endpoint de ingesta de los datos del rastreador. Los usuarios de la nube no la necesitan, 
porque por defecto el rastreador ya sabrá dónde está la versión SaaS de este endpoint, pero si lo estás alojando por tu cuenta (self-hosting), tendrás que especificarlo 
(debería ser algo como `https://openreplay.mydomain.com/ingest`) 
</Aside>

### Gestionar la "projectKey" en tu código

Como nota, la `projectKey` que configuras es la que proporciona la plataforma de OpenReplay. Por razones de seguridad, este valor no debe almacenarse codificado de forma fija (hardcoded) en tu código, sino más bien en algún tipo de archivo/sistema de configuración.

Existen diferentes opciones para resolver este problema; dependiendo de tu configuración, podrías usar algo como el [soporte de `create-react-app` para variables de entorno (ENV)](https://create-react-app.dev/docs/adding-custom-environment-variables/).
