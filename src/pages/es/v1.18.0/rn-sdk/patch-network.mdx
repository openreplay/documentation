Este método parchea las APIs de XHR y Fetch para interceptar las solicitudes de red.

## Firma

```js
import OR from '@openreplay/react-native'

OR.patchNetwork(
    global,
		(url) => url.includes('openreplaydomain'),
    { mode: 'fetch' }
)
```

## Parámetros

- `context: globalThis`: contexto actual de la aplicación (para obtener fetch/xhr de él)
- `isServiceURL: (url: string) => boolean`: función que comprueba si la solicitud actual debe ser rastreada (false) u omitida (true)
- `options: Partial<Options>`: objeto de opciones

## Opciones

- `ignoreHeaders: Array<string> | boolean`: encabezados a ignorar (por defecto: `['cookie', 'set-cookie', 'authorization']`)
- `capturePayload: boolean`: si se debe capturar la carga útil de la solicitud (por defecto: `false`)
- `sanitizer: data: RequestResponseData) => RequestResponseData`: función que sanea la solicitud/respuesta (por defecto: `undefined`)
- `mode: 'fetch' | 'xhr' | 'all'`: qué API parchear (por defecto: `'fetch'`)

> Nota: las peticiones fetch de React-Native utilizan XHR por debajo (en el momento de escribir esto), por lo que usar "all" puede provocar solicitudes duplicadas en la grabación
