## Instalación

La siguiente línea instalará el tracker y, con él, el SDK para que puedas aprovechar todas las funciones del tracker.

```bash
npm i @openreplay/react-native
```

## Inicialización

Al instanciar el tracker de OpenReplay, hay varias opciones de configuración que puedes proporcionar para personalizar muchos aspectos de la grabación y la experiencia de grabación. Debes establecer la opción `projectKey` en el constructor. Puedes obtener este valor en tu panel de OpenReplay en 'Preferences > Projects'.

<Aside type="tip">
	Ten en cuenta que si tienes varios proyectos, debes tener cuidado con qué clave de proyecto eliges.
</Aside>


### Añadir el código de seguimiento

Añade el siguiente código a tu archivo raíz:

```js
import Openreplay from '@openreplay/react-native';

// ...
useEffect(() => {
	OpenReplay.tracker.startSession(
		'yourProjectKey',
		options, // explained below, set {} if empty
		'https://local.openreplay.instance/ingest' // not required if you're using managed version
	);
}, [])
```

### Sanear datos

Consulta [cómo sanear datos en las repeticiones de React Native](/rn-sdk/sanitization) para más detalles.

## Opciones de inicialización

De forma predeterminada, todas las opciones están marcadas como `true`:

- `crashes: Bool` Habilita crashlytics.
- `analytics: Bool` Habilita el seguimiento analítico de las vistas marcadas.
- `performances: Bool` Habilita el listener de rendimiento.
- `logs: Bool` Habilita el listener de logs.
- `screen: Bool` Habilita el grabador de pantalla.
