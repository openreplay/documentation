Si la información que registra nuestro tracker de forma predeterminada cuando ocurre un error de JS no es suficiente, puedes utilizar este método. El método `handleError` reporta manualmente excepciones capturadas, promesas rechazadas o eventos de error. Consulta [reporte de errores](/installation/error-reporting#manuallyloggingexceptions) para más detalles.

## Firma

```js 
handleError: (e: Error | ErrorEvent | PromiseRejectionEvent, metadata?: Record<string, any>) => void
```

## Parámetros

- `e: Error | ErrorEvent | PromiseRejectionEvent`: El objeto de error que se está capturando.
- `metadata: Record<string, any>`: **[Opcional]** Una estructura tipo diccionario con toda la información adicional que quieras guardar asociada a este error.

## Valor de retorno

Este método no devuelve ningún valor; la información guardada puede verse dentro de la plataforma de OpenReplay.
