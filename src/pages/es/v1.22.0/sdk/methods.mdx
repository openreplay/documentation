## Métodos

- [Constructor](/sdk/constructor): Conoce todas las opciones de inicialización del tracker.
- [event()](/sdk/methodsevent): Envía eventos personalizados para que se registren como parte de tus repeticiones con este método.
- [getSessionToken()](/sdk/methods/get-session-token): Obtén el token de la sesión activa actual.
- [getSessionID()](/sdk/methods/get-session-id): Obtén el ID de la sesión activa actual.
- [getSessionURL()](/sdk/methods/get-session-url): Obtén la URL para reproducir la sesión actual. 
- [handleError()](/sdk/methods/handle-error): Captura información de errores personalizados y guárdala como parte de tu grabación.
- [isActive()](/sdk/methods/is-active): Obtén el estado actual de la sesión activa.
- [issue()](/sdk/methods/issue): Captura eventos técnicos personalizados y guárdalos como parte de la repetición.
- [setUserID()](/sdk/methods/set-user-id): Identifica a tus usuarios con este método.
- [setMetadata()](/sdk/methods/set-metadata): Establece un campo de metadatos y guárdalo como parte de la grabación.
- [start()](/sdk/methods/start): Inicia la grabación de una sesión.
- [stop()](/sdk/methods/stop): Detén la grabación de una sesión con este método.
- [clearPersistFlag()](/sdk/methods/clear-persist-flag): Elimina las Feature Flags persistentes del sessionStorage del navegador.
- [getFeatureFlag()](/sdk/methods/get-feature-flag): Devuelve un `IFeatureFlag` por clave si existe.
- [onFlagsReload()](/sdk/methods/on-flags-reload): Establece el callback que se utilizará cuando se carguen las Feature Flags.
- [isFlagEnabled()](/sdk/methods/is-flag-enabled): Devuelve `true` si la flag con esta clave existe y está activa.
- [reloadFlags()](/sdk/methods/reload-flags): Recarga todas las flags (puede ser útil si la sesión del usuario tuvo algún cambio en los metadatos, el userId o cualquier otra información que pudiera ser requerida para ciertas condiciones de las flags).
- [getAllFeatureFlags()](/sdk/methods/get-all-feature-flags): Devuelve todas las flags (`IFeatureFlag[]`) que están activas en este momento.
- [coldStart()](/sdk/methods/cold-start): Comienza a almacenar mensajes en el búfer (actualizando el búfer) con la posibilidad de llamar a `start()` para habilitar la grabación de la sesión y enviar el contenido del búfer.
- [startOfflineRecording()](/sdk/methods/start-offline-recording): Inicia la grabación de sesión sin conexión que puede enviarse con `uploadOfflineRecording`.
- [uploadOfflineRecording()](/sdk/methods/upload-offline-recording): Sube el búfer de sesión almacenado al backend.
- [forceFlushBatch()](/sdk/methods/force-flush-batch): Fuerza el envío del lote actual de mensajes sin esperar a que se complete.
- [trackWs()](/sdk/methods/track-ws): Crea un hook para capturar mensajes de WebSocket.
