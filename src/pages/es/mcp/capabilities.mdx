Una vez que hayas [configurado](/mcp/setup) la app MCP e iniciado sesión, puedes pedirle a Claude (u otro host MCP) que consulte tus datos de OpenReplay en lenguaje natural. Esta página enumera lo que el MCP expone actualmente, con prompts de ejemplo para cada caso.

<Aside type="tip" title="Consejo" >
  Todos los prompts funcionan tanto si haces referencia a tu proyecto por **nombre** (p. ej. *"MyApp"*) como por **ID de sitio** (p. ej. *"site 1"*). Si no lo especificas, el MCP usa el proyecto más reciente que hayas mencionado.
</Aside>

## Buscar y filtrar sesiones

El MCP admite los mismos filtros disponibles en la interfaz de OpenReplay: navegador, sistema operativo, dispositivo, país, rutas de URL, clics, entradas, solicitudes de red, errores, metadatos, ID de usuario y duración.

```
Show me 20 recent sessions in MyApp
```

```
Sessions from Chrome users in France or Tunisia
```

```
Sessions with failed requests to /api/users in MyApp
```

```
Sessions where the user visited the signup page then clicked "Subscribe"
```

```
Sessions longer than 20 minutes for user tahay@asayer.io
```

El resultado se muestra como una lista interactiva dentro del chat, de modo que puedes hacer clic en cualquier sesión.

## Embudos

Los embudos miden la conversión a lo largo de una serie de URL, clics, entradas o eventos personalizados. El abandono se muestra paso a paso.

```
Show me the checkout funnel last week: /cart → /checkout → /confirm
```

```
Conversion from /pricing to /signup this month
```

```
Visited /pricing then clicked Subscribe
```

```
Page view /checkout followed by a JS exception
```

## Top de X

Encuentra las páginas más visitadas, los navegadores más usados, los países principales, los sistemas operativos más comunes, los dispositivos principales o las solicitudes de red más frecuentes.

```
What are the top visited pages this week?
```

```
Show browser distribution this month
```

```
Most popular countries last 30 days
```

```
Top 5 network requests today in MyApp
```

## Recorridos de usuario

Observa cómo navegan los usuarios por tu producto: en qué páginas aterrizan, a dónde van después y dónde abandonan.

```
Show me user journeys this month
```

```
Where do users drop off after the pricing page last week?
```

```
What pages do users visit after /signup?
```

```
Navigation paths for mobile users from France this week
```


## Ver la repetición de una sesión

Las repeticiones se reproducen en un reproductor integrado dentro del chat. Puedes desplazarte, pausar e inspeccionar igual que en la interfaz de OpenReplay.

```
Show me the replay of session 7891234567890
```

```
Replay the last session in the list above
```

## Combina y encadena

El verdadero valor surge al encadenar herramientas. Algunos ejemplos:

- *"Find sessions where users hit a JS exception on /checkout this week, then show me the replay of the most recent one and tell me what happened."*
- *"Pull the funnel for /pricing → /signup → /dashboard, then for the step with the biggest drop-off show me 5 sessions that dropped off there."*
- *"Plot the timeseries of sessions from France this month. If there's a spike, find a few sessions from that day and summarize what they did."*

Como Claude puede razonar sobre los resultados entre llamadas a las herramientas, puedes mantenerte a nivel de conversación en lugar de saltar entre paneles.
