Con el tracker de OpenReplay configurado y capturando sesiones de usuario, estás listo para descubrir lo que realmente sucede en tu producto. OmniSearch te ayuda a depurar problemas, identificar qué está fallando y comprender en detalle el comportamiento de los usuarios. Cierra la brecha entre los datos cuantitativos y la información cualitativa oculta en tus repeticiones de sesión.

## Comprender los eventos

Los **eventos** en OmniSearch representan las interacciones de los usuarios y los comportamientos del sistema que ocurren durante una sesión.

Se dividen en dos categorías principales: **Eventos capturados automáticamente** (seguimiento automático de la interacción del usuario) y **eventos de DevTools** (monitoreo técnico y depuración).

### Eventos capturados automáticamente
| **Nombre del evento**  | **Descripción** |
| --- | --- |
| Click | Registra los clics del usuario. |
| Text Input | Rastrea el texto introducido en los campos. |
| Visited URL | Registra las páginas visitadas |
| Tagged Element | Rastrea elementos etiquetados de forma personalizada. |

### Eventos de DevTools

|  **Nombre del evento**  | **Descripción** |
| --- | --- |
| Network Request | Monitorea la actividad de red. |
| GraphQL | Registra las consultas y respuestas de GraphQL. |
| State Action | Rastrea los cambios de estado en tu aplicación. |
| Error Message | Captura los mensajes de error en la consola. |
| Average CPU Load | Mide el uso promedio de CPU. |
| Average Memory Usage | Rastrea el consumo promedio de memoria. |

## Comprender los filtros

Los **filtros** en OmniSearch te ayudan a acotar las repeticiones de sesión según criterios específicos, lo que te permite centrarte en las sesiones de usuario más relevantes. Funcionan junto con los eventos para crear búsquedas precisas en los datos de tus sesiones.

Los filtros están organizados en cuatro categorías principales:

### Filtros de usuario

|  **Nombre del filtro**  | **Descripción** |
| --- | --- |
| User ID | Filtra usuarios específicos mediante sus identificadores únicos. |
| Anonymous User ID | Filtra sesiones de usuarios sin una identidad con sesión iniciada. |

### Filtros de sesión

|  **Nombre del filtro**  | **Descripción** |
| --- | --- |
| Country | Filtra sesiones por país. |
| City | Filtra sesiones por ciudad. |
| State/Province | Filtra sesiones por estado o provincia. |
| Operating System (OS) | Filtra sesiones por el sistema operativo utilizado. |
| Browser | Filtra sesiones por el tipo de navegador. |
| Device | Filtra sesiones por el dispositivo del usuario. |
| Platform | Filtra sesiones por el tipo de plataforma. |
| Version ID | Filtra sesiones por la versión de la aplicación. |
| Referrer | Filtra sesiones según la URL de referencia. |
| Duration | Filtra repeticiones de sesión por su duración. |
| UTM Source | Filtra sesiones por el parámetro UTM source. |
| UTM Medium | Filtra sesiones por el parámetro UTM medium. |
| UTM Campaign | Filtra sesiones por el parámetro UTM campaign. |

### Filtros de problemas

|  **Nombre del filtro**  | **Descripción** |
| --- | --- |
| Rage Click, Dead Click, Excessive Scrolling, Mouse Thrashing | Identifica sesiones con frustración del usuario causada por clics repetidos, botones o enlaces que no funcionan, desplazamiento excesivo, movimiento errático del ratón…. |
| Bad Request | Identifica sesiones con errores HTTP 400. |
| Missing Resource | Encuentra sesiones con recursos faltantes |
| CPU Issues | Identifica sesiones con un uso elevado de CPU. |
| Slow Resource | Identifica sesiones con recursos de carga lenta. |
| Slow Page Load | Identifica sesiones con tiempos de carga de página lentos. |
| Crash | Identifica sesiones donde la aplicación o la página se bloqueó. |
| Errors | Identifica sesiones con errores de la aplicación. |

### Filtros de metadatos

|  **Nombre del filtro**  | **Descripción** |
| --- | --- |
| Metadata key | Usa metadatos personalizados para filtrar sesiones. Si no están definidos, consulta cómo puedes añadir metadatos [aquí](/tutorials/metadata) |
