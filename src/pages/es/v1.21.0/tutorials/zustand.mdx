Zustand es un reemplazo muy rápido y minimalista para soluciones de gestión de estado como Redux. Con nuestro [complemento de Zustand](https://www.npmjs.com/package/@openreplay/tracker-zustand), puedes seguir los cambios de estado a lo largo del ciclo de vida de tu aplicación.

## Instalación del complemento

El complemento se puede instalar fácilmente con el siguiente comando:

```jsx
npm i @openreplay/tracker-zustand
```

## Uso del complemento en una aplicación Next.js

Para este ejemplo, vamos a utilizar la misma aplicación que se muestra en el [tutorial de Next.js de nuestro tracker](https://docs.openreplay.com/tutorials/next), así que si quieres más detalles sobre el proveedor de Context utilizado, revisa primero ese.

**Nota**: el código fuente completo de este ejemplo se encuentra en este [repositorio de Github](https://github.com/deleteman/nextjs-commerce-example/tree/zustand-store), échale un vistazo si necesitas más detalles.

El proveedor de Context que estoy usando permite la configuración de Tracker Plugins, así que después de instalar el complemento, modificaremos el archivo _app.tsx agregando la declaración de importación a continuación:

```jsx
import trackerZustand from '@openreplay/tracker-zustand'
```

Y luego, la función principal App debería verse así:

```jsx
export default function MyApp({ Component, pageProps }: AppProps) {
  const Layout = (Component as any).Layout || Noop

  useEffect(() => {
    document.body.classList?.remove('loading')
  }, [])

  let plugins = [
    {
      fn: trackerZustand,
      name: 'zustand',
    },
  ]

  return (
    <TrackerProvider
      config={{
        plugins
      }}
    >
      <Head />
      <ManagedUIContext>
        <Layout pageProps={pageProps}>
          <Component {...pageProps} />
        </Layout>
      </ManagedUIContext>
    </TrackerProvider>
  )
}
```

Esto iniciará automáticamente el complemento con el tracker, y también pondrá a disposición los datos devueltos por la ejecución de la función principal del complemento.

Lo siguiente es crear un store; en un momento agregaremos productos del sitio de comercio electrónico a este store.

### Creación de un store con el complemento de Zustand

El uso del complemento requiere que cambies la forma en que normalmente instancias un store de Zustand, así que siéntete libre de copiar y pegar el siguiente código como plantilla:

```jsx
import create from 'zustand'

const productStore = null
export const useProductsStore = (logger) => {
  if (!productStore) {
    //use a singleton pattern to make sure we only create the store the first time around
    productStore = createProductStore(logger)
  }
  return productStore
}

function createProductStore(logger) {
  return create(
    logger((set) => ({
      products: [],
      addProduct: (p) =>
        set((state) => {
          const prodFound = state.products.find((prod) => {
            return p.id == prod.id
          })
          if (prodFound) {
            p.amount = prodFound.amount + 1
          } else {
            p.amount = 1
          }

          return {
            products: [...state.products.filter((prod) => p.id != prod.id), p],
          }
        }),
    }))
  )
}
```

La función exportada creará el store la primera vez que se utilice, pero después simplemente devolverá el existente.

Por otro lado, la función createProductStore creará el store, establecerá la lógica para el método addProduct y utilizará la función logger que le proporcionamos. Este logger, como probablemente hayas adivinado, es el que recibimos al usar el complemento de Zustand.

Con el store listo, podemos comenzar a agregar productos en él.

## Agregar productos a nuestro store

El store exporta el método addProduct, que se encarga de llevar un seguimiento de los productos en nuestro carrito.

Para importar la función a tu código, debes:

1. Importar y usar el proveedor de contexto del Tracker.
2. Obtener el atributo pluginsReturnedValue del contexto.
3. Usarlo para llamar al método zustand (observa que el método lleva el nombre del atributo name cuando configuramos el complemento más arriba) y pasar un atributo de cadena para darle un nombre al store (este nombre se usará durante el replay).
4. El método zustand devolverá el logger que usaremos al llamar al hook useProductStore de zustand (el que creamos para el store).

Todo esto parece demasiado, pero esencialmente se traduce en:

```jsx
const { pluginsReturnedValues } = useContext(TrackerContext)

const useProductStoreTracked = useProductsStore(
  pluginsReturnedValues.zustand('products_store')
)
const { addProduct } = useProductStoreTracked()
```

Ahora, simplemente puedes llamar al método addProduct, y este agregará los productos del carrito al store.

## Lectura de datos del store

Como probablemente puedas adivinar, si quieres leer datos del store de Zustand, debes seguir el mismo conjunto de pasos. Pero en lugar de tomar el método addProduct, desestructuraremos el arreglo products.

Así:

```jsx
const { pluginsReturnedValues } = useContext(TrackerContext)

const useProductStoreTracked = useProductsStore(
  pluginsReturnedValues.zustand('products_store')
)
const { products } = useProductStoreTracked()
```

Luego puedes iterar sobre los products normalmente:

```jsx
<div>
  {products.map((p: Product, idx: Number) => {
    return (
      <div key={+idx}>
        {p.name}({p.amount})
      </div>
    )
  })}
</div>
```

El código anterior itera sobre la lista de products y los renderiza en pantalla. Cada vez que agregues otro producto al store, también se actualizará aquí.

## Sincronizar el estado con la repetición de sesión

Para revisar los cambios de estado durante el replay, simplemente haz clic en la nueva pestaña Zustand en la esquina inferior derecha:

![Nueva pestaña para los datos rastreados de Zustand](/tutorials//zustand/zustand-tab.png)

Al hacer clic en ella, verás el contenido del store y podrás entender cómo cambia y los datos guardados dentro de él:

![Datos de estado mostrados durante el replay](/tutorials//zustand/state-replay.png)

Observa el nombre del store; en realidad es el nombre que usamos al llamar al método zustand la primera vez.

Si no estás usando Next.js sino otro framework, la forma de usarlo debería ser similar; la clave está en la forma en que creas el store. Mientras encuentres una manera de usar la función logger al crearlo, podrás rastrear el estado durante el replay.
