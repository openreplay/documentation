Este método te permite reportar un "evento funcional". A través de los eventos funcionales puedes señalar momentos dentro de la sesión del usuario que son relevantes desde un punto de vista funcional y que, de forma predeterminada, nuestro rastreador no puede captar ni marcar por sí solo. Un ejemplo de evento funcional sería que el usuario añada un artículo al carrito de compra. Si bien nuestro rastreador podrá registrar la acción, no sabría marcarla como un evento relevante dentro de la línea de tiempo de tu sesión. Sin embargo, gracias a este método, puedes hacerlo manualmente.

Consulta [eventos personalizados](/installation/custom-events#functionalevents) para más detalles.

## Firma

```kotlin
data class User(val name: String, val age: Int)

OpenReplay.event("Test Event", User("John", 25))
```

## Parámetros

- `key: string`: El nombre de la clave donde se guardará la información. Esta clave se puede usar para filtrar y buscar dentro de la plataforma de OpenReplay.
- `payload: Encodable`: El contenido del evento. Puede ser cualquier dato codificable (JSON).
