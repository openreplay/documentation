Esta página explica cómo sanear los componentes de la interfaz de usuario en tus aplicaciones móviles utilizando los SDK de OpenReplay para iOS y Android. Para obtener más detalles sobre cómo sanear las solicitudes de red, consulta [Opciones de red (móvil)](/network-options-mobile).

## iOS - Saneamiento de componentes de la interfaz de usuario

Las vistas saneadas aparecerán difuminadas en la repetición resultante. Los componentes se eliminan automáticamente (para preservar la memoria) en cuanto dejan de ser visibles.

### SwiftUI

Añade el modificador `.sensitive()` al componente que deseas sanear.

```swift
import ORTracker

Text("Sensitive Information")
    .sensitive()
```

### UIKit

Usa el método `addIgnoredView` para marcar una vista como sensible.

```swift
import ORTracker

ORTracker.shared.addIgnoredView(view)
```

### React Native

Envuelve los componentes sensibles con el componente `ORSanitizedView` para difuminarlos en la repetición.

```js
import OR from '@openreplay/react-native';

<OR.ORSanitizedView>
  <SensitiveComponent />
</OR.ORSanitizedView>
```

## Android - Saneamiento de componentes de la interfaz de usuario

Las vistas saneadas aparecen difuminadas en la repetición resultante y se eliminan automáticamente de la memoria cuando están fuera de la pantalla.

### View

Marca una vista como sensible usando el método `addIgnoredView`.

```kotlin
import com.openreplay.tracker.OpenReplay

OpenReplay.addIgnoredView(view)
```
