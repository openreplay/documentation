Este método puede usarse para reportar un "[evento técnico](/installation/custom-events)", también conocido como "issue" (incidencia). Estas incidencias pueden usarse para reportar información adicional sobre un problema que el tracker no puede detectar de forma predeterminada. Un buen ejemplo de esto sería el usuario intentando pagar una compra en nuestro sitio de comercio electrónico y recibiendo un error de la pasarela de pago. Aunque el tracker podría registrar el payload con el error, no lo identificaría como un problema. Este método te permite marcar este comportamiento como una incidencia para un filtrado adicional de las sesiones.

<Aside type="tip">
  También tenemos un tutorial detallado sobre cómo rastrear eventos personalizados, puedes consultarlo [aquí](/tutorials/custom-events).
</Aside>

## Firma

```js
issue(key: string, payload?: any): void
```

## Parámetros

- `key: string`: El nombre de la clave donde se guardarán los datos del evento. Puedes usar este valor en los campos de filtro dentro de la plataforma de OpenReplay.
- `payload: any`: Los datos relacionados con el evento rastreado. Puedes guardar aquí cualquier objeto serializable.

## Valor de retorno

Este método no devuelve ningún valor, podrás ver la información rastreada dentro de la plataforma de OpenReplay.
