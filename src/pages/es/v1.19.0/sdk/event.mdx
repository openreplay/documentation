Este método te permite reportar un "evento funcional". A través de los eventos funcionales puedes señalar momentos dentro de la sesión del usuario que son relevantes desde un punto de vista funcional y que, de forma predeterminada, nuestro tracker no puede captar ni marcar por sí solo. Un ejemplo de evento funcional sería que el usuario añada un artículo al carrito de compra. Si bien nuestro tracker podrá registrar la acción, no sabría marcarla como un evento relevante dentro de la línea de tiempo de tu sesión. Sin embargo, gracias a este método, puedes hacerlo manualmente.

<Aside type="tip">
  También tenemos un tutorial detallado sobre cómo enviar eventos personalizados, puedes consultarlo [aquí](/tutorials/custom-events).
</Aside>

Consulta [eventos personalizados](/installation/custom-events#functionalevents) para más detalles.

## Firma

```js 
event(key: string, payload?: any, issue?: boolean): void
```

## Parámetros

- `key: string`: El nombre de la clave donde se guardará la información. Esta clave puede usarse para filtrar y buscar dentro de la plataforma de OpenReplay.
- `payload: any`: El contenido del evento. Puede ser cualquier dato serializable.
- `issue: boolean` : Indica si se trata de un evento funcional (si se establece en `false`) o de un "evento técnico" (si se establece en `true`). Su valor predeterminado es `false`.
