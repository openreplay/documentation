## Sanear datos de repetición

Puedes sanear lo que OpenReplay captura tanto a nivel global como granular. Cualquier dato ofuscado o ignorado no se capturará en el origen. En otras palabras, esos datos nunca saldrán del navegador del usuario.

### Nivel global

Los datos pueden sanearse a nivel del tracker, al configurar OpenReplay.

#### Estoy usando OpenReplay Script

![Configuración del proyecto en OpenReplay](/static/gdpr-1.png#center)

1) Haz clic en el *Tracking Code* de tu proyecto en 'Preferences > Projects'.
2) Las opciones de grabación de datos deberían aparecer justo antes del fragmento de código:
- **Inputs**: Para ignorar, ofuscar o grabar todos los valores de entrada de tus usuarios.
- **Do not record any numeric text**: Para ignorar los valores numéricos contenidos en elementos de texto.
- **Do not record email addresses**: De forma predeterminada, todos los correos electrónicos contenidos en elementos de texto se ofuscan y no aparecerán en las repeticiones. Puedes desactivar esto desmarcando la casilla.
3) Cambiar las opciones de grabación de datos afecta al fragmento de código. Asegúrate de copiar/pegar el script actualizado en tu aplicación web.

#### Estoy usando OpenReplay NPM

Hay un conjunto de opciones relacionadas con la privacidad que puedes pasar al constructor:
- `obscureTextEmails?: boolean` Ofusca los correos electrónicos en elementos de texto. Los correos electrónicos se convertirán en una cadena aleatoria de asteriscos. Predeterminado: true.
- `obscureTextNumbers?: boolean` Ofusca los números en elementos de texto. Los números se convertirán en una cadena aleatoria de asteriscos. Predeterminado: false.
- `obscureInputEmails?: boolean` Ofusca los correos electrónicos en los campos de entrada. Los valores de correo electrónico se convertirán en una cadena aleatoria de asteriscos. Predeterminado: true.
- `obscureInputNumbers?: boolean` Ofusca los números en los campos de entrada. Los valores numéricos se convertirán en una cadena aleatoria de asteriscos. Predeterminado: false.
- `obscureInputDates?: boolean` Ofusca las fechas en los campos de entrada. Los valores de fecha se convertirán en una cadena aleatoria de asteriscos. Predeterminado: false.
- `defaultInputMode?: 0 | 1 | 2` Modo de captura predeterminado para los valores de entrada. Respectivamente: plano, ofuscado o ignorado. Predeterminado: 1 (ofuscado).

El texto, los correos electrónicos y los números saneados se ofuscan o suprimen antes de enviar los datos a los servidores de OpenReplay. Por lo tanto, los cambios aplicados a las opciones anteriores no pueden ser retroactivos y solo se aplicarán a los datos recién recopilados.

### Nivel granular

También puedes sanear tus datos a nivel de código para una mayor granularidad. Esto resulta útil para ofuscar o ignorar elementos del DOM y campos de entrada.

#### Usando atributos HTML

- `data-openreplay-obscured` para enmascarar el contenido de texto de las etiquetas `<input>`, `<img>` y otros elementos HTML (es decir, `<div>`) con la excepción de `<svg>` y `<canvas>`.

```html
<!--Obscuring input fields-->
<form action="/action_page.php">
  Name: <input type="text" name="fname" data-openreplay-obscured><br>
  <input type="submit" value="Submit">
</form>
<!--Obscuring a div element-->
<div style="background-color:lightblue" data-openreplay-obscured>
  <h3>This is a sensitive information</h3>
  <p>This is an important paragraph</p>
  <img src="important_image.jpg">
</div>
```

- `data-openreplay-hidden` para censurar el contenido de las etiquetas `<input>`, `<img>`, `<svg>` y otros elementos HTML (es decir, `<div>`) con la excepción de `<canvas>`. El contenido del elemento (incluidos sus componentes hijos) se elimina, pero se conservan sus props y argumentos, como en este ejemplo:

```html
<div style="background-color:lightblue" data-openreplay-hidden>
  <div>This is a sensitive information</div>
  <svg>An important chart</svg>
</div>
```

Esto dará como resultado que el tracker grabe el siguiente DOM:

```html
<div style="background-color:lightblue" data-openreplay-hidden>
</div>
```

#### Usando la función domSanitizer

- La función `domSanitizer: (node: Element) => SanitizeLevel` evita tener que instrumentar cada componente HTML que necesita sanearse. `SanitizeLevel` puede ser `Plain` (0), `Obscured` (1) o `Hidden` (2). Esta función se pasa al constructor del tracker. A continuación se muestra un ejemplo de cómo censurar todos los elementos HTML que tienen un nombre de clase CSS específico:

```js
// Import SanitizeLevel enum
import Tracker, { SanitizeLevel } from "@openreplay/tracker";

const tracker = new Tracker({
  projectKey: PROJECT_KEY,
  domSanitizer: (node: Element) => {
    const elementClassNames = node.classList

    if (elementClassNames.contains('to_be_redacted_class')) {
      return SanitizeLevel.Hidden
    }

    if (node.id === 'sensitiveData') {
      return SanitizeLevel.Obscured
    }

    return SanitizeLevel.Plain
  }
})
tracker.start()
```

> **Nota:** Todos los elementos saneados se enmascaran/censuran a nivel del origen (tracker) para que los datos sensibles nunca lleguen a la instancia de OpenReplay.

## Sanear datos de DevTools

Puedes usar el SDK de OpenReplay para sanear los datos de red y el estado de la aplicación.

### Red

Capturar solicitudes/respuestas puede registrar información altamente sensible o privada que tu aplicación pueda estar enviando/recibiendo. Asegúrate de censurar adecuadamente tus datos de red.

#### Fetch/XHR

Usa la opción `sanitizer` para censurar datos sensibles de las solicitudes/respuestas Fetch/XHR. Consulta las [opciones de red](/installation/network-options#options) para obtener más detalles.

Además, consulta este tutorial paso a paso sobre cómo [capturar y sanear solicitudes de red](/tutorials/capture-request).

#### GraphQL

Para sanear datos de GraphQL, debes clonar las variables clave y los resultados de la operación antes de llamar al tracker. Consulta cómo [sanear los datos grabados en GraphQL](/tutorials/graphql/#sanitizing-the-recorded-data) para obtener más detalles.

### Gestión de estado

Depura los datos sensibles de tu store de estado del frontend usando el SDK de OpenReplay.

#### Redux

Usa las opciones `actionFilter`, `actionTransformer` y `stateTransformer` para censurar acciones y subárboles sensibles de tu store. Consulta la [documentación del plugin de redux](/plugins/redux/#options) para obtener más detalles.

#### VueX

Usa las opciones `filter`, `transformer` y `mutationTransformer` para censurar mutaciones y subárboles sensibles de tu store. Consulta la [documentación del plugin de vuex](/plugins/vuex/#options) para obtener más detalles.

#### Zustand

Usa las opciones `filter`, `transformer` y `mutationTransformer` para censurar mutaciones y subárboles sensibles de tu store. Consulta la [documentación del plugin de zustand](/plugins/zustand/#options) para obtener más detalles.

#### Pinia

Usa las opciones `filter`, `transformer` y `mutationTransformer` para censurar mutaciones y subárboles sensibles de tu store. Consulta la [documentación del plugin de pinia](/plugins/pinia/#options) para obtener más detalles.

#### NgRx

Usa las opciones `actionFilter`, `actionTransformer` y `stateTransformer` para censurar acciones y subárboles sensibles de tu store. Consulta la [documentación del plugin de ngrx](/plugins/ngrx/#options) para obtener más detalles.
