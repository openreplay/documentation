OpenReplay necesita acceso público (solo HTTPS) a los recursos de tu aplicación (CSS, fuentes e iconos) para renderizar correctamente las grabaciones. Copiamos esos recursos para que puedas reproducir tus sesiones aunque el contenido de tu aplicación haya cambiado. Las imágenes se manejan de forma diferente: se recuperan en tiempo real cuando se reproduce una sesión.

Por estas razones, recomendamos probar la instalación en un entorno de acceso público (es decir, staging) en lugar de en localhost. Aun así, existe una solución alternativa para el desarrollo local, aunque no es perfecta: el uso de [ngrok](https://ngrok.com/) (o cualquier solución similar) para hacer que los recursos de tu aplicación web sean accesibles para OpenReplay.

Instala [ngrok](https://ngrok.com/) y luego sigue estos pasos:
1. Ejecuta ngrok con el puerto de tu localhost (es decir, `ngrok http 3000` si tu servidor está ejecutándose en localhost:3000).
2. Abre tu aplicación en la URL de ngrok especificada (es decir, `https://558csdft.ngrok.io`). Asegúrate de usar el enlace HTTPS, de lo contrario OpenReplay no capturará nada por razones de seguridad.

Las reproducciones de sesiones deberían estar disponibles después en el panel, pero es posible que les falten imágenes como se explicó anteriormente.
