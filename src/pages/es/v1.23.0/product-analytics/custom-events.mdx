Los eventos personalizados son ideales para agregar más inteligencia al registrar eventos específicos de tu dominio junto con las repeticiones de sesión. OpenReplay utiliza 2 tipos de eventos: funcionales (`event`) y técnicos (`issue`). Todos los eventos se indexan para facilitar la búsqueda y se sincronizan con las grabaciones de sesión.

<Aside type="tip">
  También tenemos un tutorial detallado sobre cómo enviar eventos personalizados, puedes consultarlo [aquí](/tutorials/custom-events).
</Aside>

## Eventos funcionales

Los eventos funcionales se indexan y facilitan la búsqueda de grabaciones de sesión específicas. Si OpenReplay los recibió correctamente, estarán disponibles como filtros en la barra de omnisearch.

![Functional Event](/static/custom-events-1.png#center)

### Con NPM

Usa el método `event` para señalar eventos funcionales como *order completed* o *product added*. Toma 2 parámetros: *name* (string) y *payload* (objeto JS no recursivo).

```js
tracker.event('product_added', 'shoes'); // after tracker.start()
```

## Eventos técnicos

Los eventos técnicos se muestran en la repetición de sesión en la pestaña *Events* de DevTools y como anotaciones en la reproducción. También se tienen en cuenta en los embudos (Funnels) para correlacionar las caídas de conversión con problemas técnicos.

![Technical Event](/static/custom-events-2.png#center)

### Con NPM

`issue` se usa para enviar eventos técnicos, como errores, que pueden ocurrir en tu stack u otros sistemas posteriores. `issue` toma 2 parámetros: *name* (string) y *payload* (objeto JS no recursivo).

```js
tracker.issue('payment_error', {code: 500, context: 42}); // after tracker.start()
```
