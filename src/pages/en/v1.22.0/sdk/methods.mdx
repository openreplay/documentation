## Methods

- [Constructor](/sdk/constructor): Learn all about the initialization options for the tracker.
- [event()](/sdk/methodsevent): Send custom events to be recorded as part of your replays with this method.
- [getSessionToken()](/sdk/methods/get-session-token): Get the currently active session's token.
- [getSessionID()](/sdk/methods/get-session-id): Get the currently active session's ID.
- [getSessionURL()](/sdk/methods/get-session-url): Get the url to replay the current session. 
- [handleError()](/sdk/methods/handle-error): Capture custom error information and save it as part of your recording.
- [isActive()](/sdk/methods/is-active): Get the current state of the active session.
- [issue()](/sdk/methods/issue): Capture custom technical events and save them as part of the replay.
- [setUserID()](/sdk/methods/set-user-id): Identify your users with this method.
- [setMetadata()](/sdk/methods/set-metadata): Set a metadata field and save it as part of the recording.
- [start()](/sdk/methods/start): Start recording a session.
- [stop()](/sdk/methods/stop): Stop the recording of a session with this method.
- [clearPersistFlag()](/sdk/methods/clear-persist-flag): Removes persistent Feature Flags from browser's sessionStorage.
- [getFeatureFlag()](/sdk/methods/get-feature-flag): Returns a `IFeatureFlag` by key if eixsts.
- [onFlagsReload()](/sdk/methods/on-flags-reload): Sets the callback to use when Feature Flags are loaded.
- [isFlagEnabled()](/sdk/methods/is-flag-enabled): Returns `true` if flag with this key does exist and is active.
- [reloadFlags()](/sdk/methods/reload-flags): Reloads all flags (can be useful if user session had any change of metadata, userId or any other information that could be required for certain flag conditions).
- [getAllFeatureFlags()](/sdk/methods/get-all-feature-flags): Returns all flags (`IFeatureFlag[]`) that are active right now.
- [coldStart()](/sdk/methods/cold-start): Starts buffering messages (refreshing buffer) with the possibility to call `start()` to enable session recording and send the buffer content.
- [startOfflineRecording()](/sdk/methods/start-offline-recording): Starts offline session recording which can be sent with `uploadOfflineRecording`.
- [uploadOfflineRecording()](/sdk/methods/upload-offline-recording): Uploads the stored session buffer to the backend.
- [forceFlushBatch()](/sdk/methods/force-flush-batch): Force sends the current batch of messages without waiting for its completion.
- [trackWs()](/sdk/methods/track-ws): Create a hook to capture WebSocket messages.
