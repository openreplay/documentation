## Informations de timing manquantes

Lorsque CORS est en vigueur, de nombreuses valeurs des propriétés de timing sont renvoyées à zéro, à moins que la politique d'accès du serveur n'autorise le partage de ces valeurs. Pour cela, le serveur fournissant la ressource doit envoyer l'en-tête de réponse HTTP `Timing-Allow-Origin` avec une valeur spécifiant l'origine ou les origines autorisées à obtenir les valeurs d'horodatage restreintes.

Ceci est particulièrement important pour les ressources cross-origin, telles que celles chargées depuis un domaine différent de celui qui sert la page principale. Si l'en-tête `Timing-Allow-Origin` n'est pas correctement configuré, OpenReplay ne pourra pas enregistrer les informations de timing de ces ressources.

Par exemple, pour permettre à https://application.example.com de voir les informations de timing des ressources lors d'une requête vers le backend sur https://data.example.com, le backend doit inclure les en-têtes suivants :

```http
Timing-Allow-Origin: https://application.example.com
```
