La Gestion des données dans OpenReplay vous permet de suivre des événements personnalisés et des propriétés utilisateur pour alimenter votre analyse produit. Cette page couvre à la fois l'intégration du SDK pour l'envoi des données et l'interface utilisateur pour les gérer.

Pour la documentation complète du SDK, consultez la [référence de l'Analytics SDK](/sdk/analytics).

## Vue d'ensemble des événements

OpenReplay capture deux types d'événements :

- **Événements capturés automatiquement** : des interactions suivies automatiquement comme les vues de page, les clics et les soumissions de formulaires
- **Événements personnalisés** : des événements propres à votre domaine que vous définissez, tels que « achat finalisé » ou « fonctionnalité activée »

Les événements personnalisés vous permettent de suivre les actions utilisateur spécifiques qui comptent pour votre activité et de les utiliser dans des fonctionnalités d'analyse telles que les [Tendances](/product-analytics/trends), les [Entonnoirs](/product-analytics/funnels) et les [Parcours](/product-analytics/journeys).

## Capture des événements

Pour suivre des événements personnalisés, utilisez le module d'événements d'analyse du SDK. Pour la documentation complète de l'API, consultez la [référence de l'Events SDK](/sdk/analytics/events).

### Exemple de base

```ts
// Track a simple event
tracker.analytics.events.sendEvent("Button Clicked", {
  button_text: "Start",
  page: "home",
})
```

### Super propriétés

Les super propriétés sont des paires clé/valeur persistantes attachées automatiquement à chaque événement suivi :

```ts
// Set super properties that apply to all events
tracker.analytics.events.setProperty({
  app_version: "1.42.0",
  env: "prod",
})

// This event will include app_version and env automatically
tracker.analytics.events.sendEvent("Signup Started", {
  plan: "pro",
})
```

## Enrichissement des utilisateurs

Les propriétés utilisateur vous permettent de stocker des attributs sur vos utilisateurs à des fins de segmentation et d'analyse. Pour la documentation complète de l'API, consultez la [référence du People SDK](/sdk/analytics/people).

### Identification des utilisateurs

```ts
// Identify the current user
tracker.analytics.people.identify("user_123")

// Set user properties
tracker.analytics.people.setProperties({
  email: "user@example.com",
  plan: "pro",
})
```

### Types de propriétés

OpenReplay prend en charge plusieurs types de propriétés :

| Type | Description | Exemples |
|------|-------------|----------|
| **Suivi automatique** | Capturées automatiquement par le SDK | Appareil, navigateur, OS, paramètres UTM, référent |
| **Spéciales** | Propriétés spéciales au comportement défini | `$user_id`, `$email`, `$name` |
| **Personnalisées** | Propriétés définies par l'utilisateur | `plan type`, `company`, `signup date` |

## Gestion des données dans l'interface utilisateur

OpenReplay fournit une interface de Gestion des données pour parcourir, filtrer et organiser vos utilisateurs, vos événements et leurs propriétés respectives.

### Personnes

<img src="/data-management/people-page.png" alt="Page des événements" style={{ width: '700px', height: 'auto' }} />

L'onglet Personnes offre une vue de vos utilisateurs identifiés :

- Parcourez les utilisateurs et leurs propriétés
- Recherchez par identifiant d'utilisateur ou par valeurs de propriétés
- Consultez l'activité de l'utilisateur et l'historique des sessions
- Accédez aux profils individuels des utilisateurs

### Événements

<img src="/data-management/events-page.png" alt="Page des événements" style={{ width: '700px', height: 'auto' }} />

L'onglet Événements affiche tous les types d'événements suivis dans votre projet :

- Parcourez et recherchez les événements par nom
- Filtrez par type d'événement (capturés automatiquement ou personnalisés)
- Modifiez les métadonnées de l'événement (nom d'affichage, description)
- Contrôlez la visibilité de l'événement dans les tableaux de bord et les filtres

### Propriétés

<img src="/data-management/props-page.png" alt="Page des événements" style={{ width: '700px', height: 'auto' }} />

L'onglet Propriétés vous permet de gérer à la fois les propriétés utilisateur et les propriétés d'événement :

- Basculez entre les propriétés utilisateur et les propriétés d'événement
- Consultez les types de propriétés et les distributions de valeurs
- Modifiez les noms d'affichage des propriétés
- Contrôlez la visibilité des propriétés dans les tableaux de bord et les filtres 

### Activité

<img src="/data-management/activity-page.png" alt="Page des événements" style={{ width: '700px', height: 'auto' }} />

L'onglet Activité affiche un flux en temps réel des événements :

- Surveillez les événements au fur et à mesure de leur ingestion
- Filtrez par type d'événement ou par utilisateur
- Vérifiez que les événements sont correctement capturés

### Actions

Les Actions vous permettent de regrouper plusieurs événements dans une collection unique et réutilisable dotée d'un nom et d'une description personnalisés. Au lieu de sélectionner des événements individuels à chaque fois, vous pouvez définir une Action une seule fois et l'utiliser dans tout OpenReplay.

<img src="/data-management/actions-list.png" alt="Liste des actions" style={{ width: '700px', height: 'auto' }} />

Pour créer une Action :

1. Accédez à l'onglet **Actions** dans la Gestion des données
2. Cliquez sur **Créer une action**
3. Donnez-lui un nom et une description facultative
4. Sélectionnez les événements à inclure dans la collection
5. Enregistrez l'Action

Une fois créées, les Actions peuvent être utilisées dans :

- **Recherche de sessions** — filtrez les sessions par Action pour trouver rapidement l'activité utilisateur pertinente
- **Tableaux de bord** — utilisez les Actions comme métrique dans vos graphiques et widgets
- **Analyse des utilisateurs** — segmentez et analysez les utilisateurs en fonction de l'occurrence d'une Action

<img src="/data-management/actions-search.png" alt="Utilisation des Actions dans la Recherche de sessions" style={{ width: '700px', height: 'auto' }} />

### Alias compatibles avec Mixpanel

Toutes les méthodes prennent en charge une couche de compatibilité fournie pour plus de familiarité :

```ts
tracker.analytics.people.set            // setProperties
tracker.analytics.people.set_once       // setPropertiesOnce
tracker.analytics.people.append         // appendValues
tracker.analytics.people.union          // appendUniqueValues
tracker.analytics.people.incrementBy    // increment
tracker.analytics.events.register       // setProperty
tracker.analytics.events.register_once  // setPropertiesOnce
tracker.analytics.events.unregister     // unsetProperties
tracker.analytics.events.track          // sendEvent
```
