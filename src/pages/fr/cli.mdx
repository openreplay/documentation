À partir de la v1.10.0, nous avons publié notre utilitaire CLI. Grâce à lui, les tâches les plus courantes que vous auriez à exécuter pour gérer votre version auto-hébergée d'OpenReplay peuvent désormais être réalisées avec une simple commande.


## Installation de la CLI

La CLI est fournie avec chaque version auto-hébergée d'OpenReplay.

## Utilisation de la CLI

La CLI utilise la syntaxe suivante :

```bash 
openreplay [command] <service>
```

La liste des commandes disponibles est la suivante :

- `-h or --help` : Cette commande affiche la liste des commandes et des services disponibles.
- `-s or --status` : Si vous voulez connaître l'état des services déployés, cette commande vous indiquera tout ce que vous devez savoir.
- `-u or --upgrade` : Utilisée pour mettre à niveau votre installation vers la dernière version disponible.
- `-U or --deprecated-upgrade /path/to/old_vars.yaml`
- `-r or --restart` : Vous permet de redémarrer tous les services. Cela permet de corriger les services qui ne répondent pas.
- `-R or --Reload` : Recharge la configuration d'un service, à n'utiliser donc que lorsque vous modifiez quelque chose.
- `-p or --install-packages` : Met à jour toutes les dépendances tierces (comme `kubectl`, `helm`, etc).
- `-i or --install domain.name.com` : Installe OpenReplay sur la machine.
- `-e or --edit` : Modifie la configuration installée d'OpenReplay et la recharge.
- `-l or --logs SERVICE` : Affiche les journaux d'un service en particulier.
