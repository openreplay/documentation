Commence à mettre en mémoire tampon les messages sans démarrer la session réelle. La mémoire tampon conservera les 30 dernières secondes d'événements enregistrés. Vous pouvez activer la session en appelant `start()` ou via un déclencheur conditionnel.

## Signature

```ts
coldStart(startOpts?: Partial<StartOptions>, isConditional?: boolean): Promise<void>
```

### Paramètres
Avec `startOpts`, vous pouvez personnaliser différents aspects de l'enregistrement :
- `userID: string` : Utilisé pour définir manuellement le `userID` afin de le suivre à travers les sessions. Il s'agit d'une valeur de type chaîne de caractères et elle peut être tout ce que vous voulez.
- `metadata: Record<string, string>` : Définit manuellement les valeurs des métadonnées. Consultez la [section Métadonnées](/installation/metadata/) pour en savoir plus à ce sujet.
- `forceNew: boolean` : Utilisé pour forcer une nouvelle session après le rechargement de la page. Par défaut, il est défini sur `false`, de sorte qu'après un rechargement, la session est conservée.
- `sessionHash: string` : Utilisé pour les sessions persistantes (sticky sessions). Utile si vous avez une application multisite ou si vous devez rediriger l'utilisateur hors de votre site puis le ramener (comme vers une passerelle de paiement).
- `isConditional` : Détermine si le tracker doit attendre que la condition de déclenchement soit remplie avant de démarrer automatiquement la session.

> Remarque : L'enregistrement conditionnel n'est pris en charge que dans nos offres Cloud et Enterprise Edition.

### Valeur de retour

Cette méthode ne renvoie aucune valeur.
