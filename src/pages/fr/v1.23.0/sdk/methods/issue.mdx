Cette méthode peut être utilisée pour signaler un "[événement technique](/installation/custom-events)", également appelé "issue" (incident). Ces incidents peuvent servir à signaler des informations supplémentaires concernant un problème que le tracker ne peut pas détecter par défaut. Un bon exemple serait celui d'un utilisateur essayant de régler un achat sur notre site e-commerce et recevant une erreur de la passerelle de paiement. Bien que le tracker puisse enregistrer le payload contenant l'erreur, il ne l'identifierait pas comme un problème. Cette méthode vous permet de marquer ce comportement comme un incident afin de filtrer davantage les sessions.

<Aside type="tip">
  Nous proposons également un tutoriel détaillé sur la manière de suivre les événements personnalisés, vous pouvez le consulter [ici](/tutorials/custom-events).
</Aside>

## Signature

```js
issue(key: string, payload?: any): void
```

## Paramètres

- `key: string` : Le nom de la clé sous laquelle les données de l'événement seront enregistrées. Vous pouvez utiliser cette valeur dans les champs de filtre au sein de la plateforme OpenReplay.
- `payload: any` : Les données liées à l'événement suivi. Vous pouvez enregistrer ici n'importe quel objet sérialisable.

## Valeur de retour

Cette méthode ne renvoie aucune valeur, vous pourrez voir les informations suivies au sein de la plateforme OpenReplay.
