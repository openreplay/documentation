Une fois l'application MCP [configurée](/mcp/setup) et la connexion établie, vous pouvez demander à Claude (ou à un autre hôte MCP) d'interroger vos données OpenReplay en langage courant. Cette page répertorie ce que le MCP expose actuellement, avec des exemples de prompts pour chaque cas.

<Aside type="tip" title="Astuce" >
  Tous les prompts fonctionnent, que vous référenciez votre projet par son **nom** (p. ex. *"MyApp"*) ou par son **identifiant de site** (p. ex. *"site 1"*). Si vous ne le précisez pas, le MCP utilise le projet le plus récemment mentionné.
</Aside>

## Rechercher et filtrer les sessions

Le MCP prend en charge les mêmes filtres que ceux disponibles dans l'interface d'OpenReplay : navigateur, système d'exploitation, appareil, pays, chemins d'URL, clics, saisies, requêtes réseau, erreurs, métadonnées, ID utilisateur et durée.

```
Show me 20 recent sessions in MyApp
```

```
Sessions from Chrome users in France or Tunisia
```

```
Sessions with failed requests to /api/users in MyApp
```

```
Sessions where the user visited the signup page then clicked "Subscribe"
```

```
Sessions longer than 20 minutes for user tahay@asayer.io
```

Le résultat s'affiche sous forme de liste interactive dans le chat, ce qui vous permet de cliquer sur n'importe quelle session.

## Entonnoirs

Les entonnoirs mesurent la conversion au fil d'une série d'URL, de clics, de saisies ou d'événements personnalisés. L'abandon est affiché étape par étape.

```
Show me the checkout funnel last week: /cart → /checkout → /confirm
```

```
Conversion from /pricing to /signup this month
```

```
Visited /pricing then clicked Subscribe
```

```
Page view /checkout followed by a JS exception
```

## Top de X

Trouvez les pages les plus visitées, les principaux navigateurs, les principaux pays, les principaux systèmes d'exploitation, les principaux appareils ou les principales requêtes réseau.

```
What are the top visited pages this week?
```

```
Show browser distribution this month
```

```
Most popular countries last 30 days
```

```
Top 5 network requests today in MyApp
```

## Parcours utilisateurs

Découvrez comment les utilisateurs naviguent dans votre produit : sur quelles pages ils arrivent, où ils vont ensuite, où ils abandonnent.

```
Show me user journeys this month
```

```
Where do users drop off after the pricing page last week?
```

```
What pages do users visit after /signup?
```

```
Navigation paths for mobile users from France this week
```

## Regarder le replay d'une session

Les replays sont lus dans un lecteur intégré au chat. Vous pouvez naviguer, mettre en pause et inspecter, exactement comme dans l'interface d'OpenReplay.

```
Show me the replay of session 7891234567890
```

```
Replay the last session in the list above
```

## Combiner et enchaîner

La vraie valeur vient de l'enchaînement des outils. Quelques exemples :

- *"Find sessions where users hit a JS exception on /checkout this week, then show me the replay of the most recent one and tell me what happened."*
- *"Pull the funnel for /pricing → /signup → /dashboard, then for the step with the biggest drop-off show me 5 sessions that dropped off there."*
- *"Plot the timeseries of sessions from France this month. If there's a spike, find a few sessions from that day and summarize what they did."*

Comme Claude peut raisonner sur les résultats entre les appels d'outils, vous pouvez rester au niveau de la conversation au lieu de sauter d'un tableau de bord à l'autre.
