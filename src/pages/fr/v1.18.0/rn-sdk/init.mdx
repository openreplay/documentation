## Installation

La ligne suivante installera le tracker et, avec lui, le SDK pour vous permettre de profiter de toutes les fonctionnalités du tracker.

```bash
npm i @openreplay/react-native
```

## Initialisation

Lors de l'instanciation du tracker d'OpenReplay, plusieurs options de configuration sont disponibles pour personnaliser de nombreux aspects de l'enregistrement et de l'expérience d'enregistrement. Vous devez définir l'option `projectKey` dans le constructeur. Vous pouvez obtenir cette valeur depuis votre tableau de bord OpenReplay sous 'Preferences > Projects'.

<Aside type="tip">
	Notez que si vous avez plusieurs projets, vous devez faire attention à la clé de projet que vous choisissez.
</Aside>


### Ajouter le code de suivi

Ajoutez le code suivant à votre fichier racine :

```js
import Openreplay from '@openreplay/react-native';

// ...
useEffect(() => {
	OpenReplay.tracker.startSession(
		'yourProjectKey',
		options, // explained below, set {} if empty
		'https://local.openreplay.instance/ingest' // not required if you're using managed version
	);
}, [])
```

### Assainir les données

Consultez [comment assainir les données dans les rejeux React Native](/rn-sdk/sanitization) pour plus de détails.

## Options d'initialisation

Par défaut, toutes les options sont définies sur `true` :

- `crashes: Bool` Active crashlytics.
- `analytics: Bool` Active le suivi analytique des vues marquées.
- `performances: Bool` Active l'écouteur de performances.
- `logs: Bool` Active l'écouteur de logs.
- `screen: Bool` Active l'enregistreur d'écran.
