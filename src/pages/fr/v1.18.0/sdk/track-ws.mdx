Si les informations enregistrées par défaut par notre tracker lorsqu'une erreur JS se produit ne suffisent pas, vous pouvez utiliser cette méthode. La méthode `handleError` signale manuellement les exceptions interceptées, les promesses rejetées ou les événements d'erreur. Consultez [le signalement des erreurs](/installation/error-reporting#manuallyloggingexceptions) pour plus de détails.

## Signature

```ts 
trackWs: (channelName: string) => (msgType: string, data: string, dir: 'up' | 'down') => void
```

## Paramètres

- `channelName: string` : Nom du canal WS.

## Valeur de retour

Renvoie un hook à appliquer aux messages ws qui prend comme paramètres le type de message (ou nom), les données et la direction du message (up/down === envoyé/reçu).
