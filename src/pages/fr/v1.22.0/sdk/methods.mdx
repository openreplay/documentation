## Méthodes

- [Constructor](/sdk/constructor): Découvrez toutes les options d'initialisation du tracker.
- [event()](/sdk/methodsevent): Envoyez des événements personnalisés afin qu'ils soient enregistrés dans vos rejeux avec cette méthode.
- [getSessionToken()](/sdk/methods/get-session-token): Obtenez le jeton de la session active actuelle.
- [getSessionID()](/sdk/methods/get-session-id): Obtenez l'ID de la session active actuelle.
- [getSessionURL()](/sdk/methods/get-session-url): Obtenez l'URL permettant de rejouer la session actuelle. 
- [handleError()](/sdk/methods/handle-error): Capturez des informations d'erreur personnalisées et enregistrez-les dans votre enregistrement.
- [isActive()](/sdk/methods/is-active): Obtenez l'état actuel de la session active.
- [issue()](/sdk/methods/issue): Capturez des événements techniques personnalisés et enregistrez-les dans le rejeu.
- [setUserID()](/sdk/methods/set-user-id): Identifiez vos utilisateurs avec cette méthode.
- [setMetadata()](/sdk/methods/set-metadata): Définissez un champ de métadonnées et enregistrez-le dans l'enregistrement.
- [start()](/sdk/methods/start): Démarrez l'enregistrement d'une session.
- [stop()](/sdk/methods/stop): Arrêtez l'enregistrement d'une session avec cette méthode.
- [clearPersistFlag()](/sdk/methods/clear-persist-flag): Supprime les Feature Flags persistants du sessionStorage du navigateur.
- [getFeatureFlag()](/sdk/methods/get-feature-flag): Renvoie un `IFeatureFlag` par clé s'il existe.
- [onFlagsReload()](/sdk/methods/on-flags-reload): Définit le callback à utiliser lorsque les Feature Flags sont chargés.
- [isFlagEnabled()](/sdk/methods/is-flag-enabled): Renvoie `true` si le flag portant cette clé existe et est actif.
- [reloadFlags()](/sdk/methods/reload-flags): Recharge tous les flags (peut être utile si la session de l'utilisateur a connu un changement de métadonnées, de userId ou de toute autre information pouvant être requise pour certaines conditions de flag).
- [getAllFeatureFlags()](/sdk/methods/get-all-feature-flags): Renvoie tous les flags (`IFeatureFlag[]`) qui sont actifs en ce moment.
- [coldStart()](/sdk/methods/cold-start): Commence à mettre les messages en mémoire tampon (en actualisant la mémoire tampon) avec la possibilité d'appeler `start()` pour activer l'enregistrement de la session et envoyer le contenu de la mémoire tampon.
- [startOfflineRecording()](/sdk/methods/start-offline-recording): Démarre l'enregistrement de session hors ligne qui peut être envoyé avec `uploadOfflineRecording`.
- [uploadOfflineRecording()](/sdk/methods/upload-offline-recording): Envoie la mémoire tampon de session stockée vers le backend.
- [forceFlushBatch()](/sdk/methods/force-flush-batch): Force l'envoi du lot de messages actuel sans attendre sa finalisation.
- [trackWs()](/sdk/methods/track-ws): Crée un hook pour capturer les messages WebSocket.
