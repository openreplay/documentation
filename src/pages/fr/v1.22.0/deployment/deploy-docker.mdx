<Aside type="tip">
	Il s'agit d'une fonctionnalité expérimentale et son implémentation est susceptible d'évoluer à l'avenir. OpenReplay ne peut pas fonctionner en local, donc même avec Docker, vous aurez toujours besoin d'un serveur. Découvrez nos autres [options de déploiement](/deployment).
</Aside>

La stack OpenReplay peut être installée sur une seule machine avec Docker Compose. 

## Prérequis

1. Ubuntu 20.04 LTS (la version LTS est recommandée)
2. IP publique attachée à la machine. Cela signifie que votre serveur est accessible depuis Internet.
3. Vous devez avoir un nom de domaine associé à l'adresse IP.
4. Choisissez votre type d'instance. Les spécifications minimales sont `2 vCPUs, 8 GB of RAM, 50 GB of storage`, ainsi qu'une architecture x86, faute de quoi les services backend d'OpenReplay ne démarreront tout simplement pas. Nous recommandons donc **au moins** une `t3.large` (ou un équivalent), ce qui est suffisant pour un volume faible/modéré. Si vous prévoyez un trafic élevé, vous devriez monter en gamme à partir de là.

## Déployer OpenReplay

```bash
## From your terminal
SSH_KEY=~/Downloads/openreplay-key.pem #! wherever you've saved the SSH key
INSTANCE_IP=REPLACE_WITH_INSTANCE_PUBLIC_IP
chmod 400 $SSH_KEY
ssh -i $SSH_KEY ubuntu@$INSTANCE_IP
```

2. Installez OpenReplay en indiquant le domaine sur lequel il s'exécutera (par ex. DOMAIN_NAME=openreplay.mycompany.com) :

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/openreplay/openreplay/main/scripts/docker-compose/docker-install.sh)"
```

Tout est prêt maintenant, OpenReplay devrait être accessible de manière sécurisée sur le domaine que vous avez créé en premier. Vous pouvez créer un compte en visitant la page `/signup` (c.-à-d. openreplay.mycompany.com/signup).
