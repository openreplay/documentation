Les événements personnalisés sont parfaits pour ajouter de l'intelligence en enregistrant des événements spécifiques à votre domaine en parallèle des relectures de session. OpenReplay utilise 2 types d'événements : fonctionnels (`event`) et techniques (`issue`). Tous les événements sont indexés pour faciliter la recherche et synchronisés avec les enregistrements de session.

<Aside type="tip">
  Nous avons également un tutoriel détaillé sur la façon d'envoyer des événements personnalisés, vous pouvez le consulter [ici](/tutorials/custom-events).
</Aside>

## Événements fonctionnels

Les événements fonctionnels sont indexés et facilitent la recherche d'enregistrements de session spécifiques. S'ils ont été correctement reçus par OpenReplay, ils seront disponibles en tant que filtres dans la barre omnisearch.

![Functional Event](/static/custom-events-1.png#center)

### Avec NPM

Utilisez la méthode `event` pour signaler des événements fonctionnels tels que *order completed* ou *product added*. Elle prend 2 paramètres : *name* (string) et *payload* (objet JS non récursif).

```js
tracker.event('product_added', 'shoes'); // after tracker.start()
```

## Événements techniques

Les événements techniques sont affichés dans la relecture de session sous l'onglet *Events* des DevTools et sous forme d'annotations sur la lecture. Ils sont également pris en compte dans les Funnels pour corréler les baisses de conversion avec des problèmes techniques.

![Technical Event](/static/custom-events-2.png#center)

### Avec NPM

`issue` est utilisé pour envoyer des événements techniques, comme des erreurs, qui peuvent survenir dans votre stack ou d'autres systèmes en aval. `issue` prend 2 paramètres : *name* (string) et *payload* (objet JS non récursif).

```js
tracker.issue('payment_error', {code: 500, context: 42}); // after tracker.start()
```
