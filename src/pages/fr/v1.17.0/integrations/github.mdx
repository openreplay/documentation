Intégrez GitHub à OpenReplay et créez des tickets directement depuis la page d'enregistrement.

## Avec OpenReplay OpenSource

### 1. Créer un jeton d'accès personnel

1. Connectez-vous à Github puis allez dans [Settings > Developers Settings > Personal access tokens](https://github.com/settings/tokens)
2. Cliquez sur 'Generate a new token'
3. Sélectionnez les portées `repo:status`, `repo_deployment`,`public_repo`, `read:user` et `user:email`
4. Cliquez sur 'Generate token' puis copiez le jeton

### 2. Activer GitHub dans OpenReplay

1. Allez dans le tableau de bord OpenReplay puis dans 'Preferences > Integration'
2. Sélectionnez GitHub. Si vous ne le voyez pas, c'est que vous avez dû activer Jira (vous ne pouvez pas utiliser les deux en même temps, alors assurez-vous de désactiver d'abord l'intégration Jira).
3. Collez le `token` généré précédemment

Un bouton 'Create Issue' apparaîtra alors dans tous les enregistrements afin que vous puissiez rapidement créer et attribuer des tickets dans GitHub.

## Avec OpenReplay Cloud

Allez dans 'Preferences > Integration' et cliquez sur GitHub. Vous serez redirigé vers GitHub pour l'authentification. C'est tout ce que vous avez à faire. Un bouton 'Create Issue' apparaîtra alors dans tous les enregistrements afin que vous puissiez rapidement créer et attribuer des tickets dans GitHub.
