Faire fonctionner le tracker sur une application React est plutôt simple. 

## Ajouter le code de suivi

Bien que vous puissiez [obtenir une version à copier-coller du code directement depuis la plateforme lorsque vous créez un nouveau projet](/using-or/snippet#getting-the-snippet), ce code est trop minimaliste et rendra beaucoup plus difficile sa personnalisation et l'ajout de plugins supplémentaires.

En supposant que vous construisez une SPA (Single Page Application), il vous suffit d'ajouter le code suivant à votre point d'entrée :

```jsx
import Tracker from '@openreplay/tracker';

const tracker = new Tracker({
  projectKey: "<your project key>",  
});

function YourMainComponent({props}) {
    //some code here...

    useEffect(() => {
        tracker.start();
    }, [])

    return //....
}
```

Autrement dit, vous pouvez instancier le tracker au début du script et, une fois la page chargée, vous pouvez le démarrer à l'aide d'un hook `useEffect`.

<Aside type="tip">
**Une note pour les utilisateurs en self-hosted** : si vous utilisez la version self-hosted d'OpenReplay, vous devrez également spécifier la propriété de configuration `ingestPoint`. 
Cette propriété indique le point d'ingestion (endpoint) des données du tracker. Les utilisateurs du cloud n'en ont pas besoin, 
car par défaut le tracker saura où se trouve la version SaaS de cet endpoint, mais si vous l'hébergez vous-même (self-hosting), vous devrez le spécifier 
(cela devrait ressembler à `https://openreplay.mydomain.com/ingest`) 
</Aside>

### Gérer la « projectKey » dans votre code

À noter : la `projectKey` que vous configurez est celle fournie par la plateforme d'OpenReplay. Pour des raisons de sécurité, cette valeur ne doit pas être stockée en dur (hardcoded) dans votre code, mais plutôt dans une sorte de fichier/système de configuration.

Il existe différentes options pour résoudre ce problème ; selon votre configuration, vous pourriez utiliser quelque chose comme la [prise en charge des variables d'environnement (ENV) par `create-react-app`](https://create-react-app.dev/docs/adding-custom-environment-variables/).
