Intégrez GitHub à OpenReplay et créez des issues directement depuis la page d'enregistrement.

## Avec OpenReplay Open Source

#### 1. Créez un jeton d'accès personnel

1. Connectez-vous à Github puis accédez à [Settings > Developers Settings > Personal access tokens](https://github.com/settings/tokens)
2. Cliquez sur 'Generate a new token'
3. Sélectionnez les scopes `repo:status`, `repo_deployment`,`public_repo`, `read:user` et `user:email`
4. Cliquez sur 'Generate token' puis copiez le jeton

#### 2. Activez GitHub dans OpenReplay

1. Accédez au tableau de bord OpenReplay puis à 'Preferences > Integration'
2. Sélectionnez GitHub. Si vous ne le voyez pas, c'est que vous avez probablement activé Jira (vous ne pouvez pas utiliser les deux en même temps, alors assurez-vous de désactiver d'abord l'intégration Jira).
3. Collez le `token` généré précédemment

Un bouton 'Create Issue' apparaîtra alors dans tous les enregistrements afin que vous puissiez créer et assigner rapidement des issues dans GitHub.

## Avec OpenReplay Cloud

Accédez à 'Preferences > Integration' et cliquez sur GitHub. Vous serez redirigé vers GitHub pour l'authentification. C'est tout ce que vous avez à faire. Un bouton 'Create Issue' apparaîtra alors dans tous les enregistrements afin que vous puissiez créer et assigner rapidement des issues dans GitHub.
