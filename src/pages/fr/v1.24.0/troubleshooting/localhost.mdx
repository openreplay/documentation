OpenReplay a besoin d'un accès public (HTTPS uniquement) aux ressources de votre application (CSS, polices et icônes) pour restituer correctement les enregistrements. Nous copions ces ressources afin que vous puissiez toujours rejouer vos sessions même si le contenu de votre application a changé. Les images sont gérées différemment : elles sont récupérées en temps réel lorsqu'une session est rejouée.

Pour ces raisons, nous recommandons de tester l'installation dans un environnement accessible publiquement (c'est-à-dire un environnement de préproduction) plutôt qu'en localhost. Il existe néanmoins une solution de contournement pour le développement local, même si elle n'est pas parfaite : ajoutez simplement l'option `inlineCss: 3` à votre [constructeur du tracker](/sdk/constructor/#stylesheets) afin que les feuilles de style soient alors capturées dans chaque enregistrement (au lieu d'être mises en cache de manière centralisée par le serveur). Les enregistrements de sessions devraient ensuite être disponibles dans le tableau de bord, mais il se peut que des images manquent, comme expliqué ci-dessus.

> Remarque : Vous devrez peut-être également activer l'option `__DISABLE_SECURE_MODE: true` pour autoriser une connexion non sécurisée entre le tracker et le backend sur les sites sans SSL.
