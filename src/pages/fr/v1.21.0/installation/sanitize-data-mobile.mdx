Cette page explique comment nettoyer les composants de l'interface utilisateur dans vos applications mobiles à l'aide des SDK OpenReplay pour iOS et Android. Pour plus de détails sur le nettoyage des requêtes réseau, consultez [Options réseau (mobile)](/network-options-mobile).

## iOS - Nettoyage des composants de l'interface utilisateur

Les vues nettoyées seront floutées dans la relecture résultante. Les composants sont automatiquement supprimés (afin de préserver la mémoire) dès qu'ils ne sont plus visibles.

### SwiftUI

Ajoutez le modificateur `.sensitive()` au composant que vous souhaitez nettoyer.

```swift
import ORTracker

Text("Sensitive Information")
    .sensitive()
```

### UIKit

Utilisez la méthode `addIgnoredView` pour marquer une vue comme sensible.

```swift
import ORTracker

ORTracker.shared.addIgnoredView(view)
```

### React Native

Encapsulez les composants sensibles avec le composant `ORSanitizedView` pour les flouter dans la relecture.

```js
import OR from '@openreplay/react-native';

<OR.ORSanitizedView>
  <SensitiveComponent />
</OR.ORSanitizedView>
```

## Android - Nettoyage des composants de l'interface utilisateur

Les vues nettoyées sont floutées dans la relecture résultante et automatiquement supprimées de la mémoire lorsqu'elles sont hors écran.

### View

Marquez une vue comme sensible à l'aide de la méthode `addIgnoredView`.

```kotlin
import com.openreplay.tracker.OpenReplay

OpenReplay.addIgnoredView(view)
```
