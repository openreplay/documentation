Avec le tracker OpenReplay configuré et capturant les sessions des utilisateurs, vous êtes prêt à découvrir ce qui se passe réellement dans votre produit. OmniSearch vous aide à déboguer les problèmes, à identifier ce qui ne va pas et à comprendre en détail le comportement des utilisateurs. Il comble le fossé entre les données quantitatives et les informations qualitatives cachées dans vos rejeux de session.

## Comprendre les événements

Les **événements** dans OmniSearch représentent les interactions utilisateur et les comportements du système qui se produisent au cours d'une session.

Ils sont répartis en deux catégories principales : les **événements capturés automatiquement** (suivi automatique des interactions utilisateur) et les **événements DevTools** (surveillance technique et débogage).

### Événements capturés automatiquement
| **Nom de l'événement**  | **Description** |
| --- | --- |
| Click | Enregistre les clics de l'utilisateur. |
| Text Input | Suit le texte saisi dans les champs. |
| Visited URL | Journalise les pages visitées |
| Tagged Element | Suit les éléments balisés de manière personnalisée. |

### Événements DevTools

|  **Nom de l'événement**  | **Description** |
| --- | --- |
| Network Request | Surveille l'activité réseau. |
| GraphQL | Journalise les requêtes et réponses GraphQL. |
| State Action | Suit les changements d'état dans votre application. |
| Error Message | Capture les messages d'erreur dans la console. |
| Average CPU Load | Mesure l'utilisation moyenne du CPU. |
| Average Memory Usage | Suit la consommation moyenne de mémoire. |

## Comprendre les filtres

Les **filtres** dans OmniSearch vous aident à affiner les rejeux de session selon des critères spécifiques, vous permettant de vous concentrer sur les sessions utilisateur les plus pertinentes. Ils fonctionnent conjointement avec les événements pour créer des recherches précises dans les données de vos sessions.

Les filtres sont organisés en quatre catégories principales :

### Filtres utilisateur

|  **Nom du filtre**  | **Description** |
| --- | --- |
| User ID | Filtre sur des utilisateurs spécifiques à l'aide de leurs identifiants uniques. |
| Anonymous User ID | Filtre les sessions des utilisateurs sans identité connectée. |

### Filtres de session

|  **Nom du filtre**  | **Description** |
| --- | --- |
| Country | Filtre les sessions par pays. |
| City | Filtre les sessions par ville. |
| State/Province | Filtre les sessions par état ou province. |
| Operating System (OS) | Filtre les sessions par le système d'exploitation utilisé. |
| Browser | Filtre les sessions par le type de navigateur. |
| Device | Filtre les sessions par l'appareil de l'utilisateur. |
| Platform | Filtre les sessions par le type de plateforme. |
| Version ID | Filtre les sessions par la version de l'application. |
| Referrer | Filtre les sessions en fonction de l'URL référente. |
| Duration | Filtre les rejeux de session par leur durée. |
| UTM Source | Filtre les sessions par le paramètre UTM source. |
| UTM Medium | Filtre les sessions par le paramètre UTM medium. |
| UTM Campaign | Filtre les sessions par le paramètre UTM campaign. |

### Filtres de problèmes

|  **Nom du filtre**  | **Description** |
| --- | --- |
| Rage Click, Dead Click, Excessive Scrolling, Mouse Thrashing | Identifient les sessions présentant une frustration de l'utilisateur causée par des clics répétés, des boutons ou liens cassés, un défilement excessif, des mouvements de souris erratiques…. |
| Bad Request | Identifie les sessions avec des erreurs HTTP 400. |
| Missing Resource | Trouve les sessions avec des ressources manquantes |
| CPU Issues | Identifie les sessions présentant une utilisation élevée du CPU. |
| Slow Resource | Identifie les sessions avec des ressources à chargement lent. |
| Slow Page Load | Identifie les sessions avec des temps de chargement de page lents. |
| Crash | Identifie les sessions où l'application ou la page a planté. |
| Errors | Identifie les sessions présentant des erreurs d'application. |

### Filtres de métadonnées

|  **Nom du filtre**  | **Description** |
| --- | --- |
| Metadata key | Utilisez des métadonnées personnalisées pour filtrer les sessions. Si elles ne sont pas définies, découvrez comment ajouter des métadonnées [ici](/tutorials/metadata) |
