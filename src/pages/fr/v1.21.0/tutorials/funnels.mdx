Trouver des problèmes n'est parfois pas aussi simple que de revoir un seul enregistrement. Parfois, les problèmes surviennent lorsque l'utilisateur traverse une série d'étapes.

Lorsque ces étapes mènent l'utilisateur à un résultat souhaité (comme acheter l'un de vos produits ou s'abonner à votre newsletter), elles forment ce que l'on appelle un « entonnoir ».

OpenReplay vous permet d'analyser et de suivre des entonnoirs personnalisés afin de comprendre pourquoi vos utilisateurs en sortent sans convertir.

Dans ce tutoriel, nous verrons comment configurer un entonnoir pour comprendre combien d'utilisateurs notre site e-commerce perd en passant de la page d'accueil à l'ajout d'un produit dans leur panier.

## Créer votre premier entonnoir

**Notez** que vous n'avez besoin d'aucun plugin ou module complémentaire spécial pour capturer les données nécessaires. Avec une configuration de base du tracker, vous aurez tout ce qu'il vous faut.

Les entonnoirs sont un type de métrique que vous pouvez créer dans la section Dashboards de la navigation supérieure :

![Finding the metrics section](/tutorials//funnels/getting-to-metrics.png)

Une fois dans cette section, cliquez sur « Create Metric » et choisissez le type de métrique « Funnel ».

Commencez ensuite à ajouter des étapes à l'aide des filtres ci-dessous. Voici comment notre entonnoir suit les utilisateurs depuis d'autres pages d'accueil jusqu'à l'ajout d'un produit dans leur panier :

![Filters for creating a funnel](/tutorials//funnels/funnel-filters.png)

Remarquez les événements à l'intérieur du filtre « Series 1 » :

1. D'abord, assurez-vous que le chemin est « /es », ce qui correspond à la page d'accueil de notre site.
2. Vérifiez ensuite si le chemin commence par « /es/product », ce qui désigne chacun des produits de notre site web.
3. Enfin, récupérez les sessions qui cliquent sur le bouton « Add to Cart ».

Puisque nous définissons un entonnoir, chaque nouveau filtre sera plus restrictif.

Sous les filtres, vous verrez les résultats de chaque étape, comme ceci :

![The funnel visually represented](/tutorials//funnels/funnel-visual.png)

Vous pouvez voir que nous perdons 5 utilisateurs en passant de l'étape 2 à l'étape 3.

Et parmi tous ceux qui atteignent l'étape 3, seuls 50 % d'entre eux cliquent effectivement sur le bouton. Alors, qu'est-ce qui pourrait causer cette chute soudaine de la conversion ? Nous avons quelques indices sous l'entonnoir.

![Errors inside the funnel](/tutorials//funnels/funnel-errors.png)

Nous avons des problèmes de CPU élevé, des clics morts et même quelques événements personnalisés qui pourraient nous donner des indices sur ce qui se passe.

Les problèmes fournissent un aperçu de leur effet à l'intérieur de l'entonnoir, ce qui vous permet de les examiner et de vous concentrer sur les pires.

Si vous cliquez sur l'une de ces erreurs, vous obtiendrez une liste des enregistrements affectés par celle-ci à l'intérieur de l'entonnoir, afin que vous puissiez analyser les bons enregistrements :

![Details inside a funnel error](/tutorials//funnels/funnel-details.png)

Vous pouvez maintenant ajouter la métrique à un ou plusieurs dashboards pour l'utiliser.

## Ajouter votre métrique à un dashboard

Pour ajouter votre toute nouvelle métrique à un dashboard, rendez-vous dans la section Dashboards, choisissez celui sur lequel vous souhaitez travailler, puis cliquez sur « Add Metric ».


![Adding a metric](/tutorials//funnels/add-metric-button.png)
Sélectionnez la nouvelle métrique dans la section Custom, puis cliquez sur « Add Selected to Dashboard » :

![Adding the metric to the dashboard](/tutorials//funnels/adding-metric-to-dashboard.png)
