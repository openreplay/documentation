Si les informations suivies par défaut par notre tracker lorsqu'une erreur JS se produit ne suffisent pas, vous pouvez utiliser cette méthode. La méthode `handleError` signale manuellement les exceptions capturées, les promesses rejetées ou les événements d'erreur. Consultez le [signalement des erreurs](/installation/error-reporting#manuallyloggingexceptions) pour plus de détails.

## Signature

```js 
handleError: (e: Error | ErrorEvent | PromiseRejectionEvent, metadata?: Record<string, any>) => void
```

## Paramètres

- `e: Error | ErrorEvent | PromiseRejectionEvent` : L'objet d'erreur réel qui est capturé.
- `metadata: Record<string, any>` : **[Optionnel]** Une structure de type dictionnaire contenant toutes les informations supplémentaires que vous souhaitez enregistrer en association avec cette erreur.

## Valeur de retour

Cette méthode ne renvoie aucune valeur ; les informations enregistrées peuvent être consultées dans la plateforme OpenReplay.
