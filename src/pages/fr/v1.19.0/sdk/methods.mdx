## Méthodes

- [Constructor](/sdk/constructor): Découvrez tout sur les options d'initialisation du tracker.
- [event()](/sdk/event): Envoyez des événements personnalisés afin qu'ils soient enregistrés dans vos relectures avec cette méthode.
- [getSessionToken()](/sdk/get-session-token): Obtenez le token de la session actuellement active.
- [getSessionID()](/sdk/get-session-id): Obtenez l'ID de la session actuellement active.
- [getSessionURL()](/sdk/get-session-url): Obtenez l'url pour rejouer la session actuelle. 
- [handleError()](/sdk/handle-error): Capturez des informations d'erreur personnalisées et enregistrez-les dans votre enregistrement.
- [isActive()](/sdk/is-active): Obtenez l'état actuel de la session active.
- [issue()](/sdk/issue): Capturez des événements techniques personnalisés et enregistrez-les dans la relecture.
- [setUserID()](/sdk/set-user-id): Identifiez vos utilisateurs avec cette méthode.
- [setMetadata()](/sdk/set-metadata): Définissez un champ de métadonnées et enregistrez-le dans l'enregistrement.
- [start()](/sdk/start): Démarrez l'enregistrement d'une session.
- [stop()](/sdk/stop): Arrêtez l'enregistrement d'une session avec cette méthode.
- [clearPersistFlag()](/sdk/clear-persist-flag): Supprime les Feature Flags persistants du sessionStorage du navigateur.
- [getFeatureFlag()](/sdk/get-feature-flag): Renvoie un `IFeatureFlag` par clé s'il existe.
- [onFlagsReload()](/sdk/on-flags-reload): Définit la callback à utiliser lorsque les Feature Flags sont chargés.
- [isFlagEnabled()](/sdk/is-flag-enabled): Renvoie `true` si le flag portant cette clé existe et est actif.
- [reloadFlags()](/sdk/reloadFlags): Recharge tous les flags (cela peut être utile si la session de l'utilisateur a connu un changement de métadonnées, de userId ou de toute autre information susceptible d'être requise pour certaines conditions de flags).
- [getAllFeatureFlags()](/sdk/get-all-feature-flags): Renvoie tous les flags (`IFeatureFlag[]`) actuellement actifs.
- [coldStart()](/sdk/cold-start): Commence à mettre les messages en tampon (en rafraîchissant le tampon) avec la possibilité d'appeler `start()` pour activer l'enregistrement de la session et envoyer le contenu du tampon.
- [startOfflineRecording()](/sdk/start-offline-recording): Démarre l'enregistrement de session hors ligne, qui peut être envoyé avec `uploadOfflineRecording`.
- [uploadOfflineRecording()](/sdk/upload-offline-recording): Téléverse le tampon de session stocké vers le backend.
- [forceFlushBatch()](/sdk/force-flush-batch): Force l'envoi du lot de messages actuel sans attendre son achèvement.
- [trackWs()](/sdk/track-ws): Crée un hook pour capturer les messages WebSocket.
