Cette méthode est utilisée pour démarrer le tracker. En démarrant le tracker, l'enregistrement de la session commence.

## Signature

```js
start(startOpts?: Partial<StartOptions>): Promise<StartPromiseReturn>
```

### Paramètres
La méthode `start` accepte un paramètre optionnel avec lequel vous pouvez personnaliser différents aspects de l'enregistrement :
- `userID: string` : utilisé pour définir manuellement l'ID utilisateur afin de le suivre à travers les sessions. C'est une valeur de type chaîne de caractères et elle peut être ce que vous voulez.
- `metadata: Record<string, string>` : définit manuellement les valeurs de métadonnées. Consultez la [section Metadata](/installation/metadata/) pour en savoir plus sur cette fonctionnalité.
- `forceNew: boolean` : utilisé pour forcer une nouvelle session après le rechargement de la page. Par défaut, il est défini sur `false`, donc après un rechargement la session est conservée.
- `sessionToken: string` : utilisé dans certaines intégrations.
- `assistOnly: boolean` : lance le tracker en mode assistance uniquement, ce qui ignorera l'envoi des données de session au backend (fonctionnalité de l'édition EE)

### Valeur de retour

Une fois démarrée, la méthode renverra une promesse contenant un objet d'informations de session comprenant :
- `sessionID: string` : Une valeur de type chaîne de caractères représentant l'ID de la session démarrée.
Démarre le tracker (et le processus d'enregistrement) puis renvoie une promesse contenant les informations de session (sessionID, sessionToken, userUUID).
